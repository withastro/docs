Proyek Astro baru Anda yang dibuat dari wizard CLI `create astro` sudah menyertakan beberapa file dan folder. Sisanya, Anda akan membuat sendiri dan menambahkannya ke struktur file Astro yang ada.

Berikut cara sebuah proyek Astro diorganisasi, dan beberapa file yang akan Anda temukan di proyek baru.

## Direktori dan File

Astro memanfaatkan layout folder yang “opinionated” untuk proyek Anda. Root proyek Astro seharusnya menyertakan direktori dan file berikut:

- `src/*` - Kode sumber proyek Anda (komponen, halaman, style, gambar, dll.)
- `public/*` - Aset non-kode yang tidak diproses (font, ikon, dll.)
- `package.json` - Manifest proyek.
- `astro.config.mjs` - File konfigurasi Astro. (direkomendasikan)
- `tsconfig.json` - File konfigurasi TypeScript. (direkomendasikan)

### `src/`

Folder `src/` adalah tempat sebagian besar kode sumber proyek Anda berada. Ini termasuk:

- [Halaman](/id/basics/astro-pages/)
- [Layout](/id/basics/layouts/)
- [Komponen Astro](/id/basics/astro-components/)
- [Komponen kerangka UI (React, dll.)](/id/guides/framework-components/)
- [Style (CSS, Sass)](/id/guides/styling/)
- [Markdown](/id/guides/markdown-content/)
- [Gambar yang akan dioptimalkan dan diproses oleh Astro](/id/guides/images/)

Astro memproses, mengoptimalkan, dan membundel file `src/` Anda untuk membuat situs akhir yang dikirim ke browser. Berbeda dengan direktori statis `public/`, file `src/` dibangun dan ditangani oleh Astro.

Beberapa file (seperti komponen Astro) bahkan tidak dikirim ke browser sebagaimana ditulis melainkan dirender menjadi HTML statis. File lain (seperti CSS) dikirim ke browser namun bisa dioptimalkan atau dibundel dengan CSS lain demi performa.

:::tip
Walau panduan ini menjelaskan beberapa konvensi populer yang digunakan komunitas Astro, satu-satunya direktori yang “dipesan” oleh Astro adalah `src/pages/`. Anda bebas mengganti nama dan mengorganisasi direktori lain dengan cara yang paling cocok bagi Anda.
:::

### `src/pages`

Route halaman dibuat untuk situs Anda dengan menambahkan [tipe file yang didukung](/id/basics/astro-pages/#supported-page-files) ke direktori ini.

:::caution
`src/pages` adalah subdirektori yang **wajib** ada pada proyek Astro Anda. Tanpanya, situs Anda tidak akan memiliki halaman atau route!
:::

### `src/components`

**Komponen** adalah unit kode yang dapat digunakan ulang untuk halaman HTML Anda. Ini bisa berupa [komponen Astro](/id/basics/astro-components/), atau [komponen kerangka UI](/id/guides/framework-components/) seperti React atau Vue. Umum untuk mengelompokkan dan mengorganisasi semua komponen proyek di folder ini.

Ini adalah konvensi umum di proyek Astro, tetapi tidak wajib. Silakan atur komponen Anda sesuka hati!

### `src/layouts`

[Layout](/id/basics/layouts/) adalah komponen Astro yang mendefinisikan struktur UI yang dibagikan satu atau lebih [halaman](/id/basics/astro-pages/).

Seperti `src/components`, direktori ini adalah konvensi umum namun tidak wajib.

### `src/styles`

Umum untuk menyimpan file CSS atau Sass Anda di direktori `src/styles`, tetapi ini tidak wajib. Selama style Anda berada di `src/` dan diimpor dengan benar, Astro akan menangani dan mengoptimalkannya.

### `public/`

Direktori `public/` adalah untuk file dan aset di proyek Anda yang tidak perlu diproses selama build oleh Astro. File di folder ini disalin ke folder build apa adanya, lalu situs Anda akan dibangun.

Perilaku ini membuat `public/` ideal untuk aset umum yang tidak memerlukan pemrosesan, seperti beberapa gambar dan font, atau file khusus seperti `robots.txt` dan `manifest.webmanifest`.

Anda dapat menaruh CSS dan JavaScript di direktori `public/`, tetapi ketahuilah bahwa file-file tersebut tidak akan dibundel atau dioptimalkan dalam build akhir.

:::tip
Sebagai aturan umum, CSS atau JavaScript apa pun yang Anda tulis sendiri sebaiknya berada di direktori `src/`.
:::

### `package.json`

Ini adalah file yang digunakan oleh *package manager* JavaScript untuk mengelola dependency. File ini juga mendefinisikan skrip yang umum digunakan untuk menjalankan Astro (contoh: `npm run dev`, `npm run build`).

Ada [dua jenis dependency](https://docs.npmjs.com/specifying-dependencies-and-devdependencies-in-a-package-json-file) yang bisa Anda tentukan di `package.json`: `dependencies` dan `devDependencies`. Dalam banyak kasus, keduanya bekerja sama: Astro membutuhkan semua dependency saat build, dan package manager Anda akan memasang keduanya. Kami sarankan menaruh semua dependency di `dependencies` terlebih dahulu, dan hanya gunakan `devDependencies` jika memang ada kebutuhan khusus.

Untuk bantuan membuat file `package.json` baru, lihat instruksi [penyiapan manual](/id/install-and-setup/#manual-setup).

### `astro.config.mjs`

File ini dibuat di setiap template starter dan menyertakan opsi konfigurasi untuk proyek Astro Anda. Di sini Anda bisa menentukan integrasi yang digunakan, opsi build, opsi server, dan banyak lagi.

Astro mendukung beberapa format file untuk file konfigurasi JavaScript-nya: `astro.config.js`, `astro.config.mjs`, `astro.config.cjs`, dan `astro.config.ts`. Kami menyarankan menggunakan `.mjs` pada sebagian besar kasus atau `.ts` jika ingin menulis TypeScript di file konfigurasi.

Pemanggilan file konfigurasi TypeScript ditangani menggunakan [`tsm`](https://github.com/lukeed/tsm) dan akan menghormati opsi `tsconfig` proyek Anda.

Lihat [referensi konfigurasi](/id/reference/configuration-reference/) untuk detail lengkap.

### `tsconfig.json`

File ini dibuat di setiap template starter dan menyertakan opsi konfigurasi TypeScript untuk proyek Astro Anda. Beberapa fitur (seperti impor paket npm) tidak sepenuhnya didukung di editor tanpa file `tsconfig.json`. 

Lihat [Panduan TypeScript](/id/guides/typescript/) untuk detail pengaturan konfigurasi.
