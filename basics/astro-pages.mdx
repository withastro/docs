**Halaman** adalah file yang berada di subdirektori `src/pages/` pada proyek Astro Anda. File-file ini bertanggung jawab menangani routing, pemuatan data, dan layout halaman secara keseluruhan untuk setiap halaman di situs Anda.

## File halaman yang didukung

Astro mendukung tipe file berikut di direktori `src/pages/`:

- [`.astro`](#astro-pages)
- [`.md`](#markdownmdx-pages)
- `.mdx` (dengan [MDX Integration terpasang](/id/guides/integrations-guide/mdx/#installation))
- [`.html`](#html-pages)
- `.js`/`.ts` (sebagai [endpoint](/id/guides/endpoints/))

## Routing berbasis file

Astro memanfaatkan strategi routing yang disebut **file-based routing**. Setiap file di direktori `src/pages/` menjadi endpoint di situs Anda berdasarkan path filenya.

Satu file juga dapat menghasilkan banyak halaman menggunakan [routing dinamis](/id/guides/routing/#dynamic-routes). Ini memungkinkan Anda membuat halaman meski konten berada di luar direktori khusus `/pages/`, seperti pada [content collection](/id/guides/content-collections/) atau [CMS](/id/guides/cms/).

<ReadMore>Baca lebih lanjut tentang [Routing di Astro](/id/guides/routing/).</ReadMore>

### Tautan antar halaman

Tulislah [elemen `<a>` HTML](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/a) standar di halaman Astro Anda untuk menautkan ke halaman lain di situs. Gunakan **path URL relatif ke domain root** sebagai tautan Anda, bukan path file relatif.

Sebagai contoh, untuk menautkan ke `https://example.com/authors/sonali/` dari halaman mana pun di `example.com`:

```astro title="src/pages/index.astro"
Read more <a href="/authors/sonali/">about Sonali</a>.
```

## Halaman Astro

Halaman Astro menggunakan ekstensi file `.astro` dan mendukung fitur yang sama seperti [komponen Astro](/id/basics/astro-components/).

```astro title="src/pages/index.astro"
---
---
<html lang="en">
  <head>
    <title>My Homepage</title>
  </head>
  <body>
    <h1>Welcome to my website!</h1>
  </body>
</html>
```

Sebuah halaman harus menghasilkan dokumen HTML penuh. Jika tidak disertakan secara eksplisit, Astro akan menambahkan deklarasi `<!DOCTYPE html>` dan konten `<head>` yang diperlukan ke komponen `.astro` apa pun yang berada di `src/pages/` secara bawaan. Anda dapat memilih keluar dari perilaku ini per komponen dengan menandainya sebagai [partial](#page-partials).

Untuk menghindari pengulangan elemen HTML yang sama di setiap halaman, Anda dapat memindahkan elemen `<head>` dan `<body>` umum ke [komponen layout](/id/basics/layouts/) Anda sendiri. Anda bebas menggunakan sedikit atau banyak layout sesuai kebutuhan.

```astro title="src/pages/index.astro" /</?MySiteLayout>/ {2}
---
import MySiteLayout from "../layouts/MySiteLayout.astro";
---
<MySiteLayout>
  <p>Konten halaman saya, dibungkus oleh layout!</p>
</MySiteLayout>
```

<ReadMore>Baca lebih lanjut tentang [komponen layout](/id/basics/layouts/) di Astro.</ReadMore>

## Halaman Markdown/MDX

Astro juga memperlakukan file Markdown (`.md`) apa pun di dalam `src/pages/` sebagai halaman di situs akhir Anda. Jika Anda memasang [MDX Integration](/id/guides/integrations-guide/mdx/#installation), file MDX (`.mdx`) juga diperlakukan sama.

:::tip
Pertimbangkan untuk membuat [content collections](/id/guides/content-collections/) alih-alih halaman untuk direktori file Markdown terkait yang berbagi struktur serupaâ€”seperti posting blog atau item produk.
:::

File Markdown dapat menggunakan properti frontmatter khusus `layout` untuk menentukan [komponen layout](/id/basics/layouts/) yang akan membungkus konten Markdown mereka dalam dokumen halaman `<html>...</html>` penuh.

```md {3}
---
# Contoh: src/pages/page.md
layout: ../layouts/MySiteLayout.astro
title: Halaman Markdown Saya
---
# Judul

Ini halaman saya, ditulis dengan **Markdown.**
```

## Halaman HTML

File dengan ekstensi `.html` dapat ditempatkan di direktori `src/pages/` dan digunakan langsung sebagai halaman di situs Anda. Perhatikan bahwa beberapa fitur Astro penting tidak didukung pada [Komponen HTML](/id/basics/astro-components/#html-components).

## Halaman Error 404 Kustom

Untuk halaman error 404 kustom, buat file `404.astro` atau `404.md` di `src/pages`.

Ini akan dibangun menjadi halaman `404.html`. Sebagian besar [layanan deploy](/id/guides/deploy/) akan menemukan dan menggunakannya.

## Halaman Error 500 Kustom

Untuk halaman error 500 kustom yang ditampilkan pada halaman yang [dirender sesuai permintaan](/id/guides/on-demand-rendering/), buat file `src/pages/500.astro`. Halaman kustom ini tidak tersedia untuk halaman yang diprerender.

Jika terjadi error saat merender halaman ini, halaman error 500 bawaan host Anda akan ditampilkan ke pengunjung.

<p><Since v="4.10.3" /></p>

Selama pengembangan, jika Anda memiliki `500.astro`, error yang dilempar saat runtime akan dicatat di terminal, alih-alih ditampilkan pada overlay error.

### `error`

<p><Since v="4.11.0" /></p>

`src/pages/500.astro` adalah halaman khusus yang secara otomatis menerima prop `error` untuk error apa pun yang terjadi saat merender. Ini memungkinkan Anda menggunakan detail error (misalnya dari halaman, dari middleware, dll.) untuk menampilkan informasi kepada pengunjung.

Tipe data prop `error` dapat berupa apa saja, yang dapat memengaruhi cara Anda memberi tipe atau menggunakan nilainya di kode:

```astro title="src/pages/500.astro"
---
interface Props {
  error: unknown;
}

const { error } = Astro.props;
---
<div>{error instanceof Error ? error.message : "Unknown error"}</div>
```

Untuk menghindari kebocoran informasi sensitif saat menampilkan konten dari prop `error`, pertimbangkan untuk mengevaluasi error terlebih dahulu, dan mengembalikan konten yang sesuai berdasarkan error yang terjadi. Misalnya, Anda sebaiknya tidak menampilkan *stack* error karena berisi informasi tentang bagaimana kode Anda tersusun di server.

## Page Partials

<p><Since v="3.4.0" /></p>

:::caution
Page partials dimaksudkan untuk digunakan bersama library front-end seperti [htmx](https://htmx.org/) atau [Unpoly](https://unpoly.com/). Anda juga dapat menggunakannya jika nyaman menulis JavaScript front-end tingkat rendah. Karena itu, ini adalah fitur tingkat lanjut.

Selain itu, partial tidak boleh digunakan jika komponen berisi gaya atau skrip *scoped*, karena elemen-elemen tersebut akan dihapus dari output HTML. Jika Anda memerlukan gaya *scoped*, lebih baik gunakan halaman biasa (bukan partial) bersama library front-end yang tahu cara menggabungkan konten ke dalam *head*.
:::

Partial adalah komponen halaman yang berada di `src/pages/` dan tidak dimaksudkan untuk merender sebagai halaman penuh.

Seperti komponen di luar folder ini, file-file ini tidak otomatis menyertakan deklarasi `<!DOCTYPE html>` maupun konten `<head>` seperti gaya dan skrip *scoped*.

Namun, karena berada di direktori khusus `src/pages/`, HTML yang dihasilkan tersedia pada URL yang sesuai dengan path filenya. Ini memungkinkan library perenderan (mis. [htmx](https://htmx.org/), [Stimulus](https://stimulus.hotwired.dev/), [jQuery](https://jquery.com/)) mengaksesnya di klien dan memuat bagian HTML secara dinamis pada halaman tanpa *refresh* atau navigasi halaman.

Partial, jika digabungkan dengan library perenderan, menyediakan alternatif untuk [Astro islands](/id/concepts/islands/) dan [tag `<script>`](/id/guides/client-side-scripts/) untuk membangun konten dinamis di Astro.

File halaman yang dapat mengekspor nilai untuk [`partial`](/id/reference/routing-reference/#partial) (misalnya `.astro` dan `.mdx`, tetapi bukan `.md`) dapat ditandai sebagai partial.

```astro title="src/pages/partial.astro" ins={2}
---
export const partial = true;
---
<li>Saya adalah partial!</li>
```

### Penggunaan dengan library

Partial digunakan untuk memperbarui bagian halaman secara dinamis menggunakan library seperti [htmx](https://htmx.org/).

Contoh berikut menunjukkan atribut `hx-post` yang diarahkan ke URL sebuah partial. Konten dari halaman partial akan digunakan untuk memperbarui elemen HTML yang ditarget di halaman ini. 

```astro title="src/pages/index.astro" 'hx-post="/partials/clicked/"'
<html>
  <head>
    <title>My page</title>
    <script src="https://unpkg.com/htmx.org@1.9.6"
      integrity="sha384-FhXw7b6AlE/jyjlZH5iHa/tTe9EpJ1Y55RjcgPbjeWMskSxZt1v9qkxLJWNJaGni"
      crossorigin="anonymous"></script>
  </head>
  <body>
    <section>
      <div id="parent-div">Target here</div>
    
      <button hx-post="/partials/clicked/"
        hx-trigger="click"
        hx-target="#parent-div"
        hx-swap="innerHTML"
      >
        Click Me!
      </button>
    </section>
  </body>
</html>
```

Partial `.astro` tersebut harus ada pada path file yang sesuai, dan menyertakan *export* yang menandai halaman sebagai partial:

```astro title="src/pages/partials/clicked.astro" {2}
---
export const partial = true;
---
<div>Saya diklik!</div>
```

Lihat [dokumentasi htmx](https://htmx.org/docs/) untuk detail lebih lanjut penggunaan htmx.
