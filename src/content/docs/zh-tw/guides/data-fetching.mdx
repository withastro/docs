`.astro` 檔案可以抓取遠端資料，協助你產生頁面。

## Astro 中的 `fetch()`

所有 [Astro 元件](/zh-tw/basics/astro-components/)都可以在其元件腳本中使用[全域 `fetch()` 函式](https://developer.mozilla.org/en-US/docs/Web/API/fetch)，以完整 URL（例如 `https://example.com/api`）向 API 發送 HTTP 請求。
此外，你可以透過 [`new URL("/api", Astro.url)`](/zh-tw/reference/api-reference/#url) 組出在伺服器上隨需算繪的專案頁面與端點 URL。

這個 fetch 呼叫會在建置階段執行，資料會在元件模板中可用，以產生動態 HTML。若啟用 [SSR](/zh-tw/guides/on-demand-rendering/) 模式，fetch 呼叫則會在執行階段執行。

💡 在你的 Astro 元件腳本內善用 [**最上層 `await`**](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators/await#top_level_await)。

💡 將抓取到的資料當作 props，傳給 Astro 元件與框架元件。

```astro /await fetch\\(.*?\\);/
---
// src/components/User.astro
import Contact from "../components/Contact.jsx";
import Location from "../components/Location.astro";

const response = await fetch("https://randomuser.me/api/");
const data = await response.json();
const randomUser = data.results[0];
---
<!-- 建置階段抓取到的資料可以算繪到 HTML 中 -->
<h1>User</h1>
<h2>{randomUser.name.first} {randomUser.name.last}</h2>

<!-- 建置階段抓取到的資料可以當作 props 傳給元件 -->
<Contact client:load email={randomUser.email} />
<Location city={randomUser.location.city} />
```

:::note
請注意，Astro 元件中的所有資料都是在元件算繪時被抓取。

你部署後的 Astro 網站會**在建置階段抓取一次**資料。在開發模式下，你會在元件重新整理時看到資料被抓取。若你需要在客戶端多次重新抓取資料，請使用[框架元件](/zh-tw/guides/framework-components/)，或在 Astro 元件中使用[客戶端腳本](/zh-tw/guides/client-side-scripts/)。
:::

## 框架元件中的 `fetch()`

`fetch()` 函式在任何[框架元件](/zh-tw/guides/framework-components/)中也都能全域使用：

```tsx title="src/components/Movies.tsx" /await fetch\\(.*?\\)/
import type { FunctionalComponent } from 'preact';

const data = await fetch('https://example.com/movies.json').then((response) => response.json());

// 在建置階段算繪的元件也會輸出到 CLI。
// 以 `client:*` 指令算繪時，也會輸出到瀏覽器主控台。
console.log(data);

const Movies: FunctionalComponent = () => {
  // 把結果輸出到頁面
  return <div>{JSON.stringify(data)}</div>;
};

export default Movies;
```

## GraphQL 查詢

Astro 也可以使用 `fetch()` 以任何合法的 GraphQL 查詢向 GraphQL 伺服器發送請求。

```astro title="src/components/Film.astro" "await fetch"
---
const response = await fetch(
  "https://swapi-graphql.netlify.app/.netlify/functions/index",
  {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      query: `
        query getFilm ($id:ID!) {
          film(id: $id) {
            title
            releaseDate
          }
        }
      `,
      variables: {
        id: "ZmlsbXM6MQ==",
      },
    }),
  }
);


const json = await response.json();
const { film } = json.data;
---
<h1>Fetching information about Star Wars: A New Hope</h1>
<h2>Title: {film.title}</h2>
<p>Year: {film.releaseDate}</p>
```

## 從 Headless CMS 抓取

Astro 元件可以從你喜愛的 CMS 抓取資料，然後把它算繪為你的頁面內容。透過[動態路由](/zh-tw/guides/routing/#dynamic-routes)，元件甚至可以根據你的 CMS 內容產生頁面。

關於把 Astro 與 Storyblok、Contentful、WordPress 等 headless CMS 整合的完整細節，請見我們的 [CMS 指南](/zh-tw/guides/cms/)。
