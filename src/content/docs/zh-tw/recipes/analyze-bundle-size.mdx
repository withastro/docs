了解 Astro bundle 中包含哪些內容，對提升網站效能很重要。將 bundle 視覺化可以提供線索，讓你知道專案中可以做哪些調整來縮小 bundle 大小。

## 操作指南

[`rollup-plugin-visualizer` 函式庫](https://github.com/btd/rollup-plugin-visualizer)讓你能將 Rollup bundle 視覺化並進行分析，看看哪些模組佔了多少空間。
<Steps>
1. 安裝 `rollup-plugin-visualizer`：

    <PackageManagerTabs>
        <Fragment slot="npm">
        ```shell
        npm install rollup-plugin-visualizer --save-dev
        ```
        </Fragment>
        <Fragment slot="pnpm">
        ```shell
        pnpm add rollup-plugin-visualizer --save-dev
        ```
        </Fragment>
        <Fragment slot="yarn">
        ```shell
        yarn add rollup-plugin-visualizer --save-dev
        ```
        </Fragment>
    </PackageManagerTabs>

2. 將外掛加入 `astro.config.mjs` 檔案：

    ```js
    // @ts-check
    import { defineConfig } from 'astro/config';
    import { visualizer } from "rollup-plugin-visualizer";

    export default defineConfig({
    vite: {
        plugins: [visualizer({
            emitFile: true,
            filename: "stats.html",
        })]
    }
    });
    ```
3. 執行建置指令：

    <PackageManagerTabs>
        <Fragment slot="npm">
        ```shell
        npm run build
        ```
        </Fragment>
        <Fragment slot="pnpm">
        ```shell
        pnpm build
        ```
        </Fragment>
        <Fragment slot="yarn">
        ```shell
        yarn build
        ```
        </Fragment>
    </PackageManagerTabs>

4. 找出你專案的 `stats.html` 檔案。

		對完全靜態的網站來說，它會位於 `dist/` 目錄的根目錄，可以讓你看到 bundle 中包含了什麼。
		
		如果你的 Astro 專案使用隨需算繪，會有兩個 `stats.html` 檔案。一個給客戶端，另一個給伺服器端，兩者分別位於 `dist/client` 和 `dist/server/` 目錄的根目錄。
		
		如何解讀這些檔案、以及如何設定特定選項，請參閱 [Rollup Plugin Visualizer 的文件](https://github.com/btd/rollup-plugin-visualizer#how-to-use-generated-files)。
		
</Steps>

:::note
由於 Astro 採用獨特的水合方式，建置產物不一定能代表客戶端最終會收到的 bundle。

Rollup visualizer 會顯示整個網站使用的所有相依套件，但不會以單一頁面為單位拆解 bundle 大小。
:::
