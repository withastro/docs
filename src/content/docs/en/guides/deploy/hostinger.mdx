[Hostinger](https://www.hostinger.com/) is a web hosting provider that supports static sites and Node.js applications.

This guide covers deploying both static and server-rendered Astro projects to Hostinger using [hPanel](https://www.hostinger.com/tutorials/what-is-hpanel).

## Prerequisites

- A [Hostinger account](https://www.hostinger.com/) with an active hosting plan.
  - Static site deployment is supported on all plans that include hPanel.
  - Server-rendered deployment requires a plan that supports Node.js applications, such as Business Web Hosting or Cloud hosting.
- An Astro project ready to deploy.

## Static site deployment

Astro projects use `output: 'static'` by default, so no additional configuration is required to deploy a static site.

You can upload the contents of your `dist/` folder to Hostinger using either the hPanel **File Manager** or an FTP client.

### File Manager upload

<Steps>
1. Build your Astro project locally:

   <PackageManagerTabs>
     <Fragment slot="npm">
     ```shell
     npm run build
     ```
     </Fragment>
     <Fragment slot="pnpm">
     ```shell
     pnpm build
     ```
     </Fragment>
     <Fragment slot="yarn">
     ```shell
     yarn build
     ```
     </Fragment>
   </PackageManagerTabs>

2. Log in to [hPanel](https://hpanel.hostinger.com/) and open your website.

3. Open **Files** > **File Manager** and navigate to the `public_html` directory.

4. Upload the contents of your local `dist/` folder into `public_html`. You can drag and drop files, or upload an archive and extract it in place.

5. Visit your domain to confirm your site is live.
</Steps>

### FTP upload

<Steps>
1. In hPanel, go to **Files** > **FTP Accounts** to find or create your FTP credentials.

2. Connect to your hosting account using an FTP client such as [FileZilla](https://filezilla-project.org/).

3. Upload the contents of your local `dist/` folder to the `public_html` directory on the server.

4. Visit your domain to confirm your site is live.
</Steps>

## Server-rendered deployment using Node.js

For [on-demand rendering](/en/guides/on-demand-rendering/), deploy your Astro project as a Node.js application on a Hostinger plan that supports Node.js.

### Add the Node.js adapter

Add the [Node.js adapter](/en/guides/integrations-guide/node/) to enable on-demand rendering with the following `astro add` command:

<PackageManagerTabs>
  <Fragment slot="npm">
  ```shell
  npx astro add node
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```shell
  pnpm astro add node
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```shell
  yarn astro add node
  ```
  </Fragment>
</PackageManagerTabs>

<ReadMore>See the [Node.js adapter guide](/en/guides/integrations-guide/node/) for additional configuration options.</ReadMore>

Make sure your `package.json` has a `start` script that runs the built server:

```json title="package.json"
"scripts": {
  "start": "node ./dist/server/entry.mjs"
}
```

### Deploy from a Git repository

<Steps>
1. Push your Astro project to a GitHub, GitLab, or Bitbucket repository.

2. Log in to [hPanel](https://hpanel.hostinger.com/) and go to **Websites**.

3. Add a new website and select the **Node.js** application option.

4. Connect your Git provider and select your repository and branch.

5. Configure the build settings:

   - **Build command:** `npm run build`
   - **Start command:** `npm run start`

6. Select a Node.js version that satisfies Astro's [minimum requirement](/en/install-and-setup/#prerequisites) (Node.js 22 or later).

7. Start the deployment. Hostinger will install dependencies, run your build command, and start the application.
</Steps>

### Deploy by uploading project files

<Steps>
1. Build your Astro project locally so you can upload the generated `dist/` folder along with `package.json` and `package-lock.json`.

2. In hPanel, add a new website and select the **Node.js** application option.

3. Upload your project files (or a `.zip` archive) using the File Manager.

4. Configure the application:

   - **Application root:** the folder containing your `package.json`.
   - **Start command:** `npm run start`

5. Select a supported Node.js version and start the application.
</Steps>
