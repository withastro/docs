## 为什么要使用 DAM 或托管媒体？

使用 DAM 或者说数字资源管理器，可以帮助个人、团队和组织集中管理其图像和视频资源，就像 [CMS](/zh-cn/guides/cms/) 一样。

区别在于所管理的内容类型：DAM 主要管理图像、视频、其他媒体资源比如 3D 模型，以及与这些资源相关的任何元数据。

当你的资源在多个网页或移动端页面之间使用单一数据来源时，这尤其有用。如果你所在组织的多个团队需要使用相同的资源，或者正在通过集成到其他内容系统，例如 PIM（产品信息管理工具）以实现资源到产品的连接，这些环节都非常重要。

## 哪些托管媒体系统或 DAM 可与 Astro 配合使用？

就像使用 CMS 时一样，当 Astro 在处理内容的 _渲染_ 时，你将需要使用无头 DAM，它允许你通过 API 或 SDK 来获取资源并与之交互。

一些无头 DAM，例如 Cloudinary，提供了 Astro [集成](/zh-cn/guides/integrations-guide/) 使你可以轻松获取资源并将其显示在你的网站或应用程序上。

## 我可以使用 Astro 而不用托管媒体系统或者 DAM 吗？

当然！Astro 提供了内置的 [图像存储](/zh-cn/guides/images/#在哪里存储图像) 方式，包括对引用远程图像的支持。
