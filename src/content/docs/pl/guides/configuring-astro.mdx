Astro to elastyczny framework bez narzuconych rozwiązań, który pozwala na konfigurację projektu na wiele różnych sposobów. Oznacza to, że rozpoczęcie pracy z nowym projektem może wydawać się przytłaczające: nie ma "jednego najlepszego sposobu" na skonfigurowanie projektu Astro!

Przewodniki w tej sekcji "Konfiguracja" pomogą Ci zapoznać się z różnymi plikami, które pozwalają na konfigurację i dostosowanie aspektów Twojego projektu oraz środowiska programistycznego.

Jeśli to Twój pierwszy projekt Astro lub minęło już trochę czasu, odkąd ostatnio konfigurowałeś nowy projekt, skorzystaj z poniższych przewodników i dokumentacji referencyjnej.

## Plik konfiguracyjny Astro

[Plik konfiguracyjny Astro](/pl/reference/configuration-reference/) to plik JavaScript znajdujący się w katalogu głównym każdego projektu startowego:

```js
// astro.config.mjs
import { defineConfig } from "astro/config";

export default defineConfig({
  // tutaj umieść swoje opcje konfiguracji...
});
```

Jest wymagany tylko wtedy, gdy masz coś do skonfigurowania, ale większość projektów będzie używać tego pliku. Pomocnik `defineConfig()` zapewnia automatyczne IntelliSense w Twoim IDE i to tutaj dodasz wszystkie opcje konfiguracji, aby powiedzieć Astro, jak budować i renderować Twój projekt do HTML.

Zalecamy używanie domyślnego formatu pliku `.mjs` w większości przypadków lub `.ts`, jeśli chcesz pisać TypeScript w pliku konfiguracyjnym. Jednak `astro.config.js` i `astro.config.cjs` są również obsługiwane.

<ReadMore>Przeczytaj [dokumentację referencyjną konfiguracji](/pl/reference/configuration-reference/) Astro, aby uzyskać pełny przegląd wszystkich obsługiwanych opcji konfiguracji.</ReadMore>

## Plik konfiguracyjny TypeScript

Każdy projekt startowy Astro zawiera plik `tsconfig.json` w Twoim projekcie. [Skrypt komponentu](/pl/basics/astro-components/#the-component-script) Astro to TypeScript, który zapewnia narzędzia edytora Astro i pozwala opcjonalnie dodać składnię do Twojego JavaScript dla sprawdzania typów własnego kodu projektu.

Użyj pliku `tsconfig.json`, aby skonfigurować szablon TypeScript, który będzie wykonywał sprawdzanie typów w Twoim kodzie, skonfigurować wtyczki TypeScript, ustawić aliasy importu i więcej.

<ReadMore>Przeczytaj [przewodnik TypeScript](/pl/guides/typescript/) Astro, aby uzyskać pełny przegląd opcji TypeScript i wbudowanych typów użytkowych Astro.</ReadMore>

## Środowisko programistyczne

Podczas pracy w trybie deweloperskim możesz wykorzystać swój edytor kodu i inne narzędzia, aby ulepszyć doświadczenie programisty Astro.

Astro zapewnia własne oficjalne rozszerzenie VS Code i jest kompatybilne z kilkoma innymi popularnymi narzędziami edytora. Astro zapewnia również dostosowywalny pasek narzędzi, który wyświetla się w podglądzie przeglądarki podczas działania serwera deweloperskiego. Możesz instalować, a nawet budować własne aplikacje paska narzędzi dla dodatkowej funkcjonalności.

<ReadMore>Przeczytaj przewodniki Astro dotyczące [opcji konfiguracji edytora](/pl/editor-setup/) i [używania paska narzędzi deweloperskich](/pl/guides/dev-toolbar/), aby dowiedzieć się, jak dostosować swoje środowisko programistyczne.</ReadMore>

## Typowe zadania nowego projektu

Oto kilka pierwszych kroków, które możesz wybrać przy nowym projekcie Astro.

### Dodaj domenę wdrożenia

Aby wygenerować mapę witryny i utworzyć kanoniczne URL-e, skonfiguruj URL wdrożenia w opcji [`site`](/pl/reference/configuration-reference/#site). Jeśli wdrażasz pod określoną ścieżką (np. `www.example/docs`), możesz również skonfigurować [`base`](/pl/reference/configuration-reference/#base) dla korzenia swojego projektu.

Dodatkowo, różne hosty wdrożeniowe mogą mieć różne zachowania dotyczące końcowych ukośników na końcu Twoich URL-i (np. `example.com/about` vs `example.com/about/`). Po wdrożeniu witryny może być konieczne skonfigurowanie preferencji [`trailingSlash`](/o´pl/reference/configuration-reference/#trailingslash).

```js title="astro.config.mjs"
import { defineConfig } from "astro/config";

export default defineConfig({
  site: "https://www.example.com",
  base: "/docs",
  trailingSlash: "always",
});
```

### Dodaj metadane witryny

Astro nie używa swojego pliku konfiguracyjnego dla typowych danych SEO lub meta, tylko dla informacji wymaganych do budowania kodu projektu i renderowania go do HTML.

Zamiast tego, te informacje są dodawane do sekcji `<head>` Twojej strony w tagach HTML `<link>` i `<meta>`, tak jakbyś pisał zwykłe strony HTML.

Typowym wzorcem dla witryn Astro jest utworzenie [komponentu `.astro`](/pl/basics/astro-components/) `<Head />`, który można dodać do wspólnego [komponentu układu](/pl/basics/layouts/), aby mógł być zastosowany do wszystkich Twoich stron.

```astro title="src/components/MainLayout.astro"
---
import Head from "./Head.astro";

const { ...props } = Astro.props;
---
<html>
  <head>
    <meta charset="utf-8">
    <Head />
    <!-- Dodatkowe elementy head -->
  </head>
  <body>
    <!-- Treść strony tutaj -->
  </body>
</html>
```

Ponieważ `Head.astro` to zwykły komponent Astro, możesz importować pliki i odbierać właściwości przekazane z innych komponentów, takie jak konkretny tytuł strony.

```astro title="src/components/Head.astro"
---
import Favicon from "../assets/Favicon.astro";
import SomeOtherTags from "./SomeOtherTags.astro";

const { title = "Moja witryna Astro", ...props } = Astro.props;
---
<link rel="sitemap" href="/sitemap-index.xml">
<title>{title}</title>
<meta name="description" content="Witaj na mojej nowej witrynie Astro!">

<!-- Analityka internetowa -->
<script data-goatcounter="https://my-account.goatcounter.com/count" async src="//gc.zgo.at/count.js"></script>

<!-- Tagi Open Graph -->
<meta property="og:title" content="Moja nowa witryna Astro" />
<meta property="og:type" content="website" />
<meta property="og:url" content="http://www.example.com/" />
<meta property="og:description" content="Witaj na mojej nowej witrynie Astro!" />
<meta property="og:image" content="https://www.example.com/_astro/seo-banner.BZD7kegZ.webp">
<meta property="og:image:alt" content="">

<SomeOtherTags />

<Favicon />
```
