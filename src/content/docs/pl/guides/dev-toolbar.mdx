Podczas działania serwera deweloperskiego Astro dodaje pasek narzędzi na dole każdej strony w lokalnym podglądzie przeglądarki.

Pasek ten zawiera szereg przydatnych narzędzi do debugowania i inspekcji witryny podczas rozwoju, a także może być [rozszerzany o dodatkowe aplikacje](#rozszerzanie-paska-narzędzi) znajdujące się w katalogu integracji. Możesz nawet [stworzyć własne aplikacje](/pl/recipes/making-toolbar-apps/) z wykorzystaniem [API Dev Toolbar](/pl/reference/dev-toolbar-app-reference/)!

Pasek narzędzi jest domyślnie włączony i pojawia się po najechaniu kursorem na dolną krawędź strony. Jest to wyłącznie narzędzie deweloperskie i nie pojawi się na opublikowanej stronie.

## Wbudowane aplikacje

### Menu Astro

Aplikacja Menu Astro zapewnia szybki dostęp do różnych informacji o bieżącym projekcie oraz do dodatkowych zasobów. Umożliwia między innymi jedno-klikowy dostęp do dokumentacji Astro, repozytorium GitHub oraz serwera Discord.

Aplikacja zawiera również przycisk „Kopiuj informacje debugowania”, który uruchamia polecenie [`astro info`](/pl/reference/cli-reference/#astro-info) i kopiuje wynik do schowka. Może to być pomocne podczas zgłaszania problemów lub proszenia o pomoc.

### Inspektor

Aplikacja Inspektor dostarcza informacji o wszystkich [wyspach](/pl/concepts/islands/) znajdujących się na bieżącej stronie. Pokazuje przekazane do nich właściwości oraz dyrektywę klienta używaną do ich renderowania.

### Audyt

Aplikacja Audyt automatycznie uruchamia zestaw testów sprawdzających najczęstsze problemy z wydajnością i dostępnością strony. Jeśli wykryty zostanie jakiś problem, na pasku narzędzi pojawi się czerwona kropka. Kliknięcie aplikacji pokaże listę wyników i wyróżni powiązane elementy na stronie.

:::note
Podstawowe testy wydajności i dostępności wykonywane przez pasek narzędzi nie zastępują dedykowanych narzędzi takich jak [Pa11y](https://pa11y.org/) czy [Lighthouse](https://developers.google.com/web/tools/lighthouse), a tym bardziej oceny przez człowieka.

Celem paska narzędzi deweloperskich jest szybkie wychwytywanie typowych problemów podczas pracy nad projektem, bez konieczności przełączania się do innych narzędzi.
:::

### Ustawienia

Aplikacja Ustawienia pozwala konfigurować opcje paska narzędzi, takie jak logowanie w trybie verbose, wyłączanie powiadomień czy zmiana jego położenia na ekranie.

## Rozszerzanie paska narzędzi

Integracje Astro mogą dodawać własne aplikacje do paska narzędzi, umożliwiając rozbudowę o narzędzia dopasowane do konkretnego projektu. Możesz [znaleźć więcej aplikacji w katalogu integracji](https://astro.build/integrations/?search=&categories%5B%5D=toolbar) lub użyć [Menu Astro](#menu-astro), aby je przeglądać.

Zainstaluj dodatkowe aplikacje do paska narzędzi jak każdą inną [integrację Astro](/pl/guides/integrations-guide/), postępując zgodnie z jej instrukcjami instalacji.

## Wyłączanie paska narzędzi

Pasek narzędzi deweloperskich jest domyślnie włączony dla każdej witryny. Można go jednak wyłączyć dla poszczególnych projektów lub użytkowników.

### Dla całego projektu

Aby wyłączyć pasek narzędzi dla wszystkich pracujących nad projektem, ustaw `devToolbar: false` w [pliku konfiguracyjnym Astro](/pl/reference/configuration-reference/#devtoolbarenabled).

```js title="astro.config.mjs" ins={4-6}
import { defineConfig } from "astro/config";

export default defineConfig({
  devToolbar: {
    enabled: false
  }
});
```

Aby ponownie włączyć pasek narzędzi, usuń te linie z konfiguracji lub ustaw `enabled: true`.

### Dla pojedynczego użytkownika

Aby wyłączyć pasek narzędzi tylko dla siebie w danym projekcie, uruchom polecenie [`astro preferences`](/pl/reference/cli-reference/#astro-preferences):

```shell
astro preferences disable devToolbar
```

Aby wyłączyć pasek narzędzi we wszystkich projektach Astro dla użytkownika na danej maszynie, dodaj flagę `--global`:

```shell
astro preferences disable --global devToolbar
```

Możesz później ponownie włączyć pasek narzędzi komendą:

```shell
astro preferences enable devToolbar
```
