[Docusaurus](https://Docusaurus.io) là một công cụ xây dựng website tài liệu phổ biến được xây dựng trên React.

## Điểm tương đồng chính giữa Docusaurus và Astro

Docusaurus và Astro có một số điểm tương đồng sẽ giúp bạn di chuyển dự án:

- Cả Astro và Docusaurus đều là các công cụ xây dựng site hiện đại, dựa trên JavaScript (Jamstack) được thiết kế cho [các website hướng nội dung](/vi/concepts/why-astro/#content-driven), như các site tài liệu.

- Cả Astro và Docusaurus đều hỗ trợ [trang MDX](/vi/guides/markdown-content/). Bạn có thể sử dụng các file `.mdx` hiện có cho Astro.

- Cả Astro và Docusaurus đều sử dụng [routing dựa trên file](/vi/guides/routing/) để tự động tạo các route trang cho bất kỳ file MDX nào nằm trong `src/pages`. Việc sử dụng cấu trúc file của Astro cho nội dung hiện có và khi thêm trang mới sẽ cảm thấy quen thuộc.

- Astro có [tích hợp chính thức để sử dụng các component React](/vi/guides/integrations-guide/react/). Lưu ý rằng trong Astro, các file React **phải** có phần mở rộng `.jsx` hoặc `.tsx`.

- Astro hỗ trợ [cài đặt các package NPM](/vi/guides/imports/#npm-packages), bao gồm một số package cho React. Bạn có thể giữ một số hoặc tất cả các component React và dependencies hiện có.

- [Cú pháp giống JSX của Astro](/vi/basics/astro-components/#the-component-template) sẽ cảm thấy quen thuộc nếu bạn đã quen với việc viết React.

## Điểm khác biệt chính giữa Docusaurus và Astro

Khi bạn xây dựng lại site Docusaurus trong Astro, bạn sẽ nhận thấy một số điểm khác biệt quan trọng:

- Docusaurus là một ứng dụng một trang (SPA) dựa trên React. Các site Astro là ứng dụng nhiều trang được xây dựng sử dụng [`.astro` components](/vi/basics/astro-components/), nhưng cũng có thể hỗ trợ [React, Preact, Vue.js, Svelte, SolidJS, AlpineJS](/vi/guides/framework-components/) và templating HTML thuần.

- Docusaurus được thiết kế để xây dựng các website tài liệu và có một số tính năng website tích hợp sẵn dành riêng cho tài liệu mà bạn sẽ phải tự xây dựng trong Astro. Thay vào đó, Astro cung cấp một số tính năng này thông qua [Starlight: một theme tài liệu chính thức](https://starlight.astro.build). Website này là nguồn cảm hứng cho Starlight, và hiện đang chạy trên nó! Bạn cũng có thể tìm thêm [các theme tài liệu cộng đồng](https://astro.build/themes?search=&categories%5B%5D=docs) với các tính năng tích hợp sẵn trong Themes Showcase của chúng tôi.

- Các site Docusaurus sử dụng trang MDX cho nội dung. Theme tài liệu của Astro sử dụng file Markdown (`.md`) theo mặc định và không yêu cầu bạn sử dụng MDX. Bạn có thể tùy chọn [cài đặt tích hợp MDX của Astro](/vi/guides/integrations-guide/mdx/) (đã bao gồm trong theme Starlight của chúng tôi theo mặc định) và sử dụng file `.mdx` ngoài các file Markdown tiêu chuẩn.

## Chuyển từ Docusaurus sang Astro

Để chuyển đổi một site tài liệu Docusaurus sang Astro, hãy bắt đầu với [template starter theme tài liệu Starlight chính thức](https://starlight.astro.build) của chúng tôi, hoặc khám phá thêm các theme tài liệu cộng đồng trong [theme showcase](https://astro.build/themes?search=&categories%5B%5D=docs) của chúng tôi.

Bạn có thể truyền argument `--template` cho lệnh `create astro` để bắt đầu một dự án Astro mới với một trong các starter chính thức của chúng tôi. Hoặc, bạn có thể [bắt đầu một dự án mới từ bất kỳ repository Astro hiện có nào trên GitHub](/vi/install-and-setup/#install-from-the-cli-wizard).

  <PackageManagerTabs>
    <Fragment slot="npm">
    ```shell
    npm create astro@latest -- --template starlight
    ```
    </Fragment>
    <Fragment slot="pnpm">
    ```shell
    pnpm create astro@latest --template starlight
    ```
    </Fragment>
    <Fragment slot="yarn">
    ```shell
    yarn create astro --template starlight
    ```
    </Fragment>
  </PackageManagerTabs>

Tích hợp MDX của Astro được bao gồm theo mặc định, vì vậy bạn có thể [mang các file nội dung hiện có đến Starlight](https://starlight.astro.build/getting-started/#add-content) ngay lập tức.

Bạn có thể tìm thấy starter tài liệu của Astro, và các template chính thức khác, trên [astro.new](https://astro.new). Bạn sẽ tìm thấy link đến repository GitHub của mỗi dự án, cũng như các link một cú nhấp để mở một dự án hoạt động trong các môi trường phát triển trực tuyến IDX, StackBlitz, CodeSandbox và Gitpod.
