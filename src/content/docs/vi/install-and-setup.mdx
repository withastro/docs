Lệnh CLI [`create astro`](#install-from-the-cli-wizard) là cách nhanh nhất để bắt đầu dự án Astro mới từ đầu. Nó sẽ hướng dẫn bạn từng bước thiết lập dự án Astro mới và cho phép bạn chọn từ một số mẫu khởi động chính thức khác nhau.

Bạn cũng có thể chạy lệnh CLI với flag `template` để bắt đầu dự án bằng bất kỳ theme hoặc mẫu khởi động nào hiện có. Khám phá [bộ sưu tập themes và starters](https://astro.build/themes/) nơi bạn có thể duyệt themes cho blog, portfolio, trang tài liệu, landing page và nhiều hơn nữa!

Để cài đặt Astro thủ công, xem [hướng dẫn cài đặt thủ công từng bước](#manual-setup).

:::tip[Xem trước trực tuyến]
Bạn muốn thử Astro trên trình duyệt? Truy cập [astro.new](https://astro.new/) để duyệt các mẫu khởi động và tạo dự án Astro mới mà không cần rời khỏi trình duyệt.
:::

## Yêu cầu hệ thống

- **Node.js** - `v18.20.8` hoặc `v20.3.0`, `v22.0.0` trở lên. (`v19` và `v21` không được hỗ trợ.)
- **Trình soạn thảo văn bản** - Chúng tôi khuyên dùng [VS Code](https://code.visualstudio.com/) với [Extension Astro chính thức](https://marketplace.visualstudio.com/items?itemName=astro-build.astro-vscode).
- **Terminal** - Astro được truy cập thông qua giao diện dòng lệnh (CLI).

## Khả năng tương thích trình duyệt

Astro được xây dựng với Vite, mặc định hỗ trợ các trình duyệt có hỗ trợ JavaScript hiện đại. Để tham khảo đầy đủ, bạn có thể xem [danh sách các phiên bản trình duyệt hiện được hỗ trợ trong Vite](https://vite.dev/guide/build.html#browser-compatibility).

## Cài đặt từ CLI wizard

Bạn có thể chạy `create astro` ở bất kỳ đâu trên máy, vì vậy không cần tạo thư mục trống mới cho dự án trước khi bắt đầu. Nếu bạn chưa có thư mục trống cho dự án mới, wizard sẽ giúp tạo một thư mục tự động.

<Steps>
1. Chạy lệnh sau trong terminal để bắt đầu wizard cài đặt:

    <PackageManagerTabs>
      <Fragment slot="npm">
      ```shell
      npm create astro@latest
      ```
      </Fragment>
      <Fragment slot="pnpm">
      ```shell
      pnpm create astro@latest
      ```
      </Fragment>
      <Fragment slot="yarn">
      ```shell
      yarn create astro
      ```
      </Fragment>
    </PackageManagerTabs>

    Nếu mọi thứ diễn ra tốt đẹp, bạn sẽ thấy thông báo thành công cùng với một số bước tiếp theo được khuyến nghị.
    
2. Bây giờ dự án đã được tạo, bạn có thể `cd` vào thư mục dự án mới để bắt đầu sử dụng Astro.

3. Nếu bạn bỏ qua bước "Install dependencies?" trong CLI wizard, hãy chắc chắn cài đặt dependencies trước khi tiếp tục.

    <PackageManagerTabs>
      <Fragment slot="npm">
      ```shell
      npm install
      ```
      </Fragment>
      <Fragment slot="pnpm">
      ```shell
      pnpm install
      ```
      </Fragment>
      <Fragment slot="yarn">
      ```shell
      yarn install
      ```
      </Fragment>
    </PackageManagerTabs>

4. Bây giờ bạn có thể [khởi động Astro dev server](/vi/develop-and-build/#start-the-astro-dev-server) và xem bản xem trước trực tiếp của dự án trong khi phát triển!
</Steps>

## CLI installation flags

Bạn có thể chạy lệnh `create astro` với các flag bổ sung để tùy chỉnh quá trình thiết lập (ví dụ: trả lời "yes" cho tất cả câu hỏi, bỏ qua animation Houston) hoặc dự án mới của bạn (ví dụ: cài đặt git hay không, thêm integrations).

<ReadMore>Xem [tất cả các flag lệnh `create astro` có sẵn](https://github.com/withastro/astro/blob/main/packages/create-astro/README.md)</ReadMore>

### Thêm integrations

Bạn có thể bắt đầu dự án Astro mới và cài đặt bất kỳ [integration chính thức](/vi/guides/integrations-guide/) hoặc community integrations hỗ trợ lệnh `astro add` đồng thời bằng cách truyền tham số `--add` cho lệnh `create astro`.

Chạy lệnh sau trong terminal, thay thế bất kỳ integration nào hỗ trợ lệnh `astro add`:

<PackageManagerTabs>
  <Fragment slot="npm">
  ```shell
	# Tạo dự án mới với React và Partytown
  npm create astro@latest -- --add react --add partytown
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```shell
	# Tạo dự án mới với React và Partytown
  pnpm create astro@latest --add react --add partytown
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```shell
	# Tạo dự án mới với React và Partytown
  yarn create astro --add react --add partytown
  ```
  </Fragment>
</PackageManagerTabs>

### Sử dụng theme hoặc starter template

Bạn có thể bắt đầu dự án Astro mới dựa trên [ví dụ chính thức](https://github.com/withastro/astro/tree/main/examples) hoặc nhánh `main` của bất kỳ repository GitHub nào bằng cách truyền tham số `--template` cho lệnh `create astro`.

Chạy lệnh sau trong terminal, thay thế tên starter template Astro chính thức, hoặc username GitHub và repository của theme bạn muốn sử dụng:

<PackageManagerTabs>
  <Fragment slot="npm">
  ```shell
  # tạo dự án mới với ví dụ chính thức
  npm create astro@latest -- --template <example-name>
  # tạo một dự án mới với kho lưu trữ Gihub nhánh main
  npm create astro@latest -- --template <github-username>/<github-repo>
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```shell
  # tạo dự án mới với ví dụ chính thức
  pnpm create astro@latest --template <example-name>
  
  # tạo một dự án mới với kho lưu trữ Gihub nhánh main
  pnpm create astro@latest --template <github-username>/<github-repo>
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```shell
  # tạo dự án mới với ví dụ chính thức
  yarn create astro --template <example-name>
  
  # tạo một dự án mới với kho lưu trữ Gihub nhánh main
  yarn create astro --template <github-username>/<github-repo>
  ```
  </Fragment>
</PackageManagerTabs>

Mặc định, lệnh này sẽ sử dụng nhánh `main` của template repository. Để sử dụng tên nhánh khác, truyền nó như một phần của tham số `--template`: `<github-username>/<github-repo>#<branch>`.

## Thiết lập thủ công

Hướng dẫn này sẽ đưa bạn qua các bước cài đặt và cấu hình thủ công dự án Astro mới.

Nếu bạn không muốn sử dụng công cụ CLI `create astro` tự động, bạn có thể thiết lập dự án bằng cách làm theo hướng dẫn dưới đây.

<Steps>
1. Tạo thư mục của bạn

    Tạo thư mục trống với tên dự án, sau đó điều hướng vào đó.

    ```bash
    mkdir my-astro-project
    cd my-astro-project
    ```

    Khi đã vào thư mục mới, tạo file `package.json` cho dự án. Đây là cách bạn quản lý dependencies dự án, bao gồm Astro. Nếu bạn không quen thuộc với định dạng file này, chạy lệnh sau để tạo một file.

    <PackageManagerTabs>
      <Fragment slot="npm">
      ```shell
      npm init --yes
      ```
      </Fragment>
      <Fragment slot="pnpm">
      ```shell
      pnpm init 
      ```
      </Fragment>
      <Fragment slot="yarn">
      ```shell
      yarn init --yes
      ```
      </Fragment>
    </PackageManagerTabs>

2. Cài đặt Astro

    Đầu tiên, cài đặt dependencies dự án Astro bên trong dự án của bạn.

    :::note[Quan trọng]
    Astro phải được cài đặt locally, không phải globally. Đảm bảo bạn *không* chạy `npm install -g astro` `pnpm add -g astro` hoặc `yarn add global astro`.
    :::

    <PackageManagerTabs>
      <Fragment slot="npm">
      ```shell
      npm install astro
      ```
      </Fragment>
      <Fragment slot="pnpm">
      ```shell
      pnpm add astro
      ```
      </Fragment>
      <Fragment slot="yarn">
      ```shell
      yarn add astro
      ```
      </Fragment>
    </PackageManagerTabs>

    Sau đó, thay thế phần "scripts" placeholder trong `package.json` bằng như sau:

    ```json title="package.json" del={3} ins={4-6}
    {
      "scripts": {
        "test": "echo \"Error: no test specified\" && exit 1",
        "dev": "astro dev",
        "build": "astro build",
        "preview": "astro preview"
      },
    }
    ```

    Bạn sẽ sử dụng các scripts này sau trong hướng dẫn để khởi động Astro và chạy các lệnh khác nhau.

3. Tạo trang đầu tiên

    Trong trình soạn thảo văn bản, tạo file mới trong thư mục tại `src/pages/index.astro`. Đây sẽ là trang Astro đầu tiên trong dự án.

    Cho hướng dẫn này, copy và paste đoạn code sau (bao gồm dấu gạch `---`) vào file mới:

    ```astro title="src/pages/index.astro"
    ---
    console.log('This runs in your terminal, not the browser!');
    ---
    <!-- Dưới đây là "component template" của bạn. Nó chỉ là HTML thôi nhưng được thêm vào 1 chút phép thuật để giúp bạn có thể xây dựng được một template tuyệt với -->
    <html>
      <body>
        <h1>Hello, World!</h1>
      </body>
    </html>
    <style>
      h1 {
        color: orange;
      }
    </style>
    ```

4. Tạo static asset đầu tiên

    Bạn cũng muốn tạo thư mục `public/` để lưu trữ static assets. Astro sẽ luôn bao gồm những assets này trong bản build cuối cùng, vì vậy bạn có thể tham chiếu chúng một cách an toàn từ bên trong component templates.

    Trong trình soạn thảo văn bản, tạo file mới trong thư mục tại `public/robots.txt`. `robots.txt` là file đơn giản mà hầu hết các trang web sẽ bao gồm để nói với search bots như Google cách đối xử với trang web của bạn.

    Cho hướng dẫn này, copy và paste đoạn code sau vào file mới:

    ```diff title="public/robots.txt"
    # Example: Allow all bots to scan and index your site.
    # Full syntax: https://developers.google.com/search/docs/advanced/robots/create-robots-txt		
    User-agent: *
    Allow: /
    ```

5. Tạo `astro.config.mjs`

    Astro được cấu hình bằng `astro.config.mjs`. File này là tùy chọn nếu bạn không cần cấu hình Astro, nhưng bạn có thể muốn tạo nó ngay bây giờ.

    Tạo `astro.config.mjs` ở root của dự án, và copy code dưới đây vào đó:

    ```js title="astro.config.mjs"
    import { defineConfig } from "astro/config";

    // https://astro.build/config
    export default defineConfig({});
    ```

    Nếu bạn muốn bao gồm [UI framework components](/vi/guides/framework-components/) như React, Svelte, v.v. hoặc sử dụng các công cụ khác như MDX hoặc Partytown trong dự án, đây là nơi bạn sẽ [import và cấu hình integrations thủ công](/vi/guides/integrations-guide/).

    Đọc [tham chiếu cấu hình API](/vi/reference/configuration-reference/) của Astro để biết thêm thông tin.

6. Thêm hỗ trợ TypeScript

    TypeScript được cấu hình bằng `tsconfig.json`. Ngay cả khi bạn không viết code TypeScript, file này quan trọng để các công cụ như Astro và VS Code biết cách hiểu dự án của bạn. Một số tính năng (như npm package imports) không được hỗ trợ đầy đủ trong editor mà không có file `tsconfig.json`.

    Nếu bạn có ý định viết code TypeScript, khuyến nghị sử dụng template `strict` hoặc `strictest` của Astro. Bạn có thể xem và so sánh ba cấu hình template tại [astro/tsconfigs/](https://github.com/withastro/astro/blob/main/packages/astro/tsconfigs/).

    Tạo `tsconfig.json` ở root của dự án, và copy code dưới đây vào đó. (Bạn có thể sử dụng `base`, `strict`, hoặc `strictest` cho TypeScript template):

    ```json title="tsconfig.json" "base"
    {
      "extends": "astro/tsconfigs/base"
    }
    ```

    Đọc [hướng dẫn thiết lập TypeScript](/vi/guides/typescript/#setup) của Astro để biết thêm thông tin.

7. Các bước tiếp theo

    Nếu bạn đã làm theo các bước trên, thư mục dự án bây giờ sẽ trông như thế này:

    <FileTree>
    - node_modules/
    - public/
      - robots.txt
    - src/
      - pages/
        - index.astro
    - astro.config.mjs
    - package-lock.json hoặc `yarn.lock`, `pnpm-lock.yaml`, v.v.
    - package.json
    - tsconfig.json
    </FileTree>

8. Bây giờ bạn có thể [khởi động Astro dev server](/vi/develop-and-build/#start-the-astro-dev-server) và xem bản xem trước trực tiếp của dự án trong khi phát triển!

</Steps>
