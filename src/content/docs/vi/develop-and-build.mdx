Khi bạn đã có một dự án Astro, giờ đây bạn đã sẵn sàng để xây dựng với Astro! 🚀

## Chỉnh sửa dự án của bạn

Để thay đổi dự án của bạn, hãy mở thư mục dự án trong trình soạn thảo code. Làm việc trong chế độ phát triển với máy chủ dev đang chạy cho phép bạn xem các cập nhật trên trang web khi bạn chỉnh sửa code.

Bạn cũng có thể [tùy chỉnh các khía cạnh của môi trường phát triển](#configure-your-dev-environment) như cấu hình TypeScript hoặc cài đặt các extension chính thức của Astro.

### Khởi động máy chủ dev Astro

Astro đi kèm với máy chủ phát triển tích hợp có mọi thứ bạn cần cho việc phát triển dự án. Lệnh CLI `astro dev` sẽ khởi động máy chủ phát triển cục bộ để bạn có thể xem trang web mới của mình hoạt động lần đầu tiên.

Mỗi template starter đều có script được cấu hình sẵn sẽ chạy `astro dev` cho bạn. Sau khi điều hướng vào thư mục dự án, sử dụng trình quản lý package yêu thích của bạn để chạy lệnh này và khởi động máy chủ phát triển Astro.

<Tabs>
  <TabItem label="npm">
  ```shell
  npm run dev
  ```
  </TabItem>
  <TabItem label="pnpm">
  ```shell
  pnpm run dev
  ```
  </TabItem>
  <TabItem label="yarn">
  ```shell
  yarn run dev
  ```
  </TabItem>
</Tabs>

Nếu mọi thứ diễn ra tốt đẹp, Astro sẽ phục vụ dự án của bạn tại [http://localhost:4321/](http://localhost:4321/). Truy cập liên kết đó trong trình duyệt và xem trang web mới của bạn!

### Làm việc trong chế độ phát triển

Astro sẽ lắng nghe các thay đổi file trực tiếp trong thư mục `src/` và cập nhật bản xem trước trang web khi bạn xây dựng, vì vậy bạn sẽ không cần khởi động lại máy chủ khi thực hiện thay đổi trong quá trình phát triển. Bạn sẽ luôn có thể xem phiên bản cập nhật của trang web trong trình duyệt khi máy chủ dev đang chạy.

Khi xem trang web trong trình duyệt, bạn sẽ có quyền truy cập vào [thanh công cụ dev Astro](/vi/guides/dev-toolbar/). Khi bạn xây dựng, nó sẽ giúp bạn kiểm tra các [islands](/vi/concepts/islands/), phát hiện vấn đề về khả năng truy cập, và nhiều hơn nữa.

Nếu bạn không thể mở dự án trong trình duyệt sau khi khởi động máy chủ dev, hãy quay lại terminal nơi bạn chạy lệnh `dev` và kiểm tra thông báo hiển thị. Nó sẽ cho bạn biết nếu có lỗi xảy ra, hoặc nếu dự án của bạn đang được phục vụ tại URL khác với [http://localhost:4321/](http://localhost:4321/).

## Xây dựng và xem trước trang web

Để kiểm tra phiên bản trang web của bạn sẽ được tạo tại thời điểm build, hãy thoát máy chủ dev (<kbd>Ctrl</kbd> + <kbd>C</kbd>) và chạy lệnh build phù hợp cho trình quản lý package của bạn trong terminal:

<Tabs>
  <TabItem label="npm">
  ```shell
  npm run build
  ```
  </TabItem>
  <TabItem label="pnpm">
  ```shell
  pnpm build
  ```
  </TabItem>
  <TabItem label="yarn">
  ```shell
  yarn run build
  ```
  </TabItem>
</Tabs>

Astro sẽ xây dựng phiên bản sẵn sàng triển khai của trang web trong thư mục riêng biệt (mặc định là `dist/`) và bạn có thể theo dõi tiến trình trong terminal. Điều này sẽ cảnh báo bạn về bất kỳ lỗi build nào trong dự án trước khi bạn triển khai lên production. Nếu TypeScript được cấu hình thành `strict` hoặc `strictest`, script `build` cũng sẽ kiểm tra dự án của bạn để tìm lỗi kiểu dữ liệu.

Khi quá trình build hoàn tất, chạy lệnh `preview` phù hợp (ví dụ: `npm run preview`) trong terminal và bạn có thể xem phiên bản đã build của trang web cục bộ trong cùng cửa sổ xem trước trình duyệt.

Lưu ý rằng điều này xem trước code của bạn như nó tồn tại khi lệnh build được chạy lần cuối. Điều này nhằm mục đích cung cấp cho bạn bản xem trước về cách trang web sẽ trông như thế nào khi được triển khai lên web. Bất kỳ thay đổi nào bạn thực hiện đối với code sau khi build sẽ **không** được phản ánh khi bạn xem trước trang web cho đến khi bạn chạy lại lệnh build.

Sử dụng (<kbd>Ctrl</kbd> + <kbd>C</kbd>) để thoát xem trước và chạy lệnh terminal khác, chẳng hạn như khởi động lại máy chủ dev để quay lại [làm việc trong chế độ phát triển](#work-in-development-mode) để cập nhật khi bạn chỉnh sửa để hiển thị bản xem trước trực tiếp các thay đổi code.

<ReadMore>Đọc thêm về [Astro CLI](/vi/reference/cli-reference/) và các lệnh terminal bạn sẽ sử dụng khi xây dựng với Astro.</ReadMore>

:::tip 
Bạn có thể muốn [triển khai trang web mới ngay lập tức](/vi/guides/deploy/), trước khi bạn bắt đầu thêm hoặc thay đổi quá nhiều code. Điều này hữu ích để có phiên bản tối thiểu, hoạt động của trang web được xuất bản và có thể tiết kiệm thời gian và công sức khắc phục sự cố triển khai sau này.
:::

### Tham gia hướng dẫn giới thiệu

Xây dựng một blog Astro hoàn chỉnh bắt đầu từ một trang trống trong [hướng dẫn giới thiệu](/vi/tutorial/0-introduction/) của chúng tôi.

Đây là cách tuyệt vời để xem Astro hoạt động như thế nào và hướng dẫn bạn qua các khái niệm cơ bản về trang, layouts, components, routing, islands, và nhiều hơn nữa. Nó cũng bao gồm một đơn vị tùy chọn, thân thiện với người mới bắt đầu cho những người mới với các khái niệm phát triển web nói chung, sẽ hướng dẫn bạn cài đặt các ứng dụng cần thiết trên máy tính, tạo tài khoản GitHub, và triển khai trang web của bạn.
