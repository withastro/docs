أمر CLI [`create astro`](#install-from-the-cli-wizard) هو أسرع طريقة لبدء مشروع Astro جديد من الصفر.  
سيأخذك خطوة بخطوة في إعداد مشروعك الجديد، مع إمكانية اختيار أحد القوالب الرسمية الجاهزة.

يمكنك أيضًا تشغيل الأمر مع الوسيطة `template` لبدء مشروعك باستخدام أي قالب أو سمة جاهزة موجودة مسبقًا.  
استعرض [معرض القوالب والسمات](https://astro.build/themes/) لتجد قوالب للمدونات، المحافظ الشخصية، مواقع التوثيق، صفحات الهبوط والمزيد!

إذا كنت تفضل التثبيت اليدوي، راجع [دليل التثبيت اليدوي خطوة بخطوة](#manual-setup).

:::tip[معاينة مباشرة عبر الإنترنت]
هل تفضل تجربة Astro مباشرة من متصفحك؟  
قم بزيارة [astro.new](https://astro.new/) لاستعراض القوالب الجاهزة وبدء مشروع جديد دون مغادرة المتصفح.
:::

## المتطلبات الأساسية

- **Node.js** – الإصدارات `v18.20.8` أو `v20.3.0` أو `v22.0.0` فما فوق. (الإصدارات `v19` و `v21` غير مدعومة.)
- **محرر نصوص** – نوصي باستخدام [VS Code](https://code.visualstudio.com/) مع [الامتداد الرسمي لـ Astro](https://marketplace.visualstudio.com/items?itemName=astro-build.astro-vscode).
- **الطرفية (Terminal)** – يتم التعامل مع Astro من خلال واجهة سطر الأوامر (CLI).

## توافق المتصفح

تم بناء Astro باستخدام Vite، والذي يستهدف المتصفحات الحديثة التي تدعم JavaScript افتراضيًا.  
لمزيد من التفاصيل، راجع [قائمة إصدارات المتصفحات المدعومة في Vite](https://vite.dev/guide/build.html#browser-compatibility).

## التثبيت عبر معالج CLI

يمكنك تشغيل `create astro` من أي مكان في جهازك، ولا حاجة لإنشاء مجلد مسبق لمشروعك.  
إذا لم يكن لديك مجلد فارغ بعد، سيساعدك المعالج في إنشائه تلقائيًا.

<Steps>
1. شغّل الأمر التالي في الطرفية لبدء معالج التثبيت:

    <PackageManagerTabs>
      <Fragment slot="npm">
      ```shell
      # إنشاء مشروع جديد باستخدام npm
      npm create astro@latest
      ```
      </Fragment>
      <Fragment slot="pnpm">
      ```shell
      # إنشاء مشروع جديد باستخدام pnpm
      pnpm create astro@latest
      ```
      </Fragment>
      <Fragment slot="yarn">
      ```shell
      # إنشاء مشروع جديد باستخدام yarn
      yarn create astro
      ```
      </Fragment>
    </PackageManagerTabs>

    عند نجاح العملية، سترى رسالة نجاح متبوعة بخطوات مقترحة تالية.
    
2. بعد إنشاء المشروع، انتقل إلى مجلد المشروع باستخدام `cd` للبدء بالعمل على Astro.

3. إذا تجاوزت خطوة "Install dependencies?" أثناء المعالج، فتأكد من تثبيت التبعيات يدويًا قبل المتابعة:

    <PackageManagerTabs>
      <Fragment slot="npm">
      ```shell
      npm install
      ```
      </Fragment>
      <Fragment slot="pnpm">
      ```shell
      pnpm install
      ```
      </Fragment>
      <Fragment slot="yarn">
      ```shell
      yarn install
      ```
      </Fragment>
    </PackageManagerTabs>

4. يمكنك الآن [تشغيل خادم التطوير الخاص بـ Astro](/ar/develop-and-build/#start-the-astro-dev-server) ومشاهدة المعاينة الحية أثناء البناء.
</Steps>

يمكنك تشغيل الأمر `create astro` مع وسائط إضافية لتخصيص عملية التثبيت (مثل الإجابة "نعم" على جميع الأسئلة أو تجاوز الرسوم المتحركة)، أو لتكوين مشروعك الجديد (مثل تثبيت git أو إضافة التكاملات).

<ReadMore>اطّلع على [كافة وسائط أمر `create astro`](https://github.com/withastro/astro/blob/main/packages/create-astro/README.md).</ReadMore>

### إضافة تكاملات

يمكنك بدء مشروع Astro جديد وتثبيت أي من [التكاملات الرسمية](/ar/guides/integrations-guide/) أو تكاملات المجتمع التي تدعم أمر `astro add` بإضافة الوسيطة `--add`.

<PackageManagerTabs>

  ```shell
  npm create astro@latest -- --add react --add partytown
  ```
  </Fragment>
  <Fragment slot="pnpm">

  ```shell
  # إنشاء مشروع جديد مع React وPartytown
  ```

  </Fragment>
  <Fragment slot="yarn">

</PackageManagerTabs>

### استخدام سمة أو قالب جاهز

يمكنك إنشاء مشروع Astro جديد استنادًا إلى [أحد الأمثلة الرسمية](https://github.com/withastro/astro/tree/main/examples)  
أو إلى الفرع `main` لأي مستودع على GitHub، وذلك عن طريق تمرير الوسيطة `--template` مع الأمر `create astro`.

شغّل أحد الأوامر التالية في الطرفية (Terminal)، مع استبدال `<example-name>` باسم القالب الرسمي الذي تريد استخدامه،  
أو استبدال `<github-username>/<github-repo>` باسم المستخدم والمستودع على GitHub للقالب المطلوب:

<PackageManagerTabs>
  <Fragment slot="npm">
  ```shell
  # إنشاء مشروع جديد باستخدام مثال رسمي
  npm create astro@latest -- --template <example-name>

  # إنشاء مشروع جديد اعتمادًا على الفرع الرئيسي لأحد مستودعات GitHub
  npm create astro@latest -- --template <github-username>/<github-repo>
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```shell
  # إنشاء مشروع جديد باستخدام مثال رسمي
  pnpm create astro@latest --template <example-name>
  
  # إنشاء مشروع جديد اعتمادًا على الفرع الرئيسي لأحد مستودعات GitHub
  pnpm create astro@latest --template <github-username>/<github-repo>
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```shell
  # إنشاء مشروع جديد باستخدام مثال رسمي
  yarn create astro --template <example-name>
  
  # إنشاء مشروع جديد اعتمادًا على الفرع الرئيسي لأحد مستودعات GitHub
  yarn create astro --template <github-username>/<github-repo>
  ```
  </Fragment>
</PackageManagerTabs>

بشكلٍ افتراضي، يستخدم هذا الأمر الفرع `main` من مستودع القالب.  
وللاستخدام من فرع آخر، يمكنك تمرير اسم الفرع كجزء من وسيطة `--template` بهذا الشكل:  
`<github-username>/<github-repo>#<branch>`.

## التثبيت اليدوي

سيرشدك هذا الدليل خلال خطوات تثبيت وإعداد مشروع Astro جديد يدويًا.

إذا كنت تفضل عدم استخدام أداة CLI التلقائية `create astro`، يمكنك إعداد مشروعك بنفسك باتباع الخطوات التالية.

<Steps>
1. إنشاء المجلد الخاص بالمشروع

    أنشئ مجلدًا فارغًا باسم مشروعك، ثم انتقل إليه.

    ```bash
    mkdir my-astro-project
    cd my-astro-project
    ```

    بعد دخولك إلى المجلد، أنشئ ملف `package.json` الخاص بالمشروع.  
    هذا الملف مسؤول عن إدارة تبعيات مشروعك، بما في ذلك Astro نفسه.  
    إذا لم تكن معتادًا على هذا النوع من الملفات، يمكنك إنشاؤه بالأمر التالي:

    <PackageManagerTabs>
      <Fragment slot="npm">
      ```shell
      npm init --yes
      ```
      </Fragment>
      <Fragment slot="pnpm">
      ```shell
      pnpm init
      ```
      </Fragment>
      <Fragment slot="yarn">
      ```shell
      yarn init --yes
      ```
      </Fragment>
    </PackageManagerTabs>

2. تثبيت Astro

    أولًا، قم بتثبيت تبعيات مشروع Astro داخل مجلد مشروعك.

    :::note[مهم]
    يجب تثبيت Astro **محليًا** داخل المشروع وليس عالميًا على النظام.  
    تأكد من عدم تشغيل أي من الأوامر التالية:  
    `npm install -g astro` أو `pnpm add -g astro` أو `yarn add global astro`.
    :::

    <PackageManagerTabs>
      <Fragment slot="npm">
      ```shell
      npm install astro
      ```
      </Fragment>
      <Fragment slot="pnpm">
      ```shell
      pnpm add astro
      ```
      </Fragment>
      <Fragment slot="yarn">
      ```shell
      yarn add astro
      ```
      </Fragment>
    </PackageManagerTabs>

    بعد التثبيت، استبدل قسم "scripts" في ملف `package.json` بالمحتوى التالي:

    ```json title="package.json" del={3} ins={4-6}
    {
      "scripts": {
        "test": "echo \"Error: no test specified\" && exit 1",
        "dev": "astro dev",
        "build": "astro build",
        "preview": "astro preview"
      },
    }
    ```

    ستستخدم هذه الأوامر لاحقًا لتشغيل Astro وتنفيذ أوامره المختلفة.

3. إنشاء أول صفحة

    افتح محرر النصوص وأنشئ ملفًا جديدًا في المسار `src/pages/index.astro`.  
    هذا سيكون أول ملف صفحة في مشروعك.

    انسخ الكود التالي داخل الملف الجديد (بما في ذلك علامات `---`):

    ```astro title="src/pages/index.astro"
    ---
    // مرحبًا بك في Astro! كل ما بين هذه الشرطات الثلاثية هو "مقدمة المكون" ولا يعمل في المتصفح.
    console.log('يتم تشغيل هذا الكود في الطرفية وليس في المتصفح!');
    ---
    <!-- أدناه هو قالب المكون الخاص بك. إنه HTML عادي مع بعض الخصائص المفيدة لبناء القوالب بسهولة. -->
    <html>
      <body>
        <h1>مرحبًا بالعالم!</h1>
      </body>
    </html>
    <style>
      h1 {
        color: orange;
      }
    </style>
    ```

4. إنشاء أول ملف ثابت

    أنشئ مجلدًا باسم `public/` لتخزين الأصول الثابتة (مثل الصور وملفات robots.txt).  
    سيقوم Astro دائمًا بتضمين هذه الملفات في بناء المشروع النهائي.

    في محرر النصوص، أنشئ ملفًا جديدًا في المسار `public/robots.txt`، ثم انسخ الكود التالي:

    ```diff title="public/robots.txt"
    # مثال: السماح لجميع محركات البحث بفحص وفهرسة موقعك.
    # للمزيد من المعلومات: https://developers.google.com/search/docs/advanced/robots/create-robots-txt
    User-agent: *
    Allow: /
    ```

5. إنشاء ملف الإعداد `astro.config.mjs`

    يتم ضبط إعدادات Astro عبر ملف `astro.config.mjs`.  
    هذا الملف اختياري إذا لم تكن بحاجة إلى تخصيص الإعدادات، لكن من الأفضل إنشاؤه الآن.

    أنشئ الملف في جذر المشروع وأضف الكود التالي:

    ```js title="astro.config.mjs"
    import { defineConfig } from "astro/config";

    // https://astro.build/config
    export default defineConfig({});
    ```

    إذا كنت ترغب في استخدام [مكونات واجهات المستخدم](/ar/guides/framework-components/) مثل React أو Svelte،  
    أو إضافة أدوات مثل MDX أو Partytown، يمكنك [إعداد التكاملات يدويًا هنا](/ar/guides/integrations-guide/).

    <ReadMore>اطّلع على [مرجع إعدادات واجهة برمجة تطبيقات Astro](/ar/reference/configuration-reference/) للمزيد من التفاصيل.</ReadMore>

6. إضافة دعم TypeScript

    يتم ضبط TypeScript من خلال ملف `tsconfig.json`.  
    حتى إن لم تكن تكتب شيفرة TypeScript، فإن وجود هذا الملف ضروري لتمكين أدوات مثل Astro وVS Code من فهم المشروع بشكل صحيح.  
    بعض الميزات (مثل استيراد الحزم من npm) لن تعمل بشكل كامل في المحرر بدون هذا الملف.

    إذا كنت تنوي استخدام TypeScript، نوصي باستخدام أحد قوالب Astro `strict` أو `strictest`.  
    يمكنك الاطلاع على القوالب الثلاثة ومقارنتها من خلال [astro/tsconfigs/](https://github.com/withastro/astro/blob/main/packages/astro/tsconfigs/).

    أنشئ ملف `tsconfig.json` في جذر المشروع وأضف إليه الكود التالي (يمكنك اختيار `base` أو `strict` أو `strictest`):

    ```json title="tsconfig.json" "base"
    {
      "extends": "astro/tsconfigs/base"
    }
    ```

    <ReadMore>اقرأ [دليل إعداد TypeScript في Astro](/ar/guides/typescript/#setup) للمزيد من المعلومات.</ReadMore>

7. الخطوات التالية

    بعد تنفيذ الخطوات السابقة، يجب أن يبدو هيكل مشروعك كالتالي:

    <FileTree>
    - node_modules/
    - public/
      - robots.txt
    - src/
      - pages/
        - index.astro
    - astro.config.mjs
    - package-lock.json أو `yarn.lock` أو `pnpm-lock.yaml`
    - package.json
    - tsconfig.json
    </FileTree>

8. يمكنك الآن [تشغيل خادم التطوير الخاص بـ Astro](/ar/develop-and-build/#start-the-astro-dev-server)  
   ومشاهدة المعاينة الحية لموقعك أثناء عملية التطوير!

</Steps>
