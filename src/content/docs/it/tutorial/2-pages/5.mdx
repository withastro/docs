## Aggiungi un foglio di stile globale

Hai visto che il tag `<style>` di Astro è **limitato per default**, il che significa che influisce solo sugli elementi nel proprio file.

Ci sono diversi modi per definire gli stili **globalmente** in Astro, ma in questo tutorial creerai e importerai un file `global.css` in ciascuna delle tue pagine. Questa combinazione di foglio di stile e tag `<style>` ti permette di controllare alcuni stili a livello di sito, e di applicare stili specifici esattamente dove vuoi.

<Steps>
1. Crea un nuovo file nella posizione `src/styles/global.css` (dovrai prima creare una cartella `styles`).

2. Copia il seguente codice nel tuo nuovo file `global.css`:

    ```css title="src/styles/global.css"
    html {
      background-color: #f1f5f9;
      font-family: sans-serif;
    }

    body {
      margin: 0 auto;
      width: 100%;
      max-width: 80ch;
      padding: 1rem;
      line-height: 1.5;
    }

    * {
      box-sizing: border-box;
    }

    h1 {
      margin: 1rem 0;
      font-size: 2.5rem;
    }
    ```

3. In `about.astro`, aggiungi la seguente istruzione di importazione nel frontmatter:

    ```astro title="src/pages/about.astro" ins={2}
    ---
    import '../styles/global.css';

    const pageTitle = "About Me";

    const identity = {
      firstName: "Sarah",
      country: "Canada",
      occupation: "Technical Writer",
      hobbies: ["photography", "birdwatching", "baseball"],
    };

    const skills = ["HTML", "CSS", "JavaScript", "React", "Astro", "Writing Docs"];

    const happy = true;
    const finished = false;
    const goal = 3;

    const skillColor = "navy";
    const fontWeight = "bold";
    const textCase = "uppercase";
    ---
    ```

4. Controlla l'anteprima nel browser della tua About page e dovresti vedere i nuovi stili applicati!
</Steps>

## Prova tu stesso - Importa il tuo foglio di stile globale

Aggiungi la riga di codice necessaria a ciascun file `.astro` del tuo progetto per applicare gli stili globali a ogni pagina del tuo sito.

<details>
<summary>✅ Mostrami il codice! ✅</summary>

Aggiungi la seguente istruzione di importazione anche nei due altri file di pagina: `src/pages/index.astro` e `src/pages/blog.astro`

```astro title="src/pages/index.astro" ins={2}
---
import '../styles/global.css';
---
```
</details>
</Box>

Apporta qualsiasi modifica o aggiunta che desideri al contenuto della tua About page aggiungendo elementi HTML al modello della pagina, sia statici che dinamici. Scrivi qualsiasi JavaScript aggiuntivo nel tuo script di frontmatter per fornirti valori da utilizzare nel tuo HTML. Quando sei soddisfatto di questa pagina, esegui il commit delle modifiche su GitHub prima di passare alla lezione successiva.

### Analizza il pattern

La tua About page è ora stilizzata usando *sia* il file `global.css` importato *sia* un tag `<style>`.

- Gli stili di entrambi i metodi di styling vengono applicati?

    <p>
      <Spoiler>Sì</Spoiler>
    </p>

- Ci sono conflitti tra gli stili? E, se sì, quale viene applicato?

    <p>
      <Spoiler>Sì, `<h1>` ha una dimensione di `2.5rem` globalmente, ma `4rem` localmente nel tag `<style>`. La regola locale di `4rem` viene applicata sulla About page.</Spoiler>
    </p>

- Descrivi come `global.css` e `<style>` lavorano insieme.

    <p>
      <Spoiler>Quando gli stili confliggono, sia a livello globale che nel tag locale `<style>` di una pagina, gli stili locali dovrebbero sovrascrivere quelli globali. (Tuttavia, ci possono essere altri fattori coinvolti, quindi è sempre una buona idea ispezionare visivamente il sito per assicurarti che gli stili siano applicati correttamente!)</Spoiler>
    </p>

- Come sceglieresti se dichiarare uno stile in un file `global.css` o in un tag `<style>`?

    <p>
      <Spoiler>Se desideri che uno stile venga applicato a livello di sito, dovresti scegliere di usare un file `global.css`. Tuttavia, se desideri che gli stili vengano applicati solo al contenuto HTML in un singolo file `.astro`, e non influenzino altri elementi nel sito, dovresti scegliere un tag `<style>`.</Spoiler>
    </p>
