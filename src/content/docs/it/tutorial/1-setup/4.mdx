Questo tutorial utilizzerà GitHub per archiviare il nostro repository e connettersi a un web host. Sei libero di utilizzare il provider git online di tua scelta.

:::note
Se hai già familiarità con git e hai il tuo flusso di lavoro, allora crea un nuovo repository GitHub per il tuo progetto utilizzando il tuo metodo preferito. Passa alla pagina successiva: ["Distribuisci il tuo sito sul web"](/it/tutorial/1-setup/5/).
:::

## Crea un repository su GitHub

Sebbene ci siano diversi modi per archiviare il tuo codice locale in GitHub, questo tutorial ti guiderà attraverso un metodo che non richiede l'utilizzo di git nella riga di comando.

<Steps>
1. Accedi a GitHub.com in un browser e clicca sul <kbd>+</kbd> in alto a destra dello schermo per creare un nuovo repository.

2. Scegli un nome per il tuo repository. Questo non deve essere lo stesso nome della tua cartella di progetto.

3. Ti verranno presentate delle opzioni, ma non è necessario modificare nessuna delle impostazioni predefinite. Scorri verso il basso e clicca sul pulsante <kbd>Create Repository</kbd>.

4. Ti verranno presentati vari passaggi successivi di configurazione, ma non dovrai usarne nessuno. Prendi nota dell'URL del tuo repository. Ora puoi uscire da questa pagina senza fare nulla.
</Steps>

## Esegui il commit del tuo codice locale su GitHub

Nell'ultima sezione, hai apportato una modifica al contenuto della tua pagina. Ciò significa che i file del tuo progetto sono cambiati e VS Code dovrebbe mostrare un numero sopra l'icona del menu "Source". Questa scheda source è dove andrai regolarmente per aggiornare i tuoi file su GitHub.

<Steps>
1. Clicca sulla scheda Controllo del codice sorgente nel tuo VS Code per vedere un elenco di file che sono stati modificati. Se vedi un messaggio che ti dice che devi installare `git`, segui le istruzioni fornite, quindi ricarica VS Code.

2. Clicca sul menu <kbd>•••</kbd> "3 punti" sopra il messaggio di commit e scegli <kbd>Remoto</kbd> > <kbd>Aggiungi Remoto</kbd>.

3. Seleziona <kbd>Aggiungi remoto da GitHub</kbd>. Se necessario, segui i passaggi di autenticazione, quindi torna a VS Code e ripeti questa azione.

4. Dovresti vedere un elenco di tutti i tuoi repository su GitHub. Scegli quello che hai creato per questo progetto. Se non vedi il tuo progetto, incolla direttamente il suo URL GitHub. Potrebbe anche esserti chiesto di dare a questo repository un nome locale. Puoi selezionare qualsiasi nome tu voglia.

5. Nella parte superiore del pannello del menu, ci sarà uno spazio per inserire un **messaggio di commit** (descrizione delle modifiche ai file). Digita `commit iniziale` e premi il pulsante <kbd>Commit</kbd> per eseguire il commit di queste modifiche.

6. Potresti vedere un messaggio che ti dice che non hai commit "in staging" e ti chiede se vuoi metterli in staging. Clicca <kbd>Sempre</kbd> e continua.

7. Infine, l'elenco dei file modificati dovrebbe essere sostituito da un pulsante <kbd>Pubblica</kbd>. Clicca su questo per inviare le modifiche committate a GitHub.
</Steps>

### Visualizza il tuo progetto su GitHub

Per verificare che il tuo progetto sia archiviato correttamente su GitHub, visita GitHub.com e cerca sotto il tuo account un elenco dei tuoi repository. Scegli quello nuovo che hai creato e verifica che contenga i file del tuo progetto Astro.
