Il comando CLI [`create astro`](#install-from-the-cli-wizard) è il modo più veloce per avviare un nuovo progetto Astro da zero. Ti guiderà attraverso ogni fase della configurazione del tuo nuovo progetto Astro e ti permetterà di scegliere tra diversi template ufficiali.

Puoi anche eseguire il comando CLI con il flag `template` per avviare il tuo progetto utilizzando un tema o un template starter esistente. Esplora la nostra [vetrina di temi e starter](https://astro.build/themes/) dove puoi navigare tra temi per blog, portfolio, siti di documentazione, landing page e molto altro!

Per installare Astro manualmente, invece, consulta la nostra [guida all'installazione manuale passo-passo](#manual-setup).

:::tip[Anteprime online]
Preferisci provare Astro nel tuo browser? Visita [astro.new](https://astro.new/) per esplorare i nostri template starter e creare un nuovo progetto Astro senza lasciare il tuo browser.
:::

## Prerequisiti

- **Node.js** - `v18.17.1` o `v20.3.0`, `v22.0.0` o versioni superiori. (Le versioni `v19` e `v21` non sono supportate.)
- **Editor di testo** - Consigliamo [VS Code](https://code.visualstudio.com/) con la nostra [estensione ufficiale di Astro](https://marketplace.visualstudio.com/items?itemName=astro-build.astro-vscode).
- **Terminale** - Astro si utilizza tramite la sua interfaccia da riga di comando (CLI).

## Compatibilità del browser

Astro è costruito con Vite, che supporta di default i browser con JavaScript moderno. Per un riferimento completo, puoi consultare la [lista delle versioni dei browser attualmente supportati da Vite](https://vite.dev/guide/build.html#browser-compatibility).

## Installazione tramite il CLI wizard

Puoi eseguire `create astro` ovunque sul tuo computer, quindi non è necessario creare una nuova directory vuota per il tuo progetto prima di iniziare. Se non hai ancora una directory vuota per il tuo nuovo progetto, il wizard ti aiuterà a crearla automaticamente.

<Steps>
1. Esegui il seguente comando nel tuo terminale per avviare il wizard di installazione:

    <PackageManagerTabs>
      <Fragment slot="npm">
      ```shell
      # crea un nuovo progetto con npm
      npm create astro@latest
      ```
      </Fragment>
      <Fragment slot="pnpm">
      ```shell
      # crea un nuovo progetto con pnpm
      pnpm create astro@latest
      ```
      </Fragment>
      <Fragment slot="yarn">
      ```shell
      # crea un nuovo progetto con yarn
      yarn create astro
      ```
      </Fragment>
    </PackageManagerTabs>

    Se tutto va bene, vedrai un messaggio di successo seguito da alcuni passi successivi consigliati.

2. Ora che il tuo progetto è stato creato, puoi entrare nella tua nuova directory di progetto per iniziare a usare Astro.

3. Se hai saltato il passo "Installare le dipendenze?" durante il wizard CLI, assicurati di installare le dipendenze prima di continuare.

        <PackageManagerTabs>
      <Fragment slot="npm">
      ```shell
      npm install
      ```
      </Fragment>
      <Fragment slot="pnpm">
      ```shell
      pnpm install
      ```
      </Fragment>
      <Fragment slot="yarn">
      ```shell
      yarn install
      ```
      </Fragment>
    </PackageManagerTabs>

4. Ora puoi [avviare il server di sviluppo di Astro](/it/develop-and-build/#start-the-astro-dev-server) e vedere un'anteprima in tempo reale del tuo progetto mentre lavori!
</Steps>

## Flag di installazione CLI

Puoi eseguire il comando `create astro` con flag aggiuntivi per personalizzare il processo di configurazione (ad esempio, rispondere "sì" a tutte le domande, saltare l'animazione Houston) o il tuo nuovo progetto (ad esempio, installare Git o meno, aggiungere integrazioni).

<ReadMore>Vedi [tutti i flag disponibili per il comando `create astro`](https://github.com/withastro/astro/blob/main/packages/create-astro/README.md)</ReadMore>

### Aggiungi integrazioni

Puoi avviare un nuovo progetto Astro e installare qualsiasi [integrazione ufficiale](/it/guides/integrations-guide/) o integrazione della community che supporti il comando `astro add` allo stesso tempo, passando l'argomento `--add` al comando `create astro`.

Esegui il seguente comando nel tuo terminale, sostituendo qualsiasi integrazione che supporta il comando `astro add`:

<PackageManagerTabs>
  <Fragment slot="npm">
  ```shell
  # crea un nuovo progetto con react e tailwind
  npm create astro@latest -- --add react --add tailwind
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```shell
  # crea un nuovo progetto con react e tailwind
  pnpm create astro@latest --add react --add tailwind
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```shell
  # crea un nuovo progetto con react e tailwind
  yarn create astro --add react --add tailwind
  ```
  </Fragment>
</PackageManagerTabs>

### Usa un tema o un template starter

Puoi avviare un nuovo progetto Astro basato su un [esempio ufficiale](https://github.com/withastro/astro/tree/main/examples) o sul ramo `main` di qualsiasi repository GitHub, passando l'argomento `--template` al comando `create astro`.

Esegui il seguente comando nel tuo terminale, sostituendo il nome del template ufficiale di Astro, o il nome utente GitHub e il repository del tema che desideri utilizzare:

<PackageManagerTabs>
  <Fragment slot="npm">
  ```shell
  # crea un nuovo progetto con un esempio ufficiale
  npm create astro@latest -- --template <example-name>

  # crea un nuovo progetto basato sul ramo main di un repository GitHub
  npm create astro@latest -- --template <github-username>/<github-repo>
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```shell
  # crea un nuovo progetto con un esempio ufficiale
  pnpm create astro@latest --template <example-name>
  
  # crea un nuovo progetto basato sul ramo main di un repository GitHub
  pnpm create astro@latest --template <github-username>/<github-repo>
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```shell
  # crea un nuovo progetto con un esempio ufficiale
  yarn create astro --template <example-name>
  
  # crea un nuovo progetto basato sul ramo main di un repository GitHub
  yarn create astro --template <github-username>/<github-repo>
  ```
  </Fragment>
</PackageManagerTabs>

Per impostazione predefinita, questo comando utilizza il ramo `main` del template repository. Per utilizzare un nome di ramo diverso, passalo come parte dell'argomento `--template`: `<github-username>/<github-repo>#<branch>`.

## Installazione Manuale

Questa guida ti guiderà attraverso i passaggi per installare e configurare manualmente un nuovo progetto Astro.

Se preferisci non utilizzare il nostro strumento automatico `create astro`, puoi configurare il tuo progetto manualmente seguendo la guida qui sotto.

<Steps>
1. Crea la tua directory

    Crea una directory vuota con il nome del tuo progetto e naviga al suo interno.

    ```bash
    mkdir my-astro-project
    cd my-astro-project
    ```

    Una volta nella nuova directory, crea il file `package.json` del tuo progetto. Questo file gestirà le dipendenze del tuo progetto, inclusi Astro. Se non conosci questo formato di file, esegui il comando seguente per crearne uno.

    <PackageManagerTabs>
      <Fragment slot="npm">
      ```shell
      npm init --yes
      ```
      </Fragment>
      <Fragment slot="pnpm">
      ```shell
      pnpm init 
      ```
      </Fragment>
      <Fragment slot="yarn">
      ```shell
      yarn init --yes
      ```
      </Fragment>
    </PackageManagerTabs>

2. Installa Astro

    Prima di tutto, installa le dipendenze di Astro all'interno del tuo progetto.

    :::note[Importante]
    Astro deve essere installato localmente, non globalmente. Assicurati di non eseguire `npm install -g astro`, `pnpm add -g astro`

 o `yarn add global astro`.
    :::

    <PackageManagerTabs>
      <Fragment slot="npm">
      ```shell
      npm install astro
      ```
      </Fragment>
      <Fragment slot="pnpm">
      ```shell
      pnpm add astro
      ```
      </Fragment>
      <Fragment slot="yarn">
      ```shell
      yarn add astro
      ```
      </Fragment>
    </PackageManagerTabs>

    Poi, sostituisci qualsiasi sezione "scripts" segnaposto del tuo `package.json` con quanto segue:

    ```json title="package.json" del={2} ins={3-6}
      "scripts": {
        "test": "echo \"Errore: nessun test specificato\" && exit 1",
        "dev": "astro dev",
        "build": "astro build",
        "preview": "astro preview"
      },
    ```

    Userai questi script più avanti nella guida per avviare Astro ed eseguire i suoi diversi comandi.

3. Crea la tua prima pagina

    Nel tuo editor di testo, crea un nuovo file nella tua directory in `src/pages/index.astro`. Questa sarà la tua prima pagina Astro nel progetto.

    Per questa guida, copia e incolla il seguente frammento di codice (inclusi i trattini `---`) nel tuo nuovo file:

    ```astro title="src/pages/index.astro"
    ---
    // Benvenuto in Astro! Tutto ciò che si trova tra questi trattini tripli
    // è la "frontmatter" del componente. Non viene mai eseguito nel browser.
    console.log('Questo viene eseguito nel tuo terminale, non nel browser!');
    ---
    <!-- Qui sotto c'è il template del tuo componente. È solo HTML, ma con
        un po' di magia per aiutarti a costruire template straordinari. -->
    <html>
      <body>
        <h1>Ciao, mondo!</h1>
      </body>
    </html>
    <style>
      h1 {
        color: orange;
      }
    </style>
    ```

4. Crea il tuo primo asset statico

    Vuoi anche creare una directory `public/` per memorizzare i tuoi asset statici. Astro includerà sempre questi asset nel tuo build finale, quindi puoi tranquillamente fare riferimento ad essi all'interno dei tuoi template di componenti.

    Nel tuo editor di testo, crea un nuovo file nella tua directory in `public/robots.txt`. Il file `robots.txt` è un file semplice che molti siti includono per dire ai bot di ricerca come trattare il loro sito.

    Per questa guida, copia e incolla il seguente frammento di codice nel tuo nuovo file:

    ```diff title="public/robots.txt"
    # Esempio: Consenti a tutti i bot di scansionare e indicizzare il tuo sito.
    # Sintassi completa: https://developers.google.com/search/docs/advanced/robots/create-robots-txt
    User-agent: *
    Allow: /
    ```

5. Crea `astro.config.mjs`

    Astro viene configurato tramite `astro.config.mjs`. Questo file è facoltativo se non hai bisogno di configurare Astro, ma potrebbe essere utile crearlo ora.

    Crea `astro.config.mjs` alla radice del tuo progetto e copia il codice sottostante:

    ```js title="astro.config.mjs"
    import { defineConfig } from 'astro/config';

    // https://astro.build/config
    export default defineConfig({});
    ```

    Se desideri includere [componenti di framework UI](/it/guides/framework-components/) come React, Svelte, ecc., o utilizzare altri strumenti come Tailwind o Partytown nel tuo progetto, è qui che dovrai [importare e configurare manualmente le integrazioni](/it/guides/integrations-guide/).

    Consulta il [riferimento API della configurazione di Astro](/it/reference/configuration-reference/) per maggiori informazioni.

6. Aggiungi il supporto per TypeScript

    TypeScript viene configurato tramite `tsconfig.json`. Anche se non scrivi codice TypeScript, questo file è importante affinché strumenti come Astro e VS Code possano capire il tuo progetto. Alcune funzionalità (come l'importazione di pacchetti npm) non sono completamente supportate nell'editor senza un file `tsconfig.json`.

    Se intendi scrivere codice TypeScript, ti consigliamo di utilizzare i template `strict` o `strictest` di Astro. Puoi visualizzare e confrontare le tre configurazioni dei template su [astro/tsconfigs/](https://github.com/withastro/astro/blob/main/packages/astro/tsconfigs/).

    Crea `tsconfig.json` alla radice del tuo progetto e copia il codice sottostante in esso. (Puoi utilizzare `base`, `strict`, o `strictest` per il tuo template TypeScript):

    ```json title="tsconfig.json" "base"
    {
      "extends": "astro/tsconfigs/base"
    }
    ```

    Consulta la [guida alla configurazione di TypeScript in Astro](/it/guides/typescript/#setup) per maggiori informazioni.

7. Passi successivi

    Se hai seguito i passaggi sopra, la tua directory del progetto dovrebbe essere simile a questa:

    <FileTree>
    - node_modules/
    - public/
      - robots.txt
    - src/
      - pages/
        - index.astro
    - astro.config.mjs
    - package-lock.json o `yarn.lock`, `pnpm-lock.yaml`, ecc.
    - package.json
    - tsconfig.json
    </FileTree>

8. Ora puoi [avviare il server di sviluppo di Astro](/it/develop-and-build/#start-the-astro-dev-server) e vedere un'anteprima in tempo reale del tuo progetto mentre lavori!
</Steps>
