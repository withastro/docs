> Missing width and height attributes for `IMAGE_URL`. When using remote images, both dimensions are always required in order to avoid cumulative layout shift (CLS).

## Qu'est-ce qui a mal tourné ?
Pour les images distantes, `width` et `height` ne peuvent pas être automatiquement déduites du fichier original. Pour éviter le décalage cumulatif de la mise en page (CLS), il faut soit spécifier ces deux propriétés, soit mettre [`inferSize`](/fr/reference/modules/astro-assets/#infersize) à `true` pour récupérer les dimensions originales d'une image distante.

Si votre image se trouve dans votre dossier `src`, vous avez probablement voulu l'importer à la place. Voir [le guide des importations pour plus d'informations](/fr/guides/imports/#autres-ressources).
