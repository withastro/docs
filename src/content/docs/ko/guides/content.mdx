Astro는 블로그, 마케팅 사이트, 포트폴리오 등 콘텐츠 중심 사이트를 위한 완벽한 선택입니다!

Astro는 콘텐츠를 작성하고 발표하는 데 도움을 줍니다. Markdown/MDX를 사용하여 Astro에서 직접 블로그 게시물을 작성하거나 헤드리스 CMS에서 콘텐츠를 가져올 수 있습니다. Astro를 사용하면 콘텐츠를 중심으로 사이트를 구축할 수 있습니다. 페이지에 레이아웃을 추가하고, 게시물 색인을 만들고, 독자가 구독할 수 있도록 RSS 피드를 설정할 수 있습니다.

## 콘텐츠 작성 \{#writing-content\}

Astro에서는 다양한 방법으로 콘텐츠를 작성할 수 있습니다.
- Markdown 파일 (`.md` 또는 [대체 확장자](/ko/guides/markdown-content/))은 서식 있는 텍스트 콘텐츠를 쉽게 작성할 수 있도록 설계되었습니다.
- [공식 통합](/ko/guides/integrations-guide/)이 포함된 MDX (`.mdx`) 또는 Markdoc (`.mdoc`) 파일은 문서에 컴포넌트와 동적 표현식을 포함할 수 있습니다.
- [타사 콘텐츠 관리 시스템 (CMS)](#headless-cms-authoring)을 사용하면 콘텐츠를 `.astro` 페이지로 가져올 수 있습니다.
- 콘텐츠가 많은 페이지에서 자주 사용되지 않는 다른 옵션에는 [`.astro` 파일](/ko/basics/astro-pages/#astro-pages) 및 [`.html` 파일](/ko/basics/astro-pages/#html-pages)이 있습니다.

### Markdown 작성 \{#markdown-authoring\}
Markdown은 기본 서식과 헤더, 목록, 이미지 등 공통 요소를 사용하여 서식 있는 텍스트를 작성하기 위한 편리한 구문입니다. Astro에는 프로젝트의 Markdown 파일에 대한 지원이 내장되어 있습니다.

코드 편집기에서 `.md` 파일을 생성하여 작성하거나 즐겨 사용하는 Markdown 편집기에서 작성된 기존 파일을 가져오세요. [StackEdit](https://stackedit.io/) 및 [Dillinger](https://dillinger.io)와 같은 온라인 Markdown 편집기를 사용하면 작업 내용을 GitHub에 저장된 Astro 저장소와 동기화할 수도 있습니다.

<ReadMore>[Astro에서 Markdown 콘텐츠를 작성하는 방법](/ko/guides/markdown-content/)에 대해 자세히 알아보세요.</ReadMore>

### MDX 작성 \{#mdx-authoring\}
프로젝트에 [Astro MDX 통합](/ko/guides/integrations-guide/mdx/)을 추가하면 `.mdx` 파일을 사용하여 콘텐츠를 작성할 수 있습니다. 이를 통해 Markdown에 JavaScript 표현식과 컴포넌트를 포함할 수 있습니다. 여기에는 정적 [Astro 컴포넌트](/ko/basics/astro-components/) 및 대화형 [프레임워크 컴포넌트](/ko/guides/framework-components/)가 모두 포함됩니다. 이를 통해 텍스트 콘텐츠와 함께 배너 또는 대화형 캐러셀과 같은 UI 요소를 포함할 수 있습니다.

코드 편집기에서 프로젝트 파일과 함께 `.mdx` 파일을 직접 작성하고 편집하세요. MDX 파일은 Astro에서 [지원되는 페이지 파일 형식](/ko/basics/astro-pages/#supported-page-files)이며 [콘텐츠 컬렉션 항목](#content-collections)으로 사용될 수도 있습니다.

<ReadMore>[Astro에서 MDX를 사용하는 방법](/ko/guides/integrations-guide/mdx/)에 대해 자세히 알아보세요.</ReadMore>

### 헤드리스 CMS 작성 \{#headless-cms-authoring\}

Storyblok, WordPress 또는 Contentful과 같은 기존 콘텐츠 관리 시스템 (CMS)에서 블로그 게시물을 작성하세요. Storyblok과 같은 일부 CMS는 공식 [Astro 통합](https://www.storyblok.com/mp/announcing-storyblok-astro)을 제공합니다. 다른 CMS들은 Astro 페이지가 [원격 콘텐츠를 가져오는 데](/ko/guides/data-fetching/#fetch-from-a-headless-cms) 사용할 수 있는 JavaScript SDK를 제공합니다.

<ReadMore>다양한 리소스 선택을 위한 [CMS 가이드](/ko/guides/cms/) 컬렉션과 [Astro CMS 통합](https://astro.build/integrations/?search=cms)을 살펴보세요.</ReadMore>

## 콘텐츠 페이지 관리 \{#managing-content-pages\}

### 페이지 파일 \{#page-files\}

`src/pages` 디렉터리에 있는 Markdown 및 MDX 파일은 게시물의 파일 경로에 해당하는 URL에 구축된 Astro의 [파일 기반 라우팅](/ko/guides/routing/)을 사용하여 사이트에 페이지를 자동으로 생성합니다.

### 로컬 콘텐츠 \{#local-content\}

Markdown 및 MDX 파일을 `src/pages` 디렉터리 외부에 유지하고, 대신 `.astro` 페이지가 [해당 콘텐츠를 가져오도록](/ko/guides/markdown-content/#importing-markdown) 선택할 수도 있습니다.

### 콘텐츠 컬렉션 \{#content-collections\}

Astro의 [콘텐츠 컬렉션](/ko/guides/content-collections/)은 `src/content/` 디렉터리에 구성된 Markdown, MDX 또는 Markdoc 콘텐츠로 작업할 수 있는 타입 안정성을 갖춘 강력한 방법을 제공합니다.

### 원격 콘텐츠 \{#remote-content\}

프로젝트 외부 소스에서 [Markdown을 원격으로 가져올](/ko/guides/markdown-content/#fetching-remote-markdown) 수도 있습니다. 이를 위해서는 자체 Markdown 분석기 또는 커뮤니티 통합 (예: [`astro-remote`](https://github.com/natemoo-re/astro-remote))이 필요합니다.

## 콘텐츠 쇼케이스 \{#showcasing-your-content\}

블로그 아카이브 또는 각 블로그 태그에 대한 페이지와 같이 콘텐츠를 구성하고 표시하는 기능을 구축하기 위해 Astro를 사용하면 Markdown 및 MDX 프론트매터에서 [파일 이름과 메타데이터를 가져올 수 있습니다](/ko/reference/api-reference/#astroglob). 또는 항목을 쿼리하여 페이지 콘텐츠 및 경로를 생성하기 위해 [콘텐츠 컬렉션 도우미 유틸리티](/ko/guides/content-collections/#querying-collections)를 사용할 수도 있습니다.

더 많은 청중에게 게시하려면 피드 리더에게 배포할 수 있는 [RSS 피드를 생성](/ko/guides/rss/)하세요.

## 커뮤니티 통합 \{#community-integrations\}

공식 [`@astrojs/mdx`](/ko/guides/integrations-guide/mdx/) 및 [`@astrojs/markdoc`](/ko/guides/integrations-guide/markdoc/) 통합 외에도, Astro 프로젝트의 콘텐츠 작업을 위한 여러 타사 [커뮤니티 통합](https://astro.build/integrations/?search=content)이 있습니다.
