우리는 규모에 상관없이 모든 기여와 모든 기술 수준의 기여자를 환영합니다. 오픈 소스 프로젝트로서 우리는 기여자에게 보답하는 것을 믿습니다. 우리는 PR, 기술 문서 작성 및 모든 기능 아이디어를 현실로 만드는 데 대한 지침을 기꺼이 도와드립니다.

더욱 적극적으로 참여하고 싶으신가요? 다양한 역할, 관리자 추천 절차, 코드 검토 절차 및 행동 강령 시행에 대한 자세한 설명은 [거버넌스 문서](https://github.com/withastro/.github/blob/main/GOVERNANCE.md)를 참조하세요.

## 기여 방법

### 프로젝트 리포지토리

Astro 프로젝트에 기여할 수 있는 방법은 아주 많습니다! 모든 Astro 리포지토리에는 프로젝트 루트에 있는 `CONTRIBUTING.md` 파일에 대한 링크가 포함된 README가 있습니다.

[Astro의 GitHub 프로필](https://github.com/withastro)을 방문하여 다음 리포지토리를 찾아보세요.

- 공식 통합 및 시작 템플릿을 포함한 [기본 Astro 코드베이스](https://github.com/withastro/astro)

- Astro 웹사이트 전체인 [Astro 문서](https://github.com/withastro/docs)! 작성된 콘텐츠뿐만 아니라 접근성, CSS, UI 및 UX 문제를 해결하는 Astro 코드에도 기여할 수 있습니다. 또한 여러 언어로 문서를 제공하고 있으므로 사이트 전체 번역에 도움이 필요합니다.

- WASM으로 배포되는 Go로 작성된 [Astro 컴파일러](https://github.com/withastro/compiler)

- Astro 언어 (`.astro` 파일)에 필요한 편집기 도구인 Astro의 [언어 도구](https://github.com/withastro/language-tools)

- Astro의 공식 문서 프레임워크인 [Starlight](https://github.com/withastro/starlight)

- Astro의 미래가 만들어지는 곳인 [Astro 로드맵](https://github.com/withastro/roadmap)! Astro 프로젝트에 대한 아이디어, 제안 및 공식 RFC 제안

### 기여 유형

자신의 코드나 콘텐츠를 기여하는 것 외에도 PR에 리뷰 댓글을 남기고, 기존 GitHub 이슈 및 토론에 아이디어를 추가하고, "고정된" 이슈 유지 관리 작업에 참여하는 등 다양한 방법으로 크게 기여할 수 있습니다!

모든 PR, 특히 번역 PR에는 검토자가 필요합니다! PR을 검토하고 댓글, 제안 또는 "LGTM!" ("Looks Good To Me!") 승인 메시지를 남기는 것은 모든 리포지토리에서 시작하고, Astro에 대해 자세히 알아보는 좋은 방법입니다.

또한 매우 활발한 [Discord](https://astro.build/chat) 커뮤니티가 있습니다! 우리는 신규 회원을 환영하고, 지원 질문에 답변하고, Astro로 만들었거나 Astro를 위해 만든 것을 공유하는 사람들의 기여를 소중하게 생각합니다! 기존의 GitHub 기여 외에도 Astro는 커뮤니티에 참여하고, 블로그 게시물, 비디오 및 컨퍼런스 강연에서 Astro를 공유하고, 커뮤니티의 건전성을 유지하는 데 도움을 주는 커뮤니티 구성원을 인정하고 지원합니다.

## 문서에 기여

Astro 문서에 기여하는 데 도움이 되는 몇 가지 가이드가 있습니다.

오픈 소스에 처음 기여하는 경우든, 방금 만든 새로운 Astro 기능에 대한 문서를 추가해야 하는 경우든, 번역할 다음 페이지를 찾는 숙련된 번역가인 경우든, PR 검토자로 도움을 주는 방법에 대해 자세히 알고 싶은 경우든... 저희가 도와드리겠습니다!

오타 수정자, 작가, 번역가, 기능 개발자, 심지어 PR 검토자로 Astro 문서에 기여하는 데 도움이 되는 설명서를 찾을 수 있는 전용 사이트인 [*Astro Docs* 문서](https://contribute.docs.astro.build)를 방문하세요.
