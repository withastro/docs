## Â¿Por quÃ© Nano Stores?

La librerÃ­a [Nano Stores](https://github.com/nanostores/nanostores) te permite crear _stores_ con las que cualquier componente puede interactuar. Recomendamos Nano Stores porque:
- **Son livianas.** Nano Stores tiene la cantidad mÃ­nima de JS que puedas llegar a requerir (menos de 1 KB) con cero dependencias.
- **Son framework-agnÃ³sticas.** Â¡Esto significa que puedes compartir estado entre _frameworks_ sin contratiempos! Astro estÃ¡ construido para ser flexible, asÃ­ que amamos las soluciones que ofrecen una experiencia de desarrollo similar, sin importar tu preferencia.

Aun asÃ­, hay otras alternativas a explorar. Entre ellas puedes encontrar:
- [Svelte stores incorporadas](https://svelte.dev/tutorial/writable-stores)
- [Solid signals](https://www.solidjs.com/docs/latest) fuera de un elemento de contexto
- [API de reactividad de Vue](https://vuejs.org/guide/scaling-up/state-management.html#simple-state-management-with-reactivity-api)
- [Enviar eventos personalizados del navegador](https://developer.mozilla.org/es/docs/Web/Events/Creating_and_triggering_events) entre componentes

<summary>**ðŸ™‹ Â¿Por quÃ© usamos `.get()` aquÃ­ en vez de un helper `useStore`?**</summary>

HabrÃ¡s notado que estamos llamando a `cartItems.get()` aquÃ­, en vez de usar el helper `useStore` de nuestros ejemplos de React / Preact / Solid / Vue. Como estamos leyendo este valor cuando un **evento** es accionado (`addToCart` en este caso), y no estamos intentando renderizar ese valor, en este caso no necesitamos `useStore`.
</details>
