## ¿Por qué usar un CMS?

Un Sistema de Gestión de Contenido te permite escribir contenido y gestionar recursos fuera de tu proyecto de Astro.

Esto desbloquea nuevas funcionalidades para trabajar con contenido. La mayoría de los CMS te ofrecen un editor de contenido visual, la capacidad de especificar tipos estándar de contenido y una forma de colaborar con otros. 

Un CMS puede ser útil para contenido que sigue una estructura particular, a menudo ofreciéndote una experiencia tipo panel de control y herramientas de edición WYSIWYG. Podrías usar un CMS para escribir publicaciones de blog usando el editor de texto enriquecido del CMS en lugar de archivos Markdown. O podrías usar un CMS para mantener listados de productos para una tienda de eCommerce, haciendo que ciertos campos sean obligatorios para evitar listados incompletos. 

Tu proyecto de Astro puede entonces obtener tu contenido desde tu CMS y mostrarlo, donde y como quieras en tu sitio.


## ¿Qué CMS funcionan bien con Astro?

Dado que Astro se encarga de la _presentación_ de tu contenido, querrás elegir un CMS _headless_, como los de la lista anterior. Esto significa que el CMS te ayuda a escribir tu contenido, pero no genera un sitio que lo muestre. En su lugar, obtienes los datos del contenido y los usas en tu proyecto de Astro. 

Algunos CMS headless, como Storyblok, proporcionan una [integración](/es/guides/integrations/) de Astro que ayuda a obtener el contenido específicamente para un sitio de Astro. Otros proporcionan un SDK de JavaScript, una biblioteca que instalas y usas para obtener tu contenido remoto.

## ¿Puedo usar Astro sin un CMS?

¡Sí! Astro proporciona soporte integrado para [Markdown](/es/guides/markdown-content/).
