## ¿Por qué usar un DAM o medios alojados?

Usar un DAM, o Gestor de Activos Digitales, ayuda a individuos, equipos y organizaciones a gestionar sus activos de imagen y video desde una ubicación central, de manera similar a un [CMS](/es/guides/cms/).

La diferencia es el tipo de contenido que se gestiona: un DAM gestionaría principalmente imágenes, videos, otros activos multimedia como modelos 3D y cualquier metadato asociado con esos activos.

Esto puede ser útil, especialmente cuando se utiliza una única fuente de verdad para tus activos entre múltiples propiedades web o móviles. Esto es importante si eres parte de una organización que requiere que varios equipos utilicen los mismos activos, o si te estás integrando con otros sistemas de contenido como un PIM (Gestor de Información de Productos) para conectar tus activos a los productos.

## ¿Qué sistemas de medios alojados o DAMs funcionan bien con Astro?

Al igual que cuando se usa un CMS, ya que Astro maneja la _presentación_ de tu contenido, querrás usar un DAM headless que te permita obtener e interactuar con tus activos a través de una API o SDK.

Algunos DAMs headless, como Cloudinary, proporcionan una [integración](/es/guides/integrations-guide/) de Astro que te permite obtener fácilmente tus activos, así como mostrarlos en tu sitio web o aplicación.

## ¿Puedo usar Astro sin un sistema de medios alojados o DAM?

¡Sí! Astro proporciona formas integradas para [almacenar imágenes](/es/guides/images/#dónde-guardar-las-imágenes), incluido el soporte para hacer referencia a imágenes remotas.
