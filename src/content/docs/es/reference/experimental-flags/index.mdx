---
title: Configurando opciones experimentales
i18nReady: true
---

Las funcionalidades experimentales solo estarán disponibles después de activar la opción correspondiente en el archivo de configuración de Astro.


```js title="astro.config.mjs"
import { defineConfig } from 'astro/config';

export default defineConfig({
    experimental: {
        // Habilitar las opciones experimentales
        // para probar nuevas caracteristicas en desarrollo
    },
});
```


Astro ofrece opciones experimentales que permiten a los usuarios acceder anticipadamente a nuevas funcionalidades, para que puedan probarlas y brindar retroalimentación.

Estas opciones permiten a los usuarios participar en el desarrollo de funcionalidades reportando errores y compartiendo sus opiniones. Dichas funcionalidades no garantizan estabilidad y pueden incluir cambios incompatibles incluso en pequeños lanzamientos de `patch` mientras se encuentran en desarrollo activo.

Recomendamos [actualizar Astro](/es/upgrade-astro/#actualizar-a-la-última-versión) frecuentemente, y mantenerse al tanto con las notas de lanzamiento en el [changelog de Astro](https://github.com/withastro/astro/blob/main/packages/astro/CHANGELOG.md) las cuales le informarán de cualquier cambio necesario en el código de su proyecto. La documentación de las funcionalidades experimentales siempre se actualizará únicamente para la versión actualmente lanzada.
