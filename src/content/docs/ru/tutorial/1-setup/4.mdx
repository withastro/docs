В этом пособии мы будем использовать GitHub для хранения нашего репозитория и подключения к веб-хостингу. Вы можете использовать онлайн-провайдера Git по вашему выбору.

:::note
Если вы уже знакомы с Git и имеете свой собственный рабочий процесс, создайте новый репозиторий GitHub для вашего проекта, используя предпочтительный метод. Перейдите к следующей странице: ["Разместите свой сайт в Интернете"](/ru/tutorial/1-setup/5/).
:::

## Создайте репозиторий на GitHub

Хотя существует несколько способов сохранить ваш локальный код на GitHub, это пособие проведет вас через метод, который не требует использования git в командной строке.

<Steps>
1. Войдите на сайт GitHub.com в браузере и нажмите на <kbd>+</kbd> в правом верхнем углу экрана, чтобы создать новый репозиторий.

2. Выберите имя для вашего репозитория. Это имя не обязательно должно совпадать с именем вашей папки проекта.

3. Вам будут предложены варианты, но не нужно менять никаких настроек по умолчанию. Прокрутите страницу вниз и нажмите кнопку <kbd>Create Repository</kbd>(Создать репозиторий).

4. Вам будут предложены различные шаги настройки, но ни один из них вам не понадобится. Запишите URL-адрес вашего репозитория. Теперь вы можете покинуть эту страницу, ничего не делая.
</Steps>

## Зафиксируйте ваш локальный код на GitHub

В предыдущем разделе вы внесли изменения в содержимое вашей страницы. Это означает, что файлы вашего проекта изменились, и VS Code должен показать число над иконкой меню "Source". Вкладка "Source" - это место, куда вы регулярно будете переходить, чтобы обновить ваши файлы на GitHub.

<Steps>
1. Перейдите на вкладку Source Control в VS Code, чтобы увидеть список измененных файлов. Если вы увидите сообщение о необходимости установить `git`, следуйте приведенным инструкциям, а затем перезагрузите VS Code.

2. Щелкните меню <kbd>•••</kbd> "3 точки" над сообщением коммита и выберите <kbd>Remote</kbd> > <kbd>Add Remote</kbd>.

3. Выберите <kbd>Add remote from GitHub</kbd>. Если необходимо, выполните все шаги по аутентификации, затем вернитесь в VS Code и повторите это действие.

4. Вы должны увидеть список всех ваших репозиториев на GitHub. Выберите тот, который вы создали для этого проекта. Если вы не видите свой проект, вставьте его URL-адрес GitHub напрямую. Вам также может быть предложено дать этому репозиторию локальное имя. Вы можете выбрать любое имя, которое вам нравится.

5. В верхней части панели меню будет место для ввода **commit message** (описание изменений в файле). Введите `initial commit` и нажмите кнопку <kbd>Commit</kbd>, чтобы зафиксировать эти изменения.

6. Вы можете увидеть сообщение о том, что у вас нет "зафиксированных" коммитов, и спросить, хотите ли вы их зафиксировать. Выберите <kbd>Always</kbd> и продолжайте.

7. Наконец, список измененных файлов должен быть заменен кнопкой <kbd>Publish</kbd>. Нажмите ее, чтобы отправить ваши зафиксированные изменения на GitHub.
</Steps>

### Посмотрите свой проект на GitHub

Чтобы убедиться, что ваш проект успешно сохранен на GitHub, посетите сайт GitHub.com и найдите в своем аккаунте список ваших репозиториев. Выберите новый, созданный вами, и убедитесь, что он содержит файлы вашего проекта Astro.
