[Bun](https://bun.sh/)はオールインワンのJavaScriptランタイム＆ツールキットです。詳しくは[Bunのドキュメント](https://bun.sh/docs)を参照してください。

:::caution
BunをAstroと一緒に使うと、荒削りな部分が見えてくるかもしれません。いくつかのインテグレーションは期待通りに動作しないかもしれません。詳しくは[BunのAstroとの連携に関する公式ドキュメント](https://bun.sh/guides/ecosystem/astro)を参照してください。

Bunを使用していて何か問題がある場合は、[GitHubのBunのリポジトリに直接Issueを開いてください](https://github.com/oven-sh/bun/issues/new/choose)。
:::

## 前提条件

- あなたのマシンにローカルにBunがインストールされていること。Bunの公式ドキュメントの[インストール手順](https://bun.sh/docs/installation)を参照してください。

## Bunで新しいAstroプロジェクトを作成する

以下の`create-astro`コマンドを使用して、Bunで新しいAstroプロジェクトを作成します。

```bash
bun create astro my-astro-project-using-bun
```

## 依存関係のインストール

CLIウィザードで"Install dependencies?"のステップを飛ばした場合は、依存関係をインストールしてから続行してください。

```bash
bun install
```

## 型の追加

Bunは[`@types/bun`](https://www.npmjs.com/package/@types/bun)パッケージを公開しており、Bunのランタイム型が含まれています。

以下のコマンドを使って`@types/bun`をインストールしてください。

```sh
bun add -d @types/bun
```

## CLIのインストールフラグ

### Astroインテグレーションを使う

`astro add`コマンドにより、Astroの公式インテグレーションも使えます。

```bash
bun astro add react
```

### テーマやスターターテンプレートを使う

`create astro`コマンドに`--template`引数を渡すことで、[公式サンプル](https://github.com/withastro/astro/tree/main/examples)や任意のGitHubリポジトリのmainブランチをもとに新しいAstroプロジェクトを作成できます。

ターミナルで以下のコマンドを実行してください。公式のAstroスターターテンプレート名、または使用したいテーマのGitHubユーザー名とリポジトリ名に置き換えてください。

```bash
# 公式サンプルで新しいプロジェクトを作成
bun create astro@latest --template <example-name>
# GitHubリポジトリのmainブランチをもとに新しいプロジェクトを作成
bun create astro@latest --template <github-username>/<github-repo>
```

## 開発とビルド

開発サーバーを起動するには、以下のコマンドを使用します。

```bash
bun run dev
```

### サイトのビルドとプレビュー

サイトをビルドするには、以下のコマンドを使用します。

```bash
bun run build
```

ビルドが完了したら、ターミナルで適切なプレビューコマンド（例: `bun run preview`）を実行すると、ビルドされたサイトをローカルのブラウザプレビューウィンドウで確認できます。

## テスト

Bunには、[`bun test`コマンド](https://bun.sh/docs/cli/test)による高速で組み込みのJest互換テストランナーが付属しています。その他の[Astro用テストツール](/ja/guides/testing/)も使用できます。
