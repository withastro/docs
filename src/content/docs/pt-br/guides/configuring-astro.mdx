O Astro é um framework flexível não opinado que te permite configurar o seu projeto de várias formas diferentes. Isso significa que, ao começar um novo projeto, pode parecer difícil: não existe uma "melhor maneira" de configurar o seu projeto Astro!

O guia nessa seção "Configurações" vai te ajudar a se familiarizar com os vários tipos de arquivos que te permitem configurar e customizar os aspectos do seu projeto e da sua experiência de desenvolvimento.

Se esse é o seu primeiro projeto com Astro, ou se já faz algum tempo que você criou um projeto, você pode seguir os guias e referências nessa documentação como assistência.

## O arquivo config do Astro

O [arquivo config do Astro](/pt-br/reference/configuration-reference/) é um arquivo JavaScript que existe na raiz de todo novo projeto:

```js
// astro.config.mjs
import { defineConfig } from "astro/config";

export default defineConfig({
  // suas configurações aqui...
});
```

Só é obrigatório caso você precise fazer alguma configuração, mas a maioria dos projetos vai usar esse arquivo. O helper `defineConfig()` vai automaticamente disponibilizar o IntelliSense e é onde você vai adicionar suas configurações para dizer ao Astro como fazer a build e renderizar HTML do seu projeto.

Nós recomendados por padrão que use o formato `.mjs` na maioria das vezes, ou `.ts` se quiser escrever código TypeScript no seu arquivo de configuração. De qualquer maneira, `astro.config.js` e `astro.config.cjs` são suportados.

<ReadMore>Leia [configurações de referência](/pt-br/reference/configuration-reference/) do Astro para uma visão geral de todas as configurações suportadas.</ReadMore>

## O arquivo config TypeScript

Todo novo projeto Astro inclui um aquivo `tsconfig.json`. O [script do componente](/pt-br/basics/astro-components/#the-component-script) Astro é TypeScript, que a ferramenta do Astro disponibiliza e que opcionalmente te permite ter a sintaxe JavaScript no seu projeto e fazer a verificação de tipos.

Use o arquivo `tsconfig.json` para configurar o template do TypeScript que fará a execução da verificação de tipos no seu código.

<ReadMore>Leia o [guia do TypeScript](/pt-br/guides/typescript/) do Astro para uma visão geral das opções no TypeScript e os tipos utilitários inclusos no Astro.</ReadMore>

## Experiência de desenvolvimento

Enquanto você trabalha no modo web, você pode ter vantagens do no editor de código e outras ferramentas para melhorar a experiência de desenvolvimento com Astro.

O Astro disponibiliza sua própria extensão oficial do VS Code e é compatível com várias outros editores populares. O Astro também disponibiliza uma barra de ferramentas personalizável que é exibida no preview do browser enquanto o servidor de desenvolvimento está em execução. Você pode instalar e até mesmo construir suas próprias barras de ferramentas para funcionalidades adicionais.

<ReadMore>Leia o guia [configuração do editor](/pt-br/editor-setup/) e [usando a barra de ferramentas](/pt-br/guides/dev-toolbar/) para aprender a customizar a sua experiência de desenvolvimento.</ReadMore>

## Tarefas comuns em um novo projeto

Aqui estão alguns primeiros passos que você pode escolher fazer em um novo projeto Astro.

### Adicione seu domínio deploy

Para gerar o sitemap e criar URLs canônicas, configure a sua URL de desenvolvimento em opções de [`site`](/pt-br/reference/configuration-reference/#site). Se você está fazendo deploy para um caminho (e.g. `www.example.com/docs`), você também pode configurar a [`base`](/pt-br/reference/configuration-reference/#base) para a raiz do seu projeto.

Adicionalmente, diferentes hospedagens podem ter comportamentos diferentes, como uma barra no final da sua URL. (e.g. `example.com/about` vs `example.com/about/`). Assim que o seu site estiver no ar, você pode configurar as preferências da sua [`trailingSlash`](/pt-br/reference/configuration-reference/#trailingslash)(barra final).

```js title="astro.config.mjs"
import { defineConfig } from "astro/config";

export default defineConfig({
  site: "https://www.example.com",
  base: "/docs",
  trailingSlash: "always",
});
```

### Adicionando metadata ao site

O Astro não usa seu arquivo de configuração para SEO comum ou meta dados, apenas informações que precisam ser feito o build do código do seu projeto e renderiza-lo para HTML.

Por isso, essa informação é adicionada ao `<head>` usando o padrão HTML `<link>` e `<meta>` tags, somente se forem escritas diretamente na sua página HTML.

Um padrão comum para sites com Astro, é criar um [component `.astro`](/pt-br/basics/astro-components/) `<Head />` que pode ser adicionado á um [component layout](/pt-br/basics/layouts/) em comum e então ser aplicado a todas as suas páginas.

```astro title="src/components/MainLayout.astro"
---
import Head from "./Head.astro";

const { ...props } = Astro.props;
---
<html>
  <head>
    <meta charset="utf-8">
    <Head />
    <!-- Elementos adicionais do head -->
  </head>
  <body>
    <!-- O conteúdo da página vai aqui -->
  </body>
</html>
```

Como `Head.astro` é apenas um componente regular do Astro, você pode importar arquivos e receber propriedades passadas de outro componente, como um título específico em uma página.

```astro title="src/components/Head.astro"
---
import Favicon from "../assets/Favicon.astro";
import SomeOtherTags from "./SomeOtherTags.astro";

const { title = "Meu website Astro", ...props } = Astro.props;
---
<link rel="sitemap" href="/sitemap-index.xml">
<title>{title}</title>
<meta name="description" content="Bem-vindo(a) ao meu novo site Astro">

<!-- Web analytics -->
<script data-goatcounter="https://my-account.goatcounter.com/count" async src="//gc.zgo.at/count.js"></script>

<!-- Open Graph tags -->
<meta property="og:title" content="Meu novo website Astro" />
<meta property="og:type" content="website" />
<meta property="og:url" content="http://www.example.com/" />
<meta property="og:description" content="Bem-vindo(a) ao meu novo site Astro" />
<meta property="og:image" content="https://www.example.com/_astro/seo-banner.BZD7kegZ.webp">
<meta property="og:image:alt" content="">

<SomeOtherTags />

<Favicon />
```
