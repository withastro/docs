Astro é a escolha perfeita para seu site focado em conteúdo: blogs, sites de marketing, portfólios, e mais!

Astro ajuda você a criar e apresentar seu conteúdo. Você pode escrever um post de blog diretamente no Astro usando Markdown/MDX, ou obter seu conteúdo de um CMS headless. Astro permite que você construa seu site ao redor do seu conteúdo: você pode adicionar um layout para suas páginas, criar um índice dos posts, e configurar um feed RSS para permitir que leitores se inscrevam.

## Escrevendo Conteúdo

No Astro, você pode criar seu conteúdo de várias formas: 
- Em arquivos Markdown (`.md` ou [extensões alternativas](/pt-br/guides/markdown-content/)), projetados para facilitar a escrita de conteúdo de texto.
- Em arquivos MDX (`.mdx`) ou Markdoc (`.mdoc`) com [uma integração oficial](/pt-br/guides/integrations-guide/), os quais podem incluir componentes e expressões dinâmicas em seu documento.
- Usando um [sistema de gerenciamento de conteúdo (CMS) de terceiros](#criação-com-cms-headless), e puxando esse conteúdo para uma página `.astro`.
- Outras opções (usadas menos frequentemente para páginas com muito conteúdo) incluem [arquivos `.astro`](/pt-br/basics/astro-pages/#páginas-astro) e [arquivos `.html`](/pt-br/basics/astro-pages/#páginas-html).

### Criação Markdown
Markdown é uma sintaxe conveniente para escrever texto rico com formatação básica e elementos comuns como cabeçalhos, listas, e imagens. Astro possui suporte embutido para arquivos Markdown em seu projeto.

Crie e escreva um novo arquivo `.md` em seu editor de código ou traga um arquivo existente escrito no seu editor Markdown favorito. Alguns editores Markdown online como [StackEdit](https://stackedit.io/) e [Dillinger](https://dillinger.io) permitirão até que você edite e sincronize seu trabalho com o repositório Astro armazenado no GitHub.

<ReadMore>Saiba mais sobre [escrever conteúdo Markdown no Astro](/pt-br/guides/markdown-content/).</ReadMore>

### Criação MDX
Se você adicionar a [integração Astro MDX](/pt-br/guides/integrations-guide/mdx/) ao seu projeto, você também pode escrever conteúdo usando arquivos `.mdx`, que podem incluir expressões JavaScript e componentes dentro do seu Markdown. Isso inclui tanto [componentes Astro](/pt-br/basics/astro-components/) estáticos quanto [componentes de framework](/pt-br/guides/framework-components/) interativos. Isso permite que você inclua elementos de UI como um banner ou um carrossel interativo junto do seu conteúdo em texto.

Escreva e edite arquivos `.mdx` diretamente no seu editor de código, junto com os arquivos do seu projeto. Arquivos MDX são um [tipo de arquivo de página suportados](/pt-br/basics/astro-pages/#tipos-de-arquivos-suportados) no Astro, e também podem ser usados como [entradas de coleção de conteúdo](#coleções-de-conteúdo).

<ReadMore>Saiba mais sobre [usar MDX com Astro](/pt-br/guides/integrations-guide/mdx/).</ReadMore>

### Criação com CMS Headless

Escreva posts de blog no seu Sistema de Gerenciamento de Conteúdo (CMS) existente como Storyblok, WordPress, ou Contentful. Alguns CMSs, como Storyblok, fornecem uma [integração Astro](https://www.storyblok.com/mp/announcing-storyblok-astro) oficial. Outros expõem um SDK JavaScript que páginas Astro podem usar para [buscar seu conteúdo remoto](/pt-br/guides/data-fetching/#busque-de-um-cms-headless).

<ReadMore>Explore nossa coleção de [guias CMS](/pt-br/guides/cms/) e [integrações CMS Astro](https://astro.build/integrations/?search=cms) para uma ampla seleção de recursos.</ReadMore>

## Gerenciando páginas de conteúdo

### Arquivos de página

Arquivos Markdown e MDX que vivem no seu diretório `src/pages` gerarão automaticamente páginas no seu site usando o [roteamento baseado em arquivos](/pt-br/guides/routing/) do Astro, construídos em uma URL correspondente ao caminho de arquivo do post. 

### Conteúdo local

Você também pode escolher manter o seus arquivos Markdown e MDX fora do diretório `src/pages`, e em vez disso [importar seu conteúdo](/pt-br/guides/markdown-content/#importando-markdown) em páginas `.astro`.

### Coleções de conteúdo

As [coleções de conteúdo](/pt-br/guides/content-collections/) Astro fornecem uma maneira poderosa e com segurança de tipos para trabalhar com conteúdo Markdown, MDX, ou Markdoc organizado no diretório `src/content/`. 

### Conteúdo remoto

Você também pode [obter markdown remotamente](/pt-br/guides/markdown-content/#buscando-markdown-remoto) de uma fonte fora do seu projeto. Isso irá exigir o seu próprio parser Markdown, ou uma integração da comunidade (e.g. [`astro-remote`](https://github.com/natemoo-re/astro-remote)).

## Mostrando seu conteúdo

Para construir funcionalidades comuns para organizar e mostrar seu conteúdo, como um arquivo de blog ou uma página para cada tag de post, Astro permite que você [busque nomes de arquivo e metadados](/pt-br/reference/api-reference/#astroglob) do seu frontmatter do Markdown e MDX ou usar [utilidades de auxílio de coleções de conteúdo](/pt-br/guides/content-collections/#consultando-coleções) para consultar suas entradas e gerar rotas e páginas de conteúdo.

Para publicar para uma audiência maior, [crie um feed RSS](/pt-br/guides/rss/) que pode ser distribuído para leitores de feed.

## Integrações da Comunidade

Em adição às integrações oficiais [`@astrojs/mdx`](/pt-br/guides/integrations-guide/mdx/) e [`@astrojs/markdoc`](/pt-br/guides/integrations-guide/markdoc/), existem muitas [integrações da comunidade](https://astro.build/integrations/?search=content) de terceiros para trabalhar com conteúdo em seu projeto Astro.
