## Jalankan wizard setup Astro

Cara yang disarankan untuk membuat situs Astro baru adalah melalui wizard `create astro`.

<Steps>
1. Di command line terminal Anda, jalankan perintah berikut menggunakan manajer paket pilihan Anda:

    <PackageManagerTabs>
      <Fragment slot="npm">
      ```shell
      # create a new project with npm
      npm create astro@latest
      ```
      </Fragment>
      <Fragment slot="pnpm">
      ```shell
      # create a new project with pnpm
      pnpm create astro@latest
      ```
      </Fragment>
      <Fragment slot="yarn">
      ```shell
      # create a new project with yarn
      yarn create astro
      ```
      </Fragment>
    </PackageManagerTabs>

2. Ketik `y` untuk memasang `create-astro`.
3. Saat diminta lokasi pembuatan proyek, ketik nama folder untuk membuat direktori baru, misalnya
`./tutorial`

    :::note
    Proyek Astro baru hanya dapat dibuat di folder yang benar-benar kosong, jadi pilih nama folder yang belum ada!
    :::

4. Anda akan melihat daftar pendek *starter template*. Gunakan tombol panah (atas dan bawah) untuk memilih template minimal (kosong), lalu tekan *enter* untuk mengonfirmasi.
5. Saat ditanya apakah ingin memasang *dependencies*, masukkan `y`.
6. Saat ditanya apakah ingin menginisialisasi repositori git baru, masukkan `y`.
</Steps>

Setelah wizard instalasi selesai, Anda tidak lagi memerlukan terminal ini. Sekarang Anda dapat membuka VS Code untuk melanjutkan.

## Buka proyek Anda di VS Code

<Steps>
7. Buka VS Code. Anda akan diminta memilih folder. Pilih folder yang Anda buat saat wizard setup.
8. Jika ini pertama kalinya Anda membuka proyek Astro, Anda akan melihat notifikasi untuk memasang ekstensi yang direkomendasikan. Klik untuk melihat rekomendasi, lalu pilih [ekstensi dukungan bahasa Astro](https://marketplace.visualstudio.com/items?itemName=astro-build.astro-vscode). Ini akan menyediakan *syntax highlighting* dan *autocompletion* untuk kode Astro Anda.
9. Pastikan terminal terlihat dan Anda dapat melihat prompt perintah, misalnya:

    ```sh
    user@machine:~/tutorial$
    ```

    :::tip[Shortcut keyboard]
    Untuk menampilkan/menyembunyikan terminal, gunakan <kbd>Ctrl + J</kbd> (macOS: <kbd>Cmd âŒ˜ + J</kbd>).
    :::
</Steps>

Sekarang Anda dapat menggunakan terminal yang tertanam di jendela ini, alih-alih aplikasi Terminal komputer Anda, untuk sisa tutorial.

## Jalankan Astro dalam mode dev

Agar dapat mempratinjau berkas proyek *sebagai sebuah situs web* saat Anda bekerja, Anda perlu menjalankan Astro dalam mode pengembangan (dev).

### Mulai server dev

<Steps>
10. Jalankan perintah untuk memulai server dev Astro dengan mengetik di terminal VS Code:

    <PackageManagerTabs>
      <Fragment slot="npm">
      ```shell
      npm run dev
      ```
      </Fragment>
      <Fragment slot="pnpm">
      ```shell
      pnpm run dev
      ```
      </Fragment>
      <Fragment slot="yarn">
      ```shell
      yarn run dev
      ```
      </Fragment>
    </PackageManagerTabs>

    Kini Anda seharusnya melihat konfirmasi di terminal bahwa Astro berjalan dalam mode dev. ðŸš€
</Steps>

## Lihat pratinjau situs Anda

Berkas proyek Anda berisi semua kode yang diperlukan untuk menampilkan situs Astro, tetapi perambanlah yang bertanggung jawab menampilkannya sebagai halaman web.

<Steps>
11. Klik tautan `localhost` di jendela terminal untuk melihat pratinjau langsung situs Astro baru Anda! 

    (Astro menggunakan `http://localhost:4321` secara bawaan jika port `4321` tersedia.)

    Berikut tampilan *starter* "Empty Project" Astro di pratinjau peramban:

    ![Halaman putih kosong dengan tulisan Astro di bagian atas.](/tutorial/minimal.png)

</Steps>

:::tip[Menggunakan server dev Astro]

Saat server Astro berjalan dalam mode dev, Anda TIDAK akan dapat menjalankan perintah lain di jendela terminal tersebut. Panel ini akan memberi umpan balik saat Anda mempratinjau situs.

Anda dapat menghentikan server dev kapan saja dan kembali ke prompt perintah dengan menekan <kbd>Ctrl + C</kbd> di terminal.

Terkadang server dev akan berhenti sendiri saat Anda bekerja. Jika pratinjau langsung berhenti berfungsi, kembali ke terminal dan mulai ulang server dev dengan mengetik `npm run dev`.
:::
