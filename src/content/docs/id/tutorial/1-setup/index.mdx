Sekarang setelah Anda tahu apa yang akan dibangun, saatnya menyiapkan semua alat yang Anda perlukan! 

Unit ini menunjukkan cara menyiapkan lingkungan pengembangan dan *deploy* ke Netlify. Loncat ke [Unit 2](/id/tutorial/2-pages/) jika Anda sudah nyaman dengan lingkungan dan *workflow* Anda.

:::tip[Ambil tutorial di editor kode daring]

Ingin menyelesaikan tutorial ini di editor kode daring? Ikuti instruksi di bawah untuk memulai di Google IDX.

<details>
<summary>Menggunakan Google IDX: Ikuti instruksi ini, lalu langsung ke Unit 2!</summary>

**Siapkan IDX**

<Steps>
1. Ikuti tautan eksternal untuk [membuka template "Empty Project" di *workspace* baru pada IDX](https://astro.new/minimal?on=idx).

2. Ikuti prompt untuk masuk ke akun Google Anda jika belum masuk.

3. Masukkan nama proyek jika ingin mengubah dari bawaan "Empty Project". Klik **Create**.

4. Tunggu *workspace* dibuat. Ini mungkin memakan waktu 30–60 detik. Jika berhasil, Anda akan melihat proyek Astro dimuat di editor kode daring.

5. Tunggu IDX menjalankan dua skrip: satu untuk memasang Astro dan satu lagi untuk memulai *development server*. Anda mungkin sebentar melihat pesan bahwa *workspace* "tidak menemukan Astro" jika *workspace* terbuka sebelum pemasangan selesai. Pesan ini bisa diabaikan dan dibatalkan jika tidak hilang sendiri. 
</Steps>

**Buat Perubahan**

Jika semuanya berjalan baik, Anda akan melihat kode untuk berkas `src/pages/index.astro` dibuka berdampingan dengan pratinjau situs. Ikuti instruksi ["Tulis baris Astro pertamamu"](/id/tutorial/1-setup/3/) untuk mengubah berkas ini.

**Buat Repositori GitHub**

<Steps>
1. Arahkan ke item navigasi "Source Control" di bilah menu vertikal, atau buka dengan <kbd>CTRL + SHIFT + G</kbd>. 

2. Pilih opsi untuk *Publish to GitHub*. Ini akan membuat repositori baru di akun GitHub Anda.
3. Ikuti prompt untuk masuk ke akun GitHub Anda.
4. Setelah masuk, kembali ke tab IDX dan Anda bisa memilih nama repo baru, serta apakah ingin privat atau publik. Pilih nama apa pun dan jenis repositori apa pun untuk tutorial ini.
5. IDX akan membuat *initial commit* dan memublikasikannya ke repo GitHub baru Anda.
6. Ke depan, setiap kali ada perubahan yang perlu di-*commit* kembali ke GitHub, ikon navigasi Source Control akan menampilkan angka. Ini adalah jumlah berkas yang berubah sejak *commit* terakhir. Buka tab ini dan lakukan dua langkah (commit dan publish) untuk memasukkan pesan *commit* dan memperbarui repositori Anda.
</Steps>

**Deploy Situs Anda**

Jika Anda ingin *deploy* ke Netlify, dan memiliki versi situs yang terpublikasi saat bekerja, lanjutkan di Unit 1 ke [*Deploy* situs Anda ke web](/id/tutorial/1-setup/5/).

Jika tidak, lanjut ke [Unit 2](/id/tutorial/2-pages/) untuk mulai membangun dengan Astro!
</details>

## Ke mana tujuan Anda?

Di unit ini, Anda akan **membuat proyek baru** yang **disimpan online di GitHub** dan **terhubung ke Netlify**. 

Saat Anda menulis kode, Anda akan secara berkala *commit* perubahan ke GitHub. Netlify akan menggunakan berkas di repositori GitHub Anda untuk membangun situs web, lalu menerbitkannya di internet pada alamat unik yang dapat dilihat siapa saja.

Setiap kali Anda *commit* perubahan ke GitHub, notifikasi akan dikirim ke Netlify. Kemudian, Netlify akan secara otomatis membangun ulang dan menerbitkan ulang situs langsung Anda agar mencerminkan perubahan tersebut.
