Tutorial ini akan menggunakan GitHub untuk menyimpan repositori dan terhubung ke *web host*. Anda dipersilakan menggunakan penyedia git daring lain pilihan Anda.

:::note
Jika Anda sudah familier dengan git dan punya *workflow* sendiri, buat repo GitHub baru untuk proyek Anda dengan metode yang Anda sukai. Loncat ke halaman berikutnya: ["Deploy situs Anda ke web"](/id/tutorial/1-setup/5/).
:::

## Buat repositori di GitHub

Meskipun ada beberapa cara untuk menyimpan kode lokal Anda di GitHub, tutorial ini akan memandu Anda melalui metode yang tidak memerlukan penggunaan git di command line.

<Steps>
1. Masuk ke GitHub.com di peramban lalu klik <kbd>+</kbd> di kanan atas layar untuk membuat repositori baru.

2. Pilih nama untuk repositori Anda. Ini tidak harus sama dengan nama folder proyek Anda.

3. Anda akan melihat beberapa opsi, tetapi Anda tidak perlu mengubah nilai bawaan. Gulir ke bawah dan klik tombol <kbd>Create Repository</kbd>.

4. Anda akan ditampilkan berbagai langkah lanjutan, tetapi Anda tidak perlu menggunakannya. Catat URL repositori Anda. Anda dapat keluar dari halaman ini tanpa melakukan apa pun.
</Steps>

## Commit kode lokal Anda ke GitHub

Pada bagian sebelumnya, Anda membuat perubahan pada konten halaman. Ini berarti berkas proyek Anda telah berubah, dan VS Code akan menampilkan sebuah angka di ikon menu "Source". Tab *source* inilah tempat Anda akan rutin memperbarui berkas ke GitHub.

<Steps>
1. Klik tab **Source Control** di VS Code untuk melihat daftar berkas yang berubah. Jika Anda melihat pesan perlu memasang `git`, ikuti instruksinya, lalu muat ulang VS Code.

2. Klik menu <kbd>•••</kbd> (tiga titik) di atas kolom pesan commit dan pilih <kbd>Remote</kbd> > <kbd>Add Remote</kbd>.

3. Pilih <kbd>Add remote from GitHub</kbd>. Jika perlu, ikuti langkah autentikasi, lalu kembali ke VS Code dan ulangi tindakan ini.

4. Anda akan melihat daftar semua repositori GitHub Anda. Pilih repositori yang tadi Anda buat untuk proyek ini. Jika tidak terlihat, tempelkan URL GitHub repo tersebut langsung. Anda mungkin diminta memberi nama lokal untuk repositori ini. Pilih nama apa pun yang Anda suka.

5. Di bagian atas panel, akan ada tempat untuk memasukkan **commit message** (deskripsi perubahan berkas). Ketik `initial commit` dan tekan tombol <kbd>Commit</kbd> untuk melakukan commit.

6. Anda mungkin melihat pesan bahwa belum ada yang di-*stage*, dan ditanya apakah ingin *stage*. Klik <kbd>Always</kbd> dan lanjutkan.

7. Terakhir, daftar berkas yang berubah akan digantikan tombol <kbd>Publish</kbd>. Klik untuk mengirim perubahan yang sudah di-commit ke GitHub.
</Steps>

### Lihat proyek Anda di GitHub

Untuk memastikan proyek Anda berhasil tersimpan di GitHub, buka GitHub.com dan lihat daftar repositori pada akun Anda. Pilih yang baru Anda buat, lalu pastikan berisi berkas-berkas proyek Astro Anda.
