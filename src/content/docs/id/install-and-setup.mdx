Perintah CLI [`create astro`](#install-from-the-cli-wizard) adalah cara tercepat untuk memulai proyek Astro baru dari nol. Wizard ini akan menuntun Anda melalui setiap langkah penyiapan proyek Astro baru dan memungkinkan Anda memilih beberapa template starter resmi yang berbeda. 

Anda juga bisa menjalankan perintah CLI tersebut dengan flag `template` untuk memulai proyek menggunakan tema atau template starter yang sudah ada. Jelajahi [etalase tema dan starter](https://astro.build/themes/) kami untuk menelusuri tema untuk blog, portofolio, situs dokumentasi, landing page, dan lainnya!

Untuk memasang Astro secara manual, lihat [panduan instalasi manual langkah demi langkah](#manual-setup).

:::tip[Pratinjau online]
Lebih suka mencoba Astro di browser? Kunjungi [astro.new](https://astro.new/) untuk menelusuri template starter dan memutar proyek Astro baru tanpa meninggalkan browser.
:::

## Prasyarat

- **Node.js** - `v18.20.8` atau `v20.3.0`, `v22.0.0` atau lebih tinggi. (`v19` dan `v21` tidak didukung.)
- **Editor teks** - Kami merekomendasikan [VS Code](https://code.visualstudio.com/) dengan [Ekstensi Astro Resmi](https://marketplace.visualstudio.com/items?itemName=astro-build.astro-vscode).
- **Terminal** - Astro diakses melalui antarmuka baris perintah (CLI).

## Kompatibilitas browser

Astro dibangun dengan Vite yang secara bawaan menargetkan browser dengan dukungan JavaScript modern. Untuk referensi lengkap, Anda dapat melihat [daftar versi browser yang saat ini didukung di Vite](https://vite.dev/guide/build.html#browser-compatibility).

## Instal dari wizard CLI

Anda dapat menjalankan `create astro` di mana saja di komputer Anda, jadi tidak perlu membuat direktori kosong baru sebelum mulai. Jika Anda belum memiliki direktori kosong untuk proyek baru, wizard akan membantu membuatnya secara otomatis.

<Steps>
1. Jalankan perintah berikut di terminal untuk memulai wizard instalasi:

    <PackageManagerTabs>
      <Fragment slot="npm">
      ```shell
      # buat proyek baru dengan npm
      npm create astro@latest
      ```
      </Fragment>
      <Fragment slot="pnpm">
      ```shell
      # buat proyek baru dengan pnpm
      pnpm create astro@latest
      ```
      </Fragment>
      <Fragment slot="yarn">
      ```shell
      # buat proyek baru dengan yarn
      yarn create astro
      ```
      </Fragment>
    </PackageManagerTabs>

    Jika semuanya berjalan baik, Anda akan melihat pesan sukses diikuti langkah-langkah berikutnya yang direkomendasikan.
    
2. Sekarang proyek Anda telah dibuat, Anda bisa `cd` ke direktori proyek baru dan mulai menggunakan Astro.

3. Jika Anda melewatkan langkah "Install dependencies?" saat wizard CLI, pastikan untuk memasang dependency sebelum melanjutkan.

    <PackageManagerTabs>
      <Fragment slot="npm">
      ```shell
      npm install
      ```
      </Fragment>
      <Fragment slot="pnpm">
      ```shell
      pnpm install
      ```
      </Fragment>
      <Fragment slot="yarn">
      ```shell
      yarn install
      ```
      </Fragment>
    </PackageManagerTabs>

4. Sekarang Anda dapat [menjalankan server dev Astro](/id/develop-and-build/#start-the-astro-dev-server) dan melihat pratinjau langsung proyek saat Anda membangun!
</Steps>

## Flag instalasi CLI

Anda dapat menjalankan perintah `create astro` dengan flag tambahan untuk menyesuaikan proses (mis. menjawab "yes" untuk semua pertanyaan, melewatkan animasi Houston) atau menyesuaikan proyek baru (mis. pasang git atau tidak, tambah integrasi).

<ReadMore>Lihat [semua flag `create astro` yang tersedia](https://github.com/withastro/astro/blob/main/packages/create-astro/README.md).</ReadMore>

### Tambahkan integrasi

Anda dapat memulai proyek Astro baru dan memasang [integrasi resmi](/id/guides/integrations-guide/) apa pun atau integrasi komunitas yang mendukung perintah `astro add` pada saat yang sama dengan memberikan argumen `--add` pada perintah `create astro`.

Jalankan perintah berikut di terminal, ganti integrasi dengan yang mendukung perintah `astro add`:

<PackageManagerTabs>
  <Fragment slot="npm">
  ```shell
  # buat proyek baru dengan React dan Partytown
  npm create astro@latest -- --add react --add partytown
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```shell
  # buat proyek baru dengan React dan Partytown
  pnpm create astro@latest --add react --add partytown
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```shell
  # buat proyek baru dengan React dan Partytown
  yarn create astro --add react --add partytown
  ```
  </Fragment>
</PackageManagerTabs>

### Gunakan tema atau template starter

Anda dapat memulai proyek Astro berdasarkan [contoh resmi](https://github.com/withastro/astro/tree/main/examples) atau branch `main` dari repositori GitHub apa pun dengan menambahkan argumen `--template` pada perintah `create astro`.

Jalankan perintah berikut di terminal, ganti dengan nama template starter resmi Astro, atau username dan repositori GitHub dari tema yang ingin Anda gunakan:

<PackageManagerTabs>
  <Fragment slot="npm">
  ```shell
  # buat proyek baru dengan contoh resmi
  npm create astro@latest -- --template <example-name>

  # buat proyek baru berdasarkan branch main repositori GitHub
  npm create astro@latest -- --template <github-username>/<github-repo>
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```shell
  # buat proyek baru dengan contoh resmi
  pnpm create astro@latest --template <example-name>
  
  # buat proyek baru berdasarkan branch main repositori GitHub
  pnpm create astro@latest --template <github-username>/<github-repo>
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```shell
  # buat proyek baru dengan contoh resmi
  yarn create astro --template <example-name>
  
  # buat proyek baru berdasarkan branch main repositori GitHub
  yarn create astro --template <github-username>/<github-repo>
  ```
  </Fragment>
</PackageManagerTabs>

Secara bawaan, perintah ini akan menggunakan branch `main` dari repositori template. Untuk menggunakan nama branch lain, sertakan sebagai bagian dari argumen `--template`: `<github-username>/<github-repo>#<branch>`.

## Penyiapan manual

Panduan ini akan menuntun Anda melalui langkah-langkah untuk memasang dan mengonfigurasi proyek Astro baru secara manual.

Jika Anda tidak ingin menggunakan alat otomatis `create astro`, Anda dapat menyiapkan proyek sendiri dengan mengikuti panduan di bawah ini.

<Steps>
1. Buat direktori Anda

    Buat direktori kosong dengan nama proyek Anda, lalu navigasikan ke dalamnya.

    ```bash
    mkdir my-astro-project
    cd my-astro-project
    ```

    Setelah berada di direktori baru, buat `package.json` untuk proyek Anda. Berkas ini adalah cara Anda mengelola dependency proyek, termasuk Astro. Jika Anda belum familiar dengan format berkas ini, jalankan perintah berikut untuk membuatnya.

    <PackageManagerTabs>
      <Fragment slot="npm">
      ```shell
      npm init --yes
      ```
      </Fragment>
      <Fragment slot="pnpm">
      ```shell
      pnpm init 
      ```
      </Fragment>
      <Fragment slot="yarn">
      ```shell
      yarn init --yes
      ```
      </Fragment>
    </PackageManagerTabs>

2. Pasang Astro

    Pertama, pasang dependency proyek Astro di dalam proyek Anda.

    :::note[Penting]
    Astro harus dipasang secara lokal, bukan global. Pastikan Anda *tidak* menjalankan `npm install -g astro` `pnpm add -g astro` atau `yarn add global astro`.
    :::

    <PackageManagerTabs>
      <Fragment slot="npm">
      ```shell
      npm install astro
      ```
      </Fragment>
      <Fragment slot="pnpm">
      ```shell
      pnpm add astro
      ```
      </Fragment>
      <Fragment slot="yarn">
      ```shell
      yarn add astro
      ```
      </Fragment>
    </PackageManagerTabs>

    Lalu, ganti bagian "scripts" placeholder pada `package.json` Anda dengan berikut:

    ```json title="package.json" del={3} ins={4-6}
    {
      "scripts": {
        "test": "echo \"Error: no test specified\" && exit 1",
        "dev": "astro dev",
        "build": "astro build",
        "preview": "astro preview"
      },
    }
    ```

    Anda akan menggunakan skrip-skrip ini nanti untuk menjalankan Astro dan perintah-perintahnya.

3. Buat halaman pertama Anda

    Di editor teks Anda, buat berkas baru di `src/pages/index.astro`. Ini akan menjadi halaman Astro pertama proyek Anda.

    Untuk panduan ini, salin dan tempel cuplikan kode berikut (termasuk garis `---`) ke berkas baru Anda:

    ```astro title="src/pages/index.astro"
    ---
    // Selamat datang di Astro! Segala sesuatu di antara pagar kode tiga garis ini
    // adalah "frontmatter komponen" Anda. Bagian ini tidak berjalan di browser.
    console.log('Kode ini berjalan di terminal Anda, bukan di browser!');
    ---
    <!-- Di bawah ini adalah "templat komponen" Anda. Ini hanya HTML, tetapi
        dengan sedikit sihir untuk membantu Anda membuat templat yang hebat. -->
    <html>
      <body>
        <h1>Hello, World!</h1>
      </body>
    </html>
    <style>
      h1 {
        color: orange;
      }
    </style>
    ```

4. Buat aset statis pertama Anda

    Anda juga perlu membuat direktori `public/` untuk menyimpan aset statis. Astro akan selalu menyertakan aset ini dalam build final, sehingga Anda dapat merujuknya dengan aman dari dalam templat komponen.

    Di editor teks Anda, buat berkas baru `public/robots.txt`. `robots.txt` adalah berkas sederhana yang biasanya disertakan sebagian besar situs untuk memberi tahu bot pencarian seperti Google cara memperlakukan situs Anda.

    Untuk panduan ini, salin dan tempel cuplikan kode berikut ke berkas baru Anda:

    ```diff title="public/robots.txt"
    # Contoh: Izinkan semua bot memindai dan mengindeks situs Anda.
    # Sintaks lengkap: https://developers.google.com/search/docs/advanced/robots/create-robots-txt
    User-agent: *
    Allow: /
    ```

5. Buat `astro.config.mjs`

    Astro dikonfigurasi menggunakan `astro.config.mjs`. Berkas ini opsional jika Anda tidak perlu mengonfigurasi Astro, tetapi Anda mungkin ingin membuatnya sekarang.

    Buat `astro.config.mjs` di root proyek, dan salin kode berikut:

    ```js title="astro.config.mjs"
    import { defineConfig } from "astro/config";

    // https://astro.build/config
    export default defineConfig({});
    ```

    Jika Anda ingin menyertakan [komponen kerangka UI](/id/guides/framework-components/) seperti React, Svelte, dll. atau menggunakan alat lain seperti MDX atau Partytown di proyek Anda, di sinilah Anda akan [mengimpor dan mengonfigurasi integrasi secara manual](/id/guides/integrations-guide/).

    <ReadMore>Baca [referensi konfigurasi API](/id/reference/configuration-reference/) Astro untuk informasi lebih lanjut.</ReadMore>

6. Tambahkan dukungan TypeScript

    TypeScript dikonfigurasi menggunakan `tsconfig.json`. Bahkan jika Anda tidak menulis TypeScript, berkas ini penting agar alat seperti Astro dan VS Code memahami proyek Anda. Beberapa fitur (seperti impor paket npm) tidak sepenuhnya didukung di editor tanpa `tsconfig.json`.

    Jika Anda berniat menulis TypeScript, disarankan menggunakan template `strict` atau `strictest` milik Astro. Anda dapat melihat dan membandingkan tiga konfigurasi template di [astro/tsconfigs/](https://github.com/withastro/astro/blob/main/packages/astro/tsconfigs/).

    Buat `tsconfig.json` di root proyek, dan salin kode berikut. (Anda dapat menggunakan `base`, `strict`, atau `strictest` untuk template TypeScript Anda):

    ```json title="tsconfig.json" "base"
    {
      "extends": "astro/tsconfigs/base"
    }
    ```

    <ReadMore>Baca [panduan penyiapan TypeScript](/id/guides/typescript/#setup) Astro untuk informasi lebih lanjut.</ReadMore>

7. Langkah selanjutnya

    Jika Anda telah mengikuti langkah di atas, direktori proyek Anda sekarang akan tampak seperti ini:

    <FileTree>
    - node_modules/
    - public/
      - robots.txt
    - src/
      - pages/
        - index.astro
    - astro.config.mjs
    - package-lock.json atau `yarn.lock`, `pnpm-lock.yaml`, dll.
    - package.json
    - tsconfig.json
    </FileTree>

8. Sekarang Anda dapat [menjalankan server dev Astro](/id/develop-and-build/#start-the-astro-dev-server) dan melihat pratinjau langsung proyek saat Anda membangun!

</Steps>
