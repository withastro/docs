**Astro** adalah kerangka kerja web untuk membangun **situs berorientasi konten** seperti blog, pemasaran, dan e-commerce. Astro dikenal karena memelopori [arsitektur frontend](/id/concepts/islands/) baru yang mengurangi overhead dan kompleksitas JavaScript dibanding kerangka lain. Jika Anda membutuhkan situs yang memuat cepat dan memiliki SEO hebat, Astro adalah jawabannya.

## Fitur

**Astro adalah kerangka web serba ada.** Ia menyertakan semua yang Anda perlukan untuk membuat situs, langsung built-in. Tersedia pula ratusan [integrasi](https://astro.build/integrations/) dan [API hooks](/id/reference/integrations-reference/) untuk menyesuaikan proyek sesuai kebutuhan Anda.

Beberapa sorotan di antaranya:

- **[Islands](/id/concepts/islands/):** Arsitektur web berbasis komponen yang dioptimalkan untuk situs berorientasi konten.
- **[Agnostik UI](/id/guides/framework-components/):** Mendukung React, Preact, Svelte, Vue, Solid, HTMX, web components, dan lainnya.
- **[Server-first](/id/guides/on-demand-rendering/):** Memindahkan proses render yang mahal dari perangkat pengunjung Anda.
- **[Zero JS secara bawaan](/id/basics/astro-components/):** Lebih sedikit JavaScript sisi klien yang memperlambat situs Anda.
- **[Content collections](/id/guides/content-collections/):** Mengorganisasi, memvalidasi, dan menyediakan type-safety TypeScript untuk konten Markdown Anda.
- **[Dapat dikustomisasi](/id/guides/integrations-guide/):** Partytown, MDX, dan ratusan integrasi lain untuk dipilih.

## Prinsip Desain

Berikut lima prinsip desain inti untuk menjelaskan mengapa kami membangun Astro, masalah apa yang ingin diselesaikan, dan mengapa Astro mungkin pilihan terbaik untuk proyek atau tim Anda.

Astro itu…

1. **[Berorientasi konten](#content-driven):** Astro dirancang untuk menonjolkan konten Anda.
2. **[Server-first](#server-first):** Situs berjalan lebih cepat ketika merender HTML di server.
3. **[Cepat secara bawaan](#fast-by-default):** Hampir mustahil membuat situs lambat di Astro.
4. **[Mudah digunakan](#easy-to-use):** Anda tidak perlu menjadi ahli untuk membangun sesuatu dengan Astro.
5. **[Fokus pada developer](#developer-focused):** Anda memiliki sumber daya yang diperlukan untuk sukses.

### Content-driven

**Astro dirancang untuk membangun situs kaya konten.** Ini mencakup situs pemasaran, penerbitan, dokumentasi, blog, portofolio, halaman arahan, komunitas, dan e-commerce. Jika Anda memiliki konten untuk ditampilkan, konten itu harus cepat sampai ke pembaca.

Sebaliknya, kebanyakan kerangka modern dirancang untuk membangun aplikasi web. Kerangka-kerangka ini unggul membangun pengalaman seperti aplikasi yang lebih kompleks di browser: dashboard admin, inbox, jejaring sosial, to-do list, dan bahkan aplikasi mirip native seperti [Figma](https://figma.com/) dan [Ping](https://ping.gg/). Namun dengan kompleksitas tersebut, mereka dapat kesulitan memberikan performa hebat saat menyajikan konten Anda.

Fokus Astro pada konten sejak awal sebagai static site builder memungkinkan Astro **bertumbuh secara masuk akal menjadi aplikasi web dinamis yang bertenaga namun tetap performant**, tetap menghormati konten dan audiens Anda. Fokus unik ini memungkinkan Astro membuat tradeoff dan menghadirkan fitur performa yang tidak masuk akal untuk diterapkan pada kerangka yang berfokus aplikasi.

### Server-first

**Astro memaksimalkan rendering di server dibanding rendering di browser.** Ini adalah pendekatan yang sama dengan kerangka server-side tradisional — PHP, WordPress, Laravel, Ruby on Rails, dll. — selama puluhan tahun. Namun Anda tidak perlu mempelajari bahasa sisi server kedua untuk menikmatinya. Dengan Astro, semuanya tetap hanya HTML, CSS, dan JavaScript (atau TypeScript jika Anda suka).

Pendekatan ini berbeda dengan banyak kerangka web JavaScript modern seperti Next.js, SvelteKit, Nuxt, Remix, dan lainnya. Kerangka-kerangka tersebut dibangun untuk client-side rendering seluruh situs dan menyertakan server-side rendering terutama untuk mengatasi masalah performa. Pendekatan ini dijuluki **Single-Page App (SPA)**, berlawanan dengan pendekatan **Multi-Page App (MPA)** milik Astro.

Model SPA memiliki manfaat. Namun, manfaat tersebut datang dengan biaya kompleksitas dan kompromi performa. Kompromi ini merugikan metrik penting — seperti [Time to Interactive (TTI)](https://web.dev/interactive/) — yang kurang masuk akal untuk situs berfokus konten di mana performa first-load sangat krusial.

Pendekatan server-first Astro memungkinkan Anda opt in ke rendering sisi klien hanya jika, dan tepat sejauh yang diperlukan. Anda bisa memilih menambahkan komponen kerangka UI yang berjalan di klien. Anda dapat memanfaatkan view transitions router Astro untuk kontrol lebih halus atas transisi dan animasi halaman tertentu. Rendering server-first Astro, baik pre-rendered maupun on-demand, menyediakan default yang performant yang dapat Anda tingkatkan.

### Fast by default

Performa yang baik selalu penting, namun sangat krusial untuk situs yang kesuksesannya bergantung pada tampilan konten Anda. Sudah terbukti bahwa performa buruk menurunkan engagement, konversi, dan pendapatan. Misalnya:

- Setiap 100ms lebih cepat → 1% konversi lebih banyak ([Mobify](https://web.dev/why-speed-matters/), +$380.000/tahun)
- 50% lebih cepat → 12% penjualan lebih banyak ([AutoAnything](https://www.digitalcommerce360.com/2010/08/19/web-accelerator-revs-conversion-and-sales-autoanything/))
- 20% lebih cepat → 10% konversi lebih banyak ([Furniture Village](https://www.thinkwithgoogle.com/intl/en-gb/marketing-strategies/app-and-mobile/furniture-village-and-greenlight-slash-page-load-times-boosting-user-experience/))
- 40% lebih cepat → 15% pendaftaran lebih banyak ([Pinterest](https://medium.com/pinterest-engineering/driving-user-growth-with-performance-improvements-cfc50dafadd7))
- 850ms lebih cepat → 7% konversi lebih banyak ([COOK](https://web.dev/why-speed-matters/))
- Setiap 1 detik lebih lambat → 10% pengguna lebih sedikit ([BBC](https://www.creativebloq.com/features/how-the-bbc-builds-websites-that-scale))

Pada banyak kerangka web, mudah untuk membangun situs yang terlihat hebat saat pengembangan namun terasa sangat lambat ketika dideploy. JavaScript sering menjadi biang masalah, karena banyak ponsel dan perangkat berdaya rendah jarang menyamai kecepatan laptop developer.

Keajaiban Astro terletak pada bagaimana ia menggabungkan dua nilai di atas — fokus konten dan arsitektur server-first — untuk membuat pertukaran dan menghadirkan fitur yang tidak bisa dilakukan kerangka lain. Hasilnya adalah performa web yang luar biasa untuk setiap situs, out of the box. Tujuan kami: **Hampir mustahil membangun situs yang lambat dengan Astro.** 

Situs Astro dapat [memuat 40% lebih cepat dengan 90% lebih sedikit JavaScript](https://twitter.com/t3dotgg/status/1437195415439360003) dibanding situs yang sama dibangun dengan kerangka React paling populer. Namun jangan hanya percaya kata kami: lihat performa Astro membuat Ryan Carniato (pencipta Solid.js dan Marko) [terpukau](https://youtu.be/2ZEMb_H-LYE?t=8163).

### Easy to use

**Tujuan Astro adalah dapat diakses oleh setiap pengembang web.** Astro dirancang terasa familiar dan mudah didekati terlepas dari tingkat kemampuan atau pengalaman pengembangan web sebelumnya. 

Bahasa UI `.astro` adalah superset HTML: HTML valid apa pun adalah sintaks templating Astro yang valid! Jadi, jika Anda bisa menulis HTML, Anda bisa menulis komponen Astro! Namun, `.astro` juga menggabungkan beberapa fitur favorit kami yang dipinjam dari bahasa komponen lain seperti ekspresi JSX (React) dan CSS scoping bawaan (Svelte dan Vue). Kedekatan ini dengan HTML juga memudahkan penggunaan progressive enhancement dan pola aksesibilitas umum tanpa overhead.

Kami juga memastikan Anda dapat menggunakan bahasa komponen UI favorit yang sudah Anda kenal, bahkan menggunakan ulang komponen yang mungkin sudah Anda miliki. React, Preact, Svelte, Vue, Solid, dan lainnya, termasuk web components, semuanya didukung untuk menulis komponen UI di proyek Astro.

Astro dirancang lebih sederhana daripada kerangka dan bahasa UI lainnya. Salah satu alasannya adalah Astro dirancang untuk merender di server, bukan di browser. Artinya Anda tidak perlu memikirkan: hooks (React), stale closures (juga React), refs (Vue), observables (Svelte), atoms, selectors, reactions, atau derivations. Tidak ada reaktivitas di server, sehingga semua kompleksitas itu hilang.

Salah satu slogan favorit kami adalah: **opt-in pada kompleksitas.** Kami mendesain Astro untuk menghapus sebanyak mungkin "kompleksitas wajib" dari pengalaman developer, terutama saat onboarding pertama kali. Anda bisa membangun situs contoh "Hello World" di Astro hanya dengan HTML dan CSS. Lalu, saat butuh membangun sesuatu yang lebih kuat, Anda bisa secara bertahap meraih fitur dan API baru.

### Developer-focused

Kami sangat percaya bahwa Astro hanya akan sukses jika orang senang menggunakannya. Astro memiliki semua yang Anda butuhkan untuk mendukung saat membangun dengan Astro.

Astro berinvestasi pada alat pengembang seperti pengalaman CLI yang hebat sejak Anda membuka terminal, ekstensi resmi VS Code untuk syntax highlighting, TypeScript dan Intellisense, serta dokumentasi yang aktif dipelihara oleh ratusan kontributor komunitas dan tersedia dalam 14 bahasa.

Komunitas kami di Discord yang ramah, penuh rasa hormat, dan inklusif siap memberikan dukungan, motivasi, dan dorongan. Buka thread `#support` untuk mendapatkan bantuan pada proyek Anda. Kunjungi kanal `#showcase` khusus untuk berbagi situs Astro Anda, posting blog, video, bahkan work-in-progress untuk umpan balik aman dan kritik yang membangun. Ikuti acara langsung rutin seperti panggilan komunitas mingguan kami, "Talking and Doc'ing," serta API/bug bashes.

Sebagai proyek open-source, kami menyambut kontribusi dari segala jenis dan ukuran dari anggota komunitas di seluruh tingkat pengalaman. Anda diundang untuk bergabung dalam diskusi roadmap untuk membentuk masa depan Astro, dan kami berharap Anda akan berkontribusi perbaikan dan fitur ke core codebase, compiler, docs, language tools, situs web, dan proyek lainnya.
