Anda dapat menggunakan Command-Line Interface (CLI) yang disediakan Astro untuk mengembangkan, membangun, dan mempratinjau proyek Anda dari jendela terminal.

### Perintah `astro`

Gunakan CLI dengan menjalankan salah satu **perintah** yang didokumentasikan pada halaman ini menggunakan manajer paket pilihan Anda, opsional diikuti oleh **flag**. Flag menyesuaikan perilaku sebuah perintah.

Salah satu perintah yang paling sering Anda gunakan adalah `astro dev`. Perintah ini memulai server pengembangan dan memberi Anda pratinjau langsung yang terus diperbarui dari situs di browser saat Anda bekerja:

### Skrip `package.json`

Anda juga dapat menggunakan skrip di `package.json` untuk versi perintah yang lebih singkat. Menggunakan skrip memungkinkan Anda memakai perintah yang sama seperti pada proyek lain, seperti `npm run build`.

Skrip berikut untuk perintah `astro` yang paling umum (`astro dev`, `astro build`, dan `astro preview`) akan ditambahkan secara otomatis ketika Anda membuat proyek menggunakan [wizard `create astro`](/id/install-and-setup/).

Saat Anda mengikuti instruksi untuk [menginstal Astro secara manual](/id/install-and-setup/#manual-setup), Anda diminta menambahkan skrip ini sendiri. Anda juga dapat menambahkan lebih banyak skrip secara manual untuk perintah yang sering Anda gunakan.

```json title="package.json"
{
  "scripts": {
    "dev": "astro dev",
    "build": "astro build",
    "preview": "astro preview"
  }
}
```

Anda akan sering menggunakan perintah `astro`, atau skrip yang menjalankannya, tanpa flag apa pun. Tambahkan flag ke perintah saat Anda ingin menyesuaikan perilakunya. Misalnya, Anda mungkin ingin menjalankan server pengembangan pada port yang berbeda, atau membangun situs dengan log verbose untuk debugging.

## `astro dev`

Menjalankan server pengembangan Astro. Ini adalah server HTTP lokal yang tidak melakukan bundling aset. Server menggunakan Hot Module Replacement (HMR) untuk memperbarui browser saat Anda menyimpan perubahan di editor.

Hotkey berikut dapat digunakan di terminal tempat server pengembangan Astro berjalan:

- `s + enter` untuk menyinkronkan data content layer (konten dan tipe).
- `o + enter` untuk membuka situs Astro Anda di browser.
- `q + enter` untuk keluar dari server pengembangan.

## `astro build`

Membangun situs Anda untuk deployment. Secara default, ini akan menghasilkan file statis dan menaruhnya di direktori `dist/`. Jika ada rute yang [dirender saat diminta](/id/guides/on-demand-rendering/), ini akan menghasilkan file server yang diperlukan untuk melayani situs Anda.

### Flags

Perintah ini menerima [flag umum](#common-flags) dan flag tambahan berikut:

#### `--devOutput`

<p><Since v="5.0.0" /></p>

Menghasilkan build berbasis pengembangan yang mirip dengan kode yang ditransformasikan di `astro dev`. Ini berguna untuk menguji masalah yang hanya muncul saat build dengan informasi debugging tambahan yang disertakan.

## `astro preview`

Memulai server lokal untuk melayani isi dari direktori statis Anda (`dist/` secara default) yang dibuat dengan menjalankan `astro build`.

Perintah ini memungkinkan Anda mempratinjau situs secara lokal [setelah membangun](#astro-build) untuk menangkap kesalahan pada output build sebelum melakukan deploy. Perintah ini tidak dirancang untuk dijalankan di produksi. Untuk bantuan hosting produksi, lihat panduan kami tentang [Mendeploy Situs Astro](/id/guides/deploy/).

Sejak Astro 1.5.0, [adapter Node](/id/guides/integrations-guide/node/) mendukung `astro preview` untuk build yang dibuat dengan rendering on-demand.

Dapat digabungkan dengan [flag umum](#common-flags) yang didokumentasikan di bawah.

## `astro check`

Menjalankan diagnostik (seperti pemeriksaan tipe di dalam file `.astro`) terhadap proyek Anda dan menampilkan error ke konsol. Jika ada error yang ditemukan, proses akan keluar dengan kode **1**.

Perintah ini dimaksudkan untuk digunakan dalam workflow CI.

<h3>Flags</h3>

Gunakan flag berikut untuk menyesuaikan perilaku perintah.

#### `--watch`

Perintah akan memantau setiap perubahan di proyek Anda, dan akan melaporkan error apa pun.

#### `--root <path-to-dir>`

Menentukan direktori root yang berbeda untuk diperiksa. Secara default menggunakan direktori kerja saat ini.

#### `--tsconfig <path-to-file>`

Menentukan file `tsconfig.json` yang digunakan secara manual. Jika tidak diberikan, Astro akan mencoba mencari konfigurasi, atau menurunkannya secara otomatis dari proyek.

#### `--minimumFailingSeverity <error|warning|hint>`

Menentukan tingkat keparahan minimum yang diperlukan agar keluar dengan kode error. Default: `error`.

Contoh, menjalankan `astro check --minimumFailingSeverity warning` akan membuat perintah keluar dengan error jika ada peringatan yang terdeteksi.

#### `--minimumSeverity <error|warning|hint>`

Menentukan tingkat keparahan minimum yang akan ditampilkan. Default: `hint`.

Contoh, menjalankan `astro check --minimumSeverity warning` akan menampilkan error dan peringatan, tetapi tidak menampilkan hint.

#### `--preserveWatchOutput`

Menentukan agar tidak membersihkan output antar-pemeriksaan saat dalam mode watch.

#### `--noSync`

Menentukan agar tidak menjalankan `astro sync` sebelum memeriksa proyek.

<ReadMore>Baca selengkapnya tentang [pemeriksaan tipe di Astro](/id/guides/typescript/#type-checking).</ReadMore>

## `astro sync`

<p><Since v="2.0.0" /></p>

:::tip
Menjalankan `astro dev`, `astro build` atau `astro check` juga akan menjalankan perintah `sync`.
:::

Menghasilkan tipe TypeScript untuk semua modul Astro. Ini menyiapkan [berkas `.astro/types.d.ts`](/id/guides/typescript/#setup) untuk inferensi tipe, dan mendefinisikan modul untuk fitur yang bergantung pada tipe yang dihasilkan:
- Modul `astro:content` untuk [Content Collections API](/id/guides/content-collections/).
- Modul `astro:db` untuk [Astro DB](/id/guides/astro-db/).
- Modul `astro:env` untuk [Astro Env](/id/guides/environment-variables/).
- Modul `astro:actions` untuk [Astro Actions](/id/guides/actions/)

## `astro add`

Menambahkan sebuah integrasi ke konfigurasi Anda. Baca selengkapnya di [panduan integrasi](/id/guides/integrations-guide/#automatic-integration-setup).

## `astro docs`

Membuka situs Dokumentasi Astro langsung dari terminal.

## `astro info`

Melaporkan informasi berguna tentang lingkungan Astro Anda saat ini. Berguna saat memberikan informasi ketika membuka sebuah issue.

```shell

```

Contoh output:

```
Output                   static

```

## `astro preferences`

Kelola preferensi pengguna dengan perintah `astro preferences`. Preferensi pengguna bersifat spesifik untuk masing-masing pengguna Astro, berbeda dengan berkas `astro.config.mjs` yang mengubah perilaku untuk semua orang yang bekerja pada sebuah proyek.

Preferensi pengguna dibatasi pada proyek saat ini secara default, disimpan di berkas lokal `.astro/settings.json`.

Dengan menggunakan flag `--global`, preferensi pengguna juga dapat diterapkan ke setiap proyek Astro pada mesin saat ini. Preferensi global disimpan di lokasi khusus sistem operasi.

<h3> Preferensi yang tersedia </h3>

- `devToolbar` — Mengaktifkan atau menonaktifkan development toolbar di browser. (Default: `true`)
- `checkUpdates` — Mengaktifkan atau menonaktifkan pemeriksaan pembaruan otomatis untuk Astro CLI. (Default: `true`)

Perintah `list` mencetak pengaturan saat ini untuk semua preferensi yang dapat dikonfigurasi. Ia juga mendukung output `--json` yang dapat dibaca mesin.

```shell
astro preferences list
```

Contoh output terminal:

| Preference               | Value |
| ------------------------ | ----- |
| devToolbar.enabled       | true  <tr></tr>|
| checkUpdates.enabled     | true  |


Anda dapat `enable`, `disable`, atau `reset` preferensi ke nilai defaultnya.

Sebagai contoh, untuk menonaktifkan devToolbar pada proyek Astro tertentu:

```shell
astro preferences disable devToolbar
```

Untuk menonaktifkan devToolbar pada semua proyek Astro di mesin saat ini:

```shell
astro preferences disable --global devToolbar
```

devToolbar nantinya dapat diaktifkan kembali dengan:

```shell
astro preferences enable devToolbar
```

Perintah `reset` mengembalikan preferensi ke nilai defaultnya:

```shell
astro preferences reset devToolbar
```

## `astro telemetry`

Mengatur konfigurasi telemetri untuk pengguna CLI saat ini. Telemetri adalah data anonim yang memberikan tim Astro wawasan tentang fitur-fitur Astro mana yang paling sering digunakan. Untuk informasi lebih lanjut lihat [halaman telemetri Astro](https://astro.build/telemetry/).

Telemetri dapat dinonaktifkan dengan perintah CLI ini:

```shell
astro telemetry disable
```

Telemetri dapat diaktifkan kembali dengan:

```shell
astro telemetry enable
```

Perintah `reset` akan mereset data telemetri:

```shell
astro telemetry reset
```

:::tip[Ingin menonaktifkan telemetri di lingkungan CI?]
Tambahkan perintah `astro telemetry disable` ke skrip CI Anda atau set variabel lingkungan `ASTRO_TELEMETRY_DISABLED`.
:::

## `astro create-key`

Menghasilkan kunci untuk mengenkripsi props yang diteruskan ke server islands.

```shell
astro create-key
```

Setel kunci ini sebagai variabel lingkungan `ASTRO_KEY` (misalnya di berkas `.env`) dan sertakan dalam CI/CD atau pengaturan build host Anda saat Anda membutuhkan [kunci enkripsi yang konstan untuk server islands](/id/guides/server-islands/#reusing-the-encryption-key) pada situasi seperti rolling deployments, hosting multi-region, atau CDN yang melakukan cache halaman yang berisi server islands.

## Common flags

### `--root <path>`

Menentukan path ke root proyek. Jika tidak ditentukan, direktori kerja saat ini diasumsikan sebagai root.

Root digunakan untuk menemukan berkas konfigurasi Astro.

```shell
astro --root myRootFolder/myProjectFolder dev
```

### `--config <path>`

Menentukan path ke berkas konfigurasi relatif terhadap root proyek. Default ke `astro.config.mjs`. Gunakan ini jika Anda menggunakan nama lain untuk berkas konfigurasi atau menaruhnya di folder lain.

```shell
astro --config config/astro.config.mjs dev
```

### `--force <string>`

<p><Since v="5.0.0" /></p>

Menghapus [cache content layer](/id/guides/content-collections/#defining-the-collection-loader), memaksa rebuild penuh.

### `--mode <string>`

<p><Since v="5.0.0" /></p>

Mengonfigurasi inline config [`mode`](/id/reference/programmatic-reference/#mode) untuk proyek Anda.

### `--outDir <path>`

<p><Since v="3.3.0" /></p>

Mengonfigurasi [`outDir`](/id/reference/configuration-reference/#outdir) untuk proyek Anda. Menggunakan flag ini akan menimpa nilai `outDir` pada berkas `astro.config.mjs` Anda, jika ada.

### `--site <url>`

Mengonfigurasi [`site`](/id/reference/configuration-reference/#site) untuk proyek Anda. Menggunakan flag ini akan menimpa nilai `site` pada berkas `astro.config.mjs` Anda, jika ada.

### `--base <pathname>`

<p><Since v="1.4.1" /></p>

Mengonfigurasi [`base`](/id/reference/configuration-reference/#base) untuk proyek Anda. Menggunakan flag ini akan menimpa nilai `base` pada berkas `astro.config.mjs` Anda, jika ada.

### `--port <number>`

Menentukan port untuk menjalankan dev server dan preview server. Default ke `4321`.

### `--host [optional host address]`

Mengatur alamat IP jaringan mana yang harus didengarkan oleh dev server dan preview server (misalnya alamat non-localhost). Ini berguna untuk menguji proyek Anda pada perangkat lokal seperti ponsel selama pengembangan.

- `--host` — mendengarkan di semua alamat, termasuk alamat LAN dan publik
- `--host <custom-address>` — mengekspos pada alamat IP jaringan di `<custom-address>`

:::caution
Jangan gunakan flag `--host` untuk mengekspos dev server dan preview server di lingkungan produksi. Server-server ini dirancang untuk penggunaan lokal saat mengembangkan situs saja.
:::

### `--allowed-hosts`

<p><Since v="5.4.0" /></p>

Menentukan hostname yang diizinkan untuk direspons Astro dalam mode `dev` atau `preview`. Dapat diisi daftar hostname yang dipisahkan koma atau `true` untuk mengizinkan hostname apa pun.

Lihat fitur [Vite `allowedHosts`](https://vite.dev/config/server-options.html#server-allowedhosts) untuk informasi lebih lanjut, termasuk implikasi keamanan saat mengizinkan hostname.

### `--verbose`

Mengaktifkan logging verbose, yang membantu saat debugging masalah.

### `--silent`

Mengaktifkan logging senyap, yang akan menjalankan server tanpa output konsol.

### `--open`

Secara otomatis membuka aplikasi di browser saat server mulai. Dapat diisi string URL penuh (mis. `--open http://example.com`) atau pathname (mis. `--open /about`) untuk menentukan URL yang akan dibuka.

## Global flags

Gunakan flag ini untuk mendapatkan informasi tentang CLI `astro`.

### `--version`

Mencetak nomor versi Astro dan keluar.

### `--help`

Mencetak pesan bantuan dan keluar.
