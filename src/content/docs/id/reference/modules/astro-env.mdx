API `astro:env` memungkinkan Anda mengonfigurasi skema bertipe (type‑safe) untuk variabel lingkungan yang Anda tetapkan. Ini memungkinkan Anda menunjukkan apakah variabel tersebut harus tersedia di server atau klien, serta mendefinisikan tipe data dan properti tambahannya. Untuk contoh dan petunjuk penggunaan, [lihat panduan `astro:env`](/id/guides/environment-variables/#type-safe-environment-variables).

## Impor dari `astro:env`

```js
import { 
  getSecret,
 } from 'astro:env/server';
```

### `getSecret()`

<p>
<Since v="5.0.0" />
</p>

Fungsi helper `getSecret()` memungkinkan pengambilan nilai mentah variabel lingkungan berdasarkan kuncinya.

Misalnya, Anda dapat mengambil nilai boolean sebagai string:

```js
import {{
  FEATURE_FLAG, // boolean
  getSecret
}} from 'astro:env/server'

getSecret('FEATURE_FLAG') // string | undefined
```

Ini juga berguna untuk mendapatkan secret yang tidak didefinisikan di skema Anda, misalnya yang bergantung pada data dinamis dari basis data atau API.

Jika Anda perlu mengambil variabel lingkungan secara terprogram, kami merekomendasikan menggunakan `getSecret()` alih‑alih `process.env` (atau yang setara). Karena implementasinya disediakan oleh adapter Anda, Anda tidak perlu memperbarui semua pemanggilan jika Anda mengganti adapter. Secara bawaan, ini menggunakan `process.env` pada mode dev dan build.
