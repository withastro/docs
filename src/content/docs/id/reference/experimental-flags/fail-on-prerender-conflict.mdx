Mengubah peringatan konflik prerender menjadi error selama proses build.

Saat ini, Astro akan memberi peringatan saat build tentang konflik antara beberapa rute dinamis yang dapat menghasilkan path output yang sama. Misalnya, `/blog/[slug]` dan `/blog/[...all]` keduanya bisa mencoba melakukan prerender pada path `/blog/post-1`. Dalam kasus seperti ini, Astro hanya akan merender [rute dengan prioritas tertinggi](/id/guides/routing/#route-priority-order) untuk path yang berkonflik. Ini memungkinkan situs Anda tetap berhasil dibuild, walaupun Anda mungkin mendapati beberapa halaman dirender oleh rute yang tidak Anda harapkan.

Dengan flag eksperimental ini diaktifkan, proses build akan langsung gagal dengan error. Ini mengharuskan Anda menyelesaikan konflik routing saat itu juga, dan memastikan Astro membangun rute sesuai keinginan Anda.

Untuk mengaktifkan perilaku ini, tambahkan feature flag `experimental.failOnPrerenderConflict` ke konfigurasi Astro Anda:

```js title="astro.config.mjs" ins={4-6}
import { defineConfig } from "astro/config"

defineConfig({
	experimental: {
		failOnPrerenderConflict: true,
	},
});
```

## Penggunaan

Setelah mengaktifkan flag ini, Anda mungkin akan menemui error tentang rute prerender yang berkonflik saat mencoba membuild proyek. Jika hal ini terjadi, Anda perlu memperbarui satu atau lebih [rute dinamis](/id/guides/routing/#dynamic-routes) untuk menghindari routing yang ambigu.
