Flag `experimental.headingIdCompat` membuat ID yang dihasilkan Astro untuk heading Markdown kompatibel dengan platform umum seperti GitHub dan npm.

Untuk mengaktifkan kompatibilitas ID heading, set flag ke `true` di konfigurasi Astro Anda:

```js title="astro.config.mjs" ins={4-6}
import { defineConfig } from "astro/config"

export default defineConfig({
  experimental: {
    headingIdCompat: true,
  }
})
```

## Penggunaan

Flag eksperimental ini memungkinkan Anda mempertahankan tanda hubung (hyphen) di akhir ID untuk heading Markdown yang berakhir dengan karakter spesial, sehingga membuat ID kompatibel dengan yang dihasilkan platform lain yang umum digunakan. Tidak memerlukan penggunaan khusus dan hanya memengaruhi bagaimana Astro menghasilkan `id` untuk heading Anda yang ditulis dengan sintaks Markdown.

Astro, seperti banyak platform, menggunakan paket populer [`github-slugger`](https://github.com/Flet/github-slugger) untuk mengonversi teks heading Markdown menjadi slug untuk digunakan pada ID. Flag eksperimental ini memungkinkan Anda menghilangkan langkah pemrosesan tambahan default Astro yang menghapus tanda hubung di akhir ID untuk heading yang berakhir dengan karakter spesial.

Sebagai contoh, heading Markdown berikut:

```md
## `<Picture />`
```

akan menghasilkan HTML berikut di Astro secara default:

```html "picture"
<h2 id="picture"><code>&lt;Picture /&gt;</code></h2>
```

Dengan `experimental.headingIdCompat`, Markdown yang sama akan menghasilkan HTML berikut, yang identik dengan platform seperti GitHub:

```html "picture-"
<h2 id="picture-"><code>&lt;Picture /&gt;</code></h2>
```

Di versi mayor mendatang, Astro akan beralih menggunakan gaya ID yang kompatibel ini secara default, namun Anda dapat ikut serta lebih awal menggunakan flag `experimental.headingIdCompat`.

## Penggunaan dengan plugin `rehypeHeadingIds`

Jika Anda [menggunakan plugin `rehypeHeadingIds`](/id/guides/markdown-content/#heading-ids-and-plugins) secara langsung, aktifkan mode kompatibilitas saat meneruskan plugin di konfigurasi Astro Anda:


```js title="astro.config.mjs" {8}
import { defineConfig } from 'astro/config';
import { rehypeHeadingIds } from '@astrojs/markdown-remark';
import { otherPluginThatReliesOnHeadingIDs } from 'some/plugin/source';

export default defineConfig({
  markdown: {
    rehypePlugins: [
      [rehypeHeadingIds, { headingIdCompat: true }],
      otherPluginThatReliesOnHeadingIDs,
    ],
  },
});
```
