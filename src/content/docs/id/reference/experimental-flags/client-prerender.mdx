**Type:** `boolean`<br />
**Default:** `false`<br />
<Since v="4.2.0" />
</p>

Mengaktifkan pra-render halaman yang telah diprefetch di sisi klien pada peramban yang didukung.

Fitur ini menggunakan [Speculation Rules Web API](https://developer.mozilla.org/en-US/docs/Web/API/Speculation_Rules_API) yang masih bersifat eksperimental dan meningkatkan perilaku `prefetch` bawaan secara global untuk melakukan prerender tautan di sisi klien.
Anda mungkin ingin meninjau [risiko yang mungkin terjadi saat melakukan prerender di sisi klien](https://developer.mozilla.org/en-US/docs/Web/API/Speculation_Rules_API#unsafe_prefetching) sebelum mengaktifkan fitur ini.

Aktifkan prerender sisi klien di `astro.config.mjs` bersama opsi konfigurasi `prefetch` yang diinginkan:

```js
// astro.config.mjs
{
  prefetch: {
    prefetchAll: true,
    defaultStrategy: 'viewport',
  },
	experimental: {
		clientPrerender: true,
	},
}
```

Tetap gunakan atribut `data-astro-prefetch` pada elemen tautan `<a />` mana pun di situs Anda untuk ikut serta melakukan prefetch.
Alih-alih menambahkan tag `<link>` ke head dokumen atau mengambil halaman dengan JavaScript, sebuah tag `<script>` akan ditambahkan dengan aturan spekulasi yang sesuai.

Prerender sisi klien memerlukan dukungan peramban. Jika Speculation Rules API tidak didukung, `prefetch` akan kembali ke strategi yang didukung.

Lihat [Panduan Prefetch](/id/guides/prefetch/) untuk opsi `prefetch` lainnya dan cara penggunaan.
