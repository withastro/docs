## Edit proyek Anda

Untuk membuat perubahan pada proyek, buka folder proyek Anda di editor kode. Bekerja dalam mode pengembangan dengan server dev berjalan memungkinkan Anda melihat pembaruan situs saat Anda mengedit kode.

Anda juga dapat [menyesuaikan aspek lingkungan pengembangan](#configure-your-dev-environment) seperti mengonfigurasi TypeScript atau memasang ekstensi editor resmi Astro.

### Mulai server dev Astro

Astro dilengkapi server pengembangan bawaan yang memiliki semua yang Anda butuhkan untuk pengembangan proyek. Perintah CLI `astro dev` akan menjalankan server pengembangan lokal sehingga Anda bisa melihat situs baru Anda beraksi untuk pertama kalinya.

Setiap template starter sudah memiliki skrip terkonfigurasi yang akan menjalankan `astro dev` untuk Anda. Setelah berpindah ke direktori proyek Anda, gunakan manajer paket favorit untuk menjalankan perintah ini dan memulai server pengembangan Astro.

<PackageManagerTabs>
  <Fragment slot="npm">
  ```shell
  npm run dev
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```shell
  pnpm run dev
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```shell
  yarn run dev
  ```
  </Fragment>
</PackageManagerTabs>

Jika semuanya berjalan baik, Astro sekarang akan menyajikan proyek Anda di [http://localhost:4321/](http://localhost:4321/). Kunjungi tautan itu di browser dan lihat situs baru Anda!

### Bekerja dalam mode pengembangan

Astro akan memantau perubahan file langsung di direktori `src/` dan memperbarui pratinjau situs saat Anda membangun, sehingga Anda tidak perlu memulai ulang server ketika melakukan perubahan selama pengembangan. Selama server dev berjalan, Anda selalu dapat melihat versi terbaru situs Anda di browser.

Saat melihat situs di browser, Anda akan memiliki akses ke [bilah alat dev Astro](/id/guides/dev-toolbar/). Saat Anda membangun, alat ini akan membantu memeriksa [islands](/id/concepts/islands/), menemukan masalah aksesibilitas, dan lainnya.

Jika Anda tidak dapat membuka proyek di browser setelah menjalankan server dev, kembali ke terminal tempat Anda menjalankan perintah `dev` dan periksa pesan yang ditampilkan. Seharusnya tertulis jika terjadi error, atau jika proyek Anda disajikan pada URL yang berbeda dari [http://localhost:4321/](http://localhost:4321/).

## Build dan preview situs Anda

Untuk memeriksa versi situs yang akan dibuat saat build, hentikan server dev (<kbd>Ctrl</kbd> + <kbd>C</kbd>) dan jalankan perintah build yang sesuai dengan manajer paket Anda di terminal:

<PackageManagerTabs>
  <Fragment slot="npm">
  ```shell
  npm run build
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```shell
  pnpm build
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```shell
  yarn run build
  ```
  </Fragment>
</PackageManagerTabs>

Astro akan membangun versi siap-deploy dari situs Anda dalam folder terpisah (bawaan: `dist/`) dan Anda bisa melihat progresnya di terminal. Ini akan memberi tahu Anda jika ada error build di proyek sebelum melakukan deploy ke produksi. Jika TypeScript dikonfigurasi ke `strict` atau `strictest`, skrip `build` juga akan memeriksa error tipe pada proyek Anda.

Setelah build selesai, jalankan perintah `preview` yang sesuai (mis. `npm run preview`) di terminal dan Anda bisa melihat versi hasil build situs Anda secara lokal di jendela pratinjau browser yang sama.

Perhatikan bahwa ini mempratinjau kode sebagaimana adanya ketika perintah build terakhir dijalankan. Ini dimaksudkan untuk memberi gambaran bagaimana situs Anda akan tampil ketika dideploy ke web. Perubahan apa pun yang Anda lakukan setelah proses build **tidak** akan tercermin saat preview hingga Anda menjalankan perintah build lagi.

Gunakan (<kbd>Ctrl</kbd> + <kbd>C</kbd>) untuk menutup preview dan menjalankan perintah terminal lain, seperti memulai ulang server dev untuk kembali ke [bekerja dalam mode pengembangan](#work-in-development-mode) yang memang memperbarui pratinjau secara langsung saat Anda mengedit.

<ReadMore>Pelajari lebih lanjut tentang [Astro CLI](/id/reference/cli-reference/) dan perintah terminal yang akan Anda gunakan saat membangun dengan Astro.</ReadMore>

:::tip
Anda mungkin ingin [langsung mendeploy situs baru](/id/guides/deploy/) sebelum Anda mulai menambah atau mengubah terlalu banyak kode. Ini membantu mempublikasikan versi minimal yang berfungsi dan dapat menghemat waktu serta usaha Anda saat troubleshooting deployment nanti.
:::
