## Opsi Deploy Cepat

Anda bisa membangun dan men-deploy situs Astro ke sejumlah host dengan cepat menggunakan UI dasbor situs mereka atau CLI.

### Antarmuka Situs Web

Cara cepat untuk men-deploy situs adalah menghubungkan repositori Git online proyek Astro Anda (mis. GitHub, GitLab, Bitbucket) ke penyedia host dan memanfaatkan continuous deployment menggunakan Git.

Platform host ini akan otomatis mendeteksi push ke repositori sumber proyek Astro Anda, membangun situs, lalu men-deploy-nya ke web pada URL khusus atau domain pribadi Anda. Sering kali, menyiapkan deployment di platform ini akan mengikuti langkah-langkah seperti berikut:

<Steps>
1. Tambahkan repositori Anda ke penyedia Git online (mis. di GitHub, GitLab, Bitbucket)

2. Pilih host yang mendukung **continuous deployment** (mis. [Netlify](/id/guides/deploy/netlify/) atau [Vercel](/id/guides/deploy/vercel/)) dan impor repositori Git Anda sebagai situs/proyek baru.

    Banyak host umum akan mengenali proyek Anda sebagai situs Astro, dan akan memilih pengaturan konfigurasi yang sesuai untuk membangun dan men-deploy situs Anda seperti di bawah. (Jika tidak, pengaturan ini bisa diubah.)

    :::note[Pengaturan deploy]
    - **Build Command:** `astro build` atau `npm run build`
    - **Publish directory:** `dist`
    :::

3. Klik "Deploy" dan website baru Anda akan dibuat pada URL unik milik host tersebut (mis. `new-astro-site.netlify.app`).
</Steps>

Host akan otomatis dikonfigurasi untuk memantau branch utama penyedia Git Anda terhadap perubahan, dan untuk membangun ulang serta menerbitkan ulang situs Anda pada setiap commit baru. Pengaturan ini biasanya dapat dikonfigurasi di UI dasbor penyedia host Anda.

### Deploy via CLI

Beberapa host memiliki command line interface (CLI) sendiri yang bisa Anda instal secara global menggunakan npm. Sering kali, penggunaan CLI untuk deploy terlihat seperti berikut:

<Steps>
1. Instal CLI host Anda secara global, contoh:

    <PackageManagerTabs>
        <Fragment slot="npm">
        ```shell
        npm install --global netlify-cli
        ```
        </Fragment>
        <Fragment slot="pnpm">
        ```shell
        pnpm add --global netlify-cli
        ```
        </Fragment>
        <Fragment slot="yarn">
        ```shell
        yarn global add netlify-cli
        ```
        </Fragment>
    </PackageManagerTabs>

2. Jalankan CLI dan ikuti instruksi untuk otorisasi, penyiapan, dll.

3. Build situs Anda dan deploy ke host

    Banyak host umum akan membangun dan men-deploy situs Anda untuk Anda. Mereka biasanya mengenali proyek Anda sebagai situs Astro, dan akan memilih pengaturan konfigurasi yang sesuai untuk build dan deploy seperti di bawah. (Jika tidak, pengaturan ini bisa diubah.)

    :::note[Pengaturan deploy]
    - **Build Command:** `astro build` atau `npm run build`
    - **Publish directory:** `dist`
    :::


    Host lain akan mengharuskan Anda untuk [membangun situs secara lokal](#membangun-situs-secara-lokal) dan men-deploy menggunakan command line.
</Steps>

## Membangun Situs Secara Lokal

Banyak host seperti Netlify dan Vercel akan membangun situs Anda untuk Anda lalu menerbitkan output build tersebut ke web. Namun, beberapa host akan mengharuskan Anda membangun secara lokal lalu menjalankan perintah deploy atau mengunggah output build Anda.

Anda juga mungkin ingin membangun secara lokal untuk mempratinjau situs, atau menangkap potensi error dan peringatan di lingkungan Anda sendiri.

Jalankan perintah `npm run build` untuk membangun situs Astro Anda.

<PackageManagerTabs>
      <Fragment slot="npm">
      ```shell
      npm run build
      ```
      </Fragment>
      <Fragment slot="pnpm">
      ```shell
      pnpm run build
      ```
      </Fragment>
      <Fragment slot="yarn">
      ```shell
      yarn run build
      ```
      </Fragment>
  </PackageManagerTabs>

Secara default, output build akan diletakkan di `dist/`. Lokasi ini dapat diubah menggunakan opsi konfigurasi [`outDir`](/id/reference/configuration-reference/#outdir).
