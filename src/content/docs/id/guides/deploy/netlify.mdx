[Netlify](https://netlify.com) menawarkan hosting dan layanan backend serverless untuk aplikasi web dan situs statis. Situs Astro apa pun bisa di-host di Netlify!

Panduan ini mencakup instruksi deployment ke Netlify melalui UI situs web atau CLI Netlify.

## Konfigurasi proyek

Proyek Astro Anda dapat di-deploy ke Netlify dengan tiga cara: sebagai situs statis, situs yang dirender di server (SSR), atau situs yang dirender di edge.

### Situs statis

Proyek Astro Anda secara default adalah situs statis. Anda tidak memerlukan konfigurasi tambahan untuk men-deploy situs Astro statis ke Netlify.

### Adapter untuk perenderan on-demand

Tambahkan [adapter Netlify](/id/guides/integrations-guide/netlify/) untuk mengaktifkan perenderan on-demand di proyek Astro Anda dan men-deploy ke Netlify dengan perintah `astro add` berikut. Ini akan memasang adapter dan melakukan perubahan yang sesuai pada file `astro.config.mjs` Anda dalam satu langkah.

```bash
npx astro add netlify
```

<ReadMore>Lihat [panduan adapter Netlify](/id/guides/integrations-guide/netlify/) untuk pemasangan manual atau opsi konfigurasi lainnya, seperti men-deploy middleware Astro Anda menggunakan Netlify Edge Functions.</ReadMore>


## Cara men-deploy

Anda bisa men-deploy ke Netlify melalui UI situs atau menggunakan CLI Netlify (command line interface). Prosesnya sama untuk situs Astro statis maupun yang dirender on-demand.

### Deployment via Website UI

Jika proyek Anda tersimpan di GitHub, GitLab, BitBucket, atau Azure DevOps, Anda dapat menggunakan UI situs Netlify untuk men-deploy situs Astro Anda.

<Steps>
1. Klik <kbd>Add a new site</kbd> di [dasbor Netlify](https://app.netlify.com/)

2. Pilih <kbd>Import an existing project</kbd>

    Saat Anda mengimpor repositori Astro dari penyedia Git Anda, Netlify biasanya mendeteksi dan mengisi otomatis pengaturan konfigurasi yang benar untuk Anda.

3. Pastikan pengaturan berikut diisi, lalu tekan tombol <kbd>Deploy</kbd>:

    - **Build Command:** `astro build` atau `npm run build`
    - **Publish directory:** `dist`

    Setelah deployment, Anda akan diarahkan ke halaman ringkasan situs. Di sana, Anda dapat mengubah detail situs Anda.
</Steps>

Perubahan apa pun di repositori sumber Anda di masa depan akan memicu deploy pratinjau dan produksi berdasarkan konfigurasi deployment Anda.

#### File `netlify.toml`

Anda bisa membuat file `netlify.toml` baru di tingkat atas repositori proyek untuk mengonfigurasi perintah build dan direktori publik, serta pengaturan proyek lainnya termasuk variabel lingkungan dan redirect. Netlify akan membaca file ini dan secara otomatis mengonfigurasi deployment Anda.

Untuk mengonfigurasi pengaturan default, buat file `netlify.toml` dengan konten berikut:

```toml
[build]
  command = "npm run build"
  publish = "dist"
```

<ReadMore>Info lebih lanjut di [“Mendeploy repositori Git Astro yang sudah ada”](https://www.netlify.com/blog/how-to-deploy-astro/#deploy-an-existing-git-repository-to-netlify) di blog Netlify</ReadMore>

### Deployment via CLI

Anda juga dapat membuat situs baru di Netlify dan menautkan repositori Git dengan memasang dan menggunakan [Netlify CLI](https://cli.netlify.com/).

<Steps>
1. Instal CLI Netlify secara global

    ```bash
    npm install --global netlify-cli
    ```

2. Jalankan `netlify login` dan ikuti instruksi untuk masuk dan mengotorisasi Netlify

3. Jalankan `netlify init` dan ikuti instruksi

4. Konfirmasikan perintah build Anda (`astro build`)

    CLI akan otomatis mendeteksi pengaturan build (`astro build`) dan direktori deploy (`dist`), serta menawarkan untuk menghasilkan [file `netlify.toml`](#file-netlifytoml) dengan pengaturan tersebut.

5. Build dan deploy dengan mendorong ke Git

    CLI akan menambahkan deploy key ke repositori, yang berarti situs Anda akan otomatis dibangun ulang di Netlify setiap kali Anda melakukan `git push`.
</Steps>

<ReadMore>Detail lebih lanjut dari Netlify: [Deploy situs Astro menggunakan Netlify CLI](https://www.netlify.com/blog/how-to-deploy-astro/#link-your-astro-project-and-deploy-using-the-netlify-cli)</ReadMore>

### Mengatur versi Node.js

Jika Anda menggunakan [build image](https://docs.netlify.com/configure-builds/get-started/#build-image-selection) lama (Xenial) di Netlify, pastikan versi Node.js Anda disetel. Astro memerlukan `v18.20.8` atau `v20.3.0` atau lebih tinggi.

Anda dapat [menentukan versi Node.js di Netlify](https://docs.netlify.com/configure-builds/manage-dependencies/#node-js-and-javascript) menggunakan:
- berkas [`.nvmrc`](https://github.com/nvm-sh/nvm#nvmrc) di direktori basis Anda.
- variabel lingkungan `NODE_VERSION` di pengaturan situs melalui dasbor proyek Netlify.

## Menggunakan Netlify Functions

Tidak diperlukan konfigurasi khusus untuk menggunakan Netlify Functions dengan Astro. Tambahkan direktori `netlify/functions` ke root proyek Anda dan ikuti [dokumentasi Netlify Functions](https://docs.netlify.com/functions/overview/) untuk memulai!
