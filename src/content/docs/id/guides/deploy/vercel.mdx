Anda dapat menggunakan [Vercel](http://vercel.com/) untuk mendeploy situs Astro ke jaringan edge global mereka tanpa konfigurasi apa pun.

Panduan ini mencakup instruksi untuk mendeploy ke Vercel melalui antarmuka situs web maupun CLI Vercel.

## Konfigurasi proyek

Proyek Astro Anda dapat dideploy ke Vercel sebagai situs statis, atau situs yang dirender di server.

### Situs statis

Proyek Astro Anda secara default adalah situs statis. Anda tidak memerlukan konfigurasi tambahan untuk mendeploy situs Astro statis ke Vercel. 

### Adapter untuk rendering sesuai permintaan

Tambahkan [adapter Vercel](/id/guides/integrations-guide/vercel/) untuk mengaktifkan [rendering sesuai permintaan](/id/guides/on-demand-rendering/) di proyek Astro Anda dengan perintah `astro add` berikut. Ini akan memasang adapter dan melakukan perubahan yang diperlukan pada file `astro.config.mjs` Anda dalam satu langkah.

<PackageManagerTabs>
  <Fragment slot="npm">
  ```shell
  npx astro add vercel
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```shell
  pnpm astro add vercel
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```shell
  yarn astro add vercel
  ```
  </Fragment>
</PackageManagerTabs>

<ReadMore>Lihat [panduan adapter Vercel](/id/guides/integrations-guide/vercel/) untuk pemasangan manual, atau opsi konfigurasi lainnya seperti mendeploy middleware Astro Anda menggunakan Vercel Edge Functions.</ReadMore>

## Cara mendeploy

Anda dapat mendeploy ke Vercel melalui antarmuka situs web atau menggunakan CLI Vercel (command line). Prosesnya sama untuk situs Astro statis maupun yang dirender sesuai permintaan.

### Deploy melalui antarmuka situs web

<Steps>
1. Push kode Anda ke repositori Git daring (GitHub, GitLab, BitBucket).

2. [Impor proyek Anda](https://vercel.com/new) ke Vercel.

3. Vercel akan secara otomatis mendeteksi Astro dan mengonfigurasi pengaturan yang benar.

4. Aplikasi Anda berhasil dideploy! (mis. [astro.vercel.app](https://astro.vercel.app/))
</Steps>

Setelah proyek Anda diimpor dan dideploy, setiap push berikutnya ke cabang mana pun akan menghasilkan [Preview Deployment](https://vercel.com/docs/concepts/deployments/preview-deployments), dan setiap perubahan pada Production Branch (umumnya “main”) akan menghasilkan [Production Deployment](https://vercel.com/docs/concepts/deployments/environments#production).

<ReadMore>Pelajari lebih lanjut tentang [Integrasi Git Vercel](https://vercel.com/docs/concepts/git).</ReadMore>

### Deploy melalui CLI

<Steps>
1. Pasang [Vercel CLI](https://vercel.com/cli) dan jalankan `vercel` untuk mendeploy.

    <PackageManagerTabs>
      <Fragment slot="npm">
      ```shell
      npm install -g vercel
      vercel
      ```
      </Fragment>
      <Fragment slot="pnpm">
      ```shell
      pnpm add -g vercel
      vercel
      ```
      </Fragment>
      <Fragment slot="yarn">
      ```shell
      yarn global add vercel
      vercel
      ```
      </Fragment>
    </PackageManagerTabs>

2. Vercel akan secara otomatis mendeteksi Astro dan mengonfigurasi pengaturan yang benar.

3. Saat ditanya `Want to override the settings? [y/N]`, pilih `N`.

4. Aplikasi Anda berhasil dideploy! (mis. [astro.vercel.app](https://astro.vercel.app/))
</Steps>

### Konfigurasi proyek dengan `vercel.json`

Anda dapat menggunakan `vercel.json` untuk menimpa perilaku default Vercel dan mengonfigurasi pengaturan tambahan. Misalnya, Anda dapat menambahkan header ke respons HTTP dari Deployment Anda. 

<ReadMore>Pelajari lebih lanjut tentang [konfigurasi proyek Vercel](https://vercel.com/docs/project-configuration).</ReadMore>
