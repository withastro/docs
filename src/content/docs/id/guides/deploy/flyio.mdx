Anda dapat mendeploy proyek Astro Anda ke [Fly.io](https://fly.io/), sebuah platform untuk menjalankan aplikasi full‑stack dan basis data dekat dengan pengguna Anda.

## Konfigurasi Proyek

Proyek Astro Anda dapat dideploy ke Fly.io sebagai situs statis, atau sebagai situs *server‑side rendered* (SSR).

### Situs Statis

Proyek Astro Anda secara bawaan adalah situs statis. Anda tidak memerlukan konfigurasi tambahan untuk mendeploy situs Astro statis ke Fly.io.

### Adapter untuk SSR

Untuk mengaktifkan *on‑demand rendering* pada proyek Astro Anda dan mendeploy ke Fly.io, tambahkan [adapter Node.js](/id/guides/integrations-guide/node/).

## Cara Mendeploy

<Steps>
1. [Daftar Fly.io](https://fly.io/docs/getting-started/log-in-to-fly/#first-time-or-no-fly-account-sign-up-for-fly) jika Anda belum memiliki akun.

2. [Instal `flyctl`](https://fly.io/docs/hands-on/install-flyctl/), pusat kendali aplikasi Anda di Fly.io.

3. Jalankan perintah berikut di terminal Anda.

    ```bash
    fly launch
    ```

	`flyctl` akan otomatis mendeteksi Astro, mengonfigurasi pengaturan yang benar, membangun image, dan mendeploy‑nya ke platform Fly.io. 
</Steps>

## Membuat Astro Dockerfile Anda

Jika Anda belum memiliki Dockerfile, `fly launch` akan membuatkannya untuk Anda, serta menyiapkan berkas `fly.toml`. Untuk halaman yang dirender *on demand*, Dockerfile ini akan menyertakan perintah mulai (*start command*) dan variabel lingkungan yang sesuai.

Sebagai alternatif, Anda dapat membuat Dockerfile sendiri menggunakan [Dockerfile generator](https://www.npmjs.com/package/@flydotio/dockerfile), lalu menjalankan perintah `npx dockerfile` untuk aplikasi Node atau `bunx dockerfile` untuk aplikasi Bun.
