[Railway](https://railway.com) adalah platform deployment yang menyederhanakan tumpukan infrastruktur Anda—dari server hingga observabilitas—dalam satu platform yang dapat diskalakan.

Panduan ini menjelaskan deployment situs Astro statis ke Railway menggunakan antarmuka web maupun alat Railway CLI.

:::tip
Untuk mendeply situs Astro dengan on-demand rendering (SSR) menggunakan Node adapter, ikuti [panduan Railway untuk mendeply situs Astro](https://docs.railway.com/guides/astro).
:::

## Konfigurasi Proyek

Sistem build bawaan Railway, [Railpack](https://docs.railway.com/reference/railpack), akan membangun proyek Astro Anda sebagai situs statis secara otomatis.

## Deploy melalui antarmuka web

<Steps>
  1. Buat [akun Railway](https://railway.com/dashboard) dan masuk.

  2. Dari dashboard Railway, buat [project](https://docs.railway.com/guides/projects) baru.
  
  3. Pilih opsi untuk deploy dari repositori GitHub, lalu pilih proyek Astro Anda.
  
  4. Hasilkan atau tambahkan domain kustom dari [pengaturan jaringan](https://docs.railway.com/guides/public-networking#railway-provided-domain) proyek Anda.
</Steps>

## Deploy melalui Railway CLI

<Steps>
  1. [Instal](https://docs.railway.com/guides/cli#installing-the-cli) alat Railway CLI.

  2. Masuk dengan perintah `railway login`.
  
  3. Dari dalam proyek Astro Anda, jalankan `railway init` dan pilih workspace serta nama proyek.
  
  4. Jalankan `railway up` untuk mendeply proyek Anda ke Railway.
  
  5. Jalankan `railway domain` untuk menghasilkan domain layanan yang disediakan Railway.
</Steps>
