Anda dapat mendeploy proyek Astro Anda di [Azion](https://console.azion.com/), sebuah platform bagi pengembang frontend untuk berkolaborasi dan mendeploy situs statis (JAMstack) maupun SSR.

## Prasyarat

Untuk memulai, Anda memerlukan:

- Akun [Azion](https://www.azion.com/). Jika belum punya, Anda dapat mendaftar akun gratis.  
- Kode aplikasi Anda disimpan di repositori [GitHub](https://github.com/).  
- [Azion CLI](https://www.azion.com/id/documentation/products/azion-cli/overview/) terpasang untuk penyiapan proyek dan deployment yang lebih cepat.

## Cara Mendeploy melalui Azion Console Dashboard

Untuk mulai membangun, ikuti langkah‑langkah berikut:

<Steps>
1. Akses [Azion Console](https://console.azion.com).  
2. Di halaman beranda, klik tombol **\+ Create**.  
   - Ini akan membuka modal dengan opsi untuk membuat aplikasi dan resource baru.  
3. Pilih opsi **Import from GitHub** lalu klik kartunya.  
   - Tindakan ini membuka halaman pengaturan.  
4. Hubungkan akun Azion Anda dengan GitHub.  
   - Jendela pop‑up akan muncul meminta otorisasi.  
5. Pilih repositori yang ingin Anda impor dari GitHub.  
6. Konfigurasikan pengaturan build:  
   - **Framework preset:** Pilih framework yang sesuai (misalnya `Astro`).  
   - **Root Directory:** Ini adalah direktori tempat kode Anda berada. Kode Anda harus berada di direktori root, bukan subdirektori. Simbol ./ muncul di bidang ini, menandakan direktori root.  
   - **Install Command:** perintah yang menjalankan proses kompilasi untuk build produksi. Perintah build dieksekusi melalui skrip. Contoh: npm run build atau npm install untuk paket NPM.  
7. Klik **Save and Deploy**.  
8. Pantau deployment menggunakan **Azion Real-Time Metrics** dan pastikan situs Anda aktif di edge.
</Steps>

## Cara Mendeploy Situs Statis Menggunakan Azion CLI

<Steps>

1. **Instal Azion CLI:**  
     
   - Unduh dan pasang [Azion CLI](https://www.azion.com/id/documentation/products/azion-cli/overview/) untuk mempermudah pengelolaan dan deployment.

   :::caution
   Azion CLI saat ini belum mendukung lingkungan Windows secara native. Namun, Anda dapat menggunakannya di Windows melalui Windows Subsystem for Linux (WSL). Ikuti [panduan instalasi WSL](https://docs.microsoft.com/en-us/windows/wsl/install) untuk menyiapkan lingkungan Linux di mesin Windows Anda.
   :::

2. **Otentikasi CLI:**  
     
   - Jalankan perintah berikut untuk mengotentikasi CLI Anda dengan akun Azion Anda.

   ```bash
   azion login
   ```

3. **Menyiapkan Aplikasi Anda:**  
     
   - Gunakan perintah berikut untuk melakukan inisialisasi dan konfigurasi proyek:

   ```bash
   azion init
   ```  

4. **Build Proyek Astro Anda:**  
     
   - Jalankan perintah build Anda secara lokal:

   ```bash
   azion build
   ```

5. **Deploy Berkas Statis Anda:**  
     
   - Deploy berkas statis menggunakan Azion CLI:

   ```bash
   azion deploy
   ```
</Steps>

Panduan ini memberikan gambaran umum untuk mendeploy aplikasi statis.

## Mengaktifkan Pengembangan Lokal Menggunakan Azion CLI

Agar pratinjau berfungsi, Anda harus menjalankan perintah berikut:

```bash
azion dev
```

Setelah Anda menginisialisasi server pengembangan lokal, aplikasi akan melalui proses `build`.

```bash 
Building your Edge Application. This process may take a few minutes
Running build step command:
...
```

Kemudian, ketika proses build selesai, akses ke aplikasi akan ditampilkan:

```bash
[Azion Bundler] [Server] › ✔  success   Function running on port http://localhost:3000 
```

## Pemecahan Masalah


### API runtime Node.js

Sebuah proyek yang menggunakan paket NPM gagal dibangun dengan pesan error seperti `[Error] Could not resolve "XXXX. The package "XXXX" wasn't found on the file system but is built into node.`:

Ini berarti paket atau impor yang Anda gunakan tidak kompatibel dengan API runtime Azion.

Jika Anda mengimpor API runtime Node.js secara langsung, silakan lihat [kompatibilitas Node.js Azion](https://www.azion.com/id/documentation/products/azion-edge-runtime/compatibility/node/) untuk langkah lebih lanjut tentang cara menyelesaikannya.

Jika Anda mengimpor paket yang di dalamnya mengimpor API runtime Node.js, periksa kepada pembuat paket apakah mereka mendukung sintaks impor `node:*`. Jika tidak, Anda mungkin perlu mencari paket alternatif.
