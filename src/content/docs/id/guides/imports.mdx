Astro mendukung sebagian besar aset statis tanpa konfigurasi apa pun. Anda dapat menggunakan pernyataan `import` di mana saja dalam JavaScript proyek Anda (termasuk frontmatter Astro) dan Astro akan menyertakan salinan aset statis tersebut yang telah dibangun serta dioptimalkan dalam build final Anda. `@import` juga didukung di dalam CSS & tag `<style>`.

## Jenis File yang Didukung

Jenis file berikut didukung langsung oleh Astro:

- Komponen Astro (`.astro`)
- Markdown (`.md`, `.markdown`, dll.)
- JavaScript (`.js`, `.mjs`)
- TypeScript (`.ts`)
- Paket NPM
- JSON (`.json`)
- CSS (`.css`)
- CSS Modules (`.module.css`)
- Gambar & Aset (`.svg`, `.jpg`, `.png`, dll.)

Selain itu, Anda dapat memperluas Astro untuk menambahkan dukungan bagi [UI framework](/id/guides/framework-components/) yang berbeda seperti komponen React, Svelte, dan Vue. Anda juga bisa memasang [integrasi Astro MDX](/id/guides/integrations-guide/mdx/) atau [integrasi Astro Markdoc](/id/guides/integrations-guide/markdoc/) untuk menggunakan file `.mdx` atau `.mdoc` di proyek Anda.

### Berkas di `public/`

Anda dapat menaruh aset statis apa pun di direktori [`public/`](/id/basics/project-structure/#public) proyek Anda, dan Astro akan menyalinnya langsung ke build final tanpa perubahan. Berkas `public/` tidak dibangun atau dibundel oleh Astro, yang berarti jenis berkas apa pun didukung.

Anda dapat mereferensikan berkas `public/` menggunakan path URL secara langsung di templat HTML Anda.

```astro
// Untuk menautkan ke /public/reports/annual/2024.pdf
Unduh <a href="/reports/annual/2024.pdf">laporan tahunan 2024 dalam bentuk PDF</a>.

// Untuk menampilkan /public/assets/cats/ginger.jpg
<img src="/assets/cats/ginger.jpg" alt="Seekor kucing oranye tidur di atas tempat tidur.">
```

## Pernyataan import

Astro menggunakan ESM, sintaks yang sama untuk [`import`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Statements/import#syntax) dan [`export`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Statements/export) yang didukung di browser.

### JavaScript

```js
import { getUser } from './user.js';
```

JavaScript dapat diimpor menggunakan sintaks ESM `import` & `export` seperti biasa.

:::note[Mengimpor berkas JSX]
Sebuah [UI framework](/id/guides/framework-components/) yang sesuai ([React](/id/guides/integrations-guide/react/), [Preact](/id/guides/integrations-guide/preact/), atau [Solid](/id/guides/integrations-guide/solid-js/)) diperlukan untuk merender berkas JSX/TSX.
Gunakan ekstensi `.jsx`/`.tsx` sesuai kebutuhan, karena Astro tidak mendukung JSX dalam berkas `.js`/`.ts`.
:::

### TypeScript

```js
import { getUser } from './user';
import type { UserType } from './user';
```

Astro menyertakan dukungan bawaan untuk [TypeScript](https://www.typescriptlang.org/). Anda dapat mengimpor berkas `.ts` dan `.tsx` secara langsung di proyek Astro Anda, dan bahkan menulis kode TypeScript langsung di dalam [skrip komponen Astro](/id/basics/astro-components/#the-component-script) serta [tag skrip](/id/guides/client-side-scripts/).

**Astro tidak melakukan pengecekan tipe apa pun sendiri.** Pengecekan tipe harus dilakukan di luar Astro, baik oleh IDE Anda maupun melalui skrip terpisah. Untuk pengecekan tipe berkas Astro, disediakan perintah [`astro check`](/id/reference/cli-reference/#astro-check).

:::note[TypeScript dan ekstensi berkas]
Sesuai [aturan resolusi modul TypeScript](https://www.typescriptlang.org/docs/handbook/module-resolution.html), ekstensi berkas `.ts` dan `.tsx` tidak boleh digunakan saat mengimpor berkas TypeScript. Sebagai gantinya, gunakan ekstensi `.js`/`.jsx` atau hilangkan ekstensi berkas sepenuhnya.

```ts
import { getUser } from './user.js'; // user.ts
import MyComponent from "./MyComponent"; // MyComponent.tsx
```
:::

<ReadMore>Baca selengkapnya tentang [dukungan TypeScript di Astro](/id/guides/typescript/).</ReadMore>


### Paket NPM

Jika Anda telah memasang sebuah paket NPM, Anda dapat mengimpornya di Astro.

```astro
---
import { Icon } from 'astro-icon';
---
```
Jika sebuah paket diterbitkan menggunakan format lama, Astro akan mencoba mengonversi paket tersebut ke ESM agar pernyataan `import` dapat berfungsi. Dalam beberapa kasus, Anda mungkin perlu menyesuaikan konfigurasi [`vite`](/id/reference/configuration-reference/#vite) agar dapat bekerja.

:::caution
Beberapa paket bergantung pada lingkungan browser. Komponen Astro berjalan di server, sehingga mengimpor paket-paket ini di frontmatter dapat [menyebabkan error](/id/guides/troubleshooting/#document-or-window-is-not-defined).
:::

### JSON

```js
// Memuat objek JSON melalui ekspor default
import json from './data.json';
```

Astro mendukung pengimporan berkas JSON langsung ke aplikasi Anda. Berkas yang diimpor mengembalikan objek JSON penuh pada impor defaultnya.

### CSS

```js
// Memuat dan menyisipkan 'style.css' ke dalam halaman
import './style.css';
```

Astro mendukung pengimporan berkas CSS langsung ke aplikasi Anda. Gaya yang diimpor tidak mengekspor apa pun, tetapi mengimpornya akan otomatis menambahkan gaya tersebut ke halaman. Ini berfungsi untuk semua berkas CSS secara default, dan dapat mendukung bahasa yang dikompilasi ke CSS seperti Sass & Less melalui plugin.

<ReadMore>Baca selengkapnya tentang kasus penggunaan impor CSS lanjutan seperti referensi URL langsung untuk berkas CSS, atau mengimpor CSS sebagai string di [panduan Styling](/id/guides/styling/#advanced).</ReadMore>

### CSS Modules

```jsx
// 1. Mengonversi class './style.module.css' menjadi nilai unik yang ter-scope.
// 2. Mengembalikan objek yang memetakan class asli ke nilai finalnya yang ter-scope.
import styles from './style.module.css';

// Contoh ini menggunakan JSX, tetapi Anda dapat menggunakan CSS Modules dengan framework apa pun.
return <div className={styles.error}>Pesan Error Anda</div>;
```

Astro mendukung CSS Modules menggunakan konvensi penamaan `[name].module.css`. Seperti berkas CSS lainnya, mengimpor satu berkas akan otomatis menerapkan CSS tersebut ke halaman. Namun, CSS Modules mengekspor objek default khusus `styles` yang memetakan class asli Anda ke identifier unik.

CSS Modules membantu Anda menerapkan scoping & isolasi komponen di frontend dengan nama class yang dihasilkan secara unik untuk stylesheet Anda.

### Aset lainnya

```jsx
// Mengembalikan objek dengan `src` dan properti lainnya
import imgReference from './image.png';
import svgReference from './image.svg';

// Komponen HTML atau UI Framework menggunakan ini untuk merender gambar
<img src={imgReference.src} alt="deskripsi gambar" />;

// Komponen Astro `<Image />` dan `<Picture />` mengakses `src` secara default
<Image src={imgReference} alt="deskripsi gambar">
```

Semua aset lain yang tidak disebutkan secara eksplisit di atas dapat diimpor melalui `import` ESM dan akan mengembalikan referensi URL ke aset hasil build final (mis. `/_astro/my-video.C7vXpQtF.mp4`) alih-alih sebuah objek.

Ini berguna untuk mereferensikan aset non-JS melalui URL, seperti membuat elemen video dengan atribut `src` yang mengarah ke aset tersebut.

Menempatkan gambar dan aset lainnya di folder `public/` juga dapat berguna sebagaimana dijelaskan pada halaman [struktur proyek](/id/basics/project-structure/#public).

<ReadMore>Baca selengkapnya tentang menambahkan parameter impor Vite (mis. `?url`, `?raw`) di [panduan penanganan aset statis Vite](https://vite.dev/guide/assets.html).</ReadMore>

:::note
Menambahkan **alt text** pada tag `<img>` sangat dianjurkan demi aksesibilitas! Jangan lupa tambahkan atribut `alt="deskripsi yang membantu"` pada elemen gambar Anda. Anda juga bisa membiarkan atributnya kosong jika gambar hanya dekoratif.
:::

## Alias

**Alias** adalah cara membuat pintasan untuk impor Anda.

Alias dapat membantu meningkatkan pengalaman pengembangan dalam basis kode dengan banyak direktori atau impor relatif.

```astro title="src/pages/about/company.astro" del="../../components" del="../../assets"
---
import Button from '../../components/controls/Button.astro';
import logoUrl from '../../assets/logo.png?url';
---
```

Dalam contoh ini, pengembang harus memahami hubungan pohon antara `src/pages/about/company.astro`, `src/components/controls/Button.astro`, dan `src/assets/logo.png`. Kemudian, jika berkas `company.astro` dipindahkan, impor-impor ini juga harus diperbarui.

Anda dapat menambahkan alias impor di `tsconfig.json`.

```json title="tsconfig.json" ins={4-5}
{
  "compilerOptions": {
    "paths": {
      "@components/*": ["./src/components/*"],
      "@assets/*": ["./src/assets/*"]
    }
  }
}
```

Server pengembangan akan otomatis restart setelah perubahan konfigurasi ini. Sekarang Anda dapat mengimpor menggunakan alias di mana saja dalam proyek Anda:

```astro title="src/pages/about/company.astro" ins="@components" ins="@assets"
---
import Button from '@components/controls/Button.astro';
import logoUrl from '@assets/logo.png?url';
---
```

## `import.meta.glob()`

[`import.meta.glob()` milik Vite](https://vite.dev/guide/features.html#glob-import) adalah cara untuk mengimpor banyak berkas sekaligus menggunakan pola glob untuk menemukan path berkas yang cocok.

`import.meta.glob()` menerima [pola glob](#glob-patterns) relatif yang cocok dengan berkas lokal yang ingin Anda impor sebagai parameter. Fungsi ini mengembalikan array dari setiap ekspor berkas yang cocok. Untuk memuat semua modul yang cocok di awal, berikan `{ eager: true }` sebagai argumen kedua:

```astro title="src/components/my-component.astro" {3,4}
---
// mengimpor semua berkas yang berakhiran `.md` di `./src/pages/post/`
const matches = import.meta.glob('../pages/post/*.md', { eager: true }); 
const posts = Object.values(matches);
---
<!-- Merender sebuah <article> untuk 5 postingan blog pertama -->
<div>
{posts.slice(0, 4).map((post) => (
  <article>
    <h2>{post.frontmatter.title}</h2>
    <p>{post.frontmatter.description}</p>
    <a href={post.url}>Baca selengkapnya</a>
  </article>
))}
</div>
```

Komponen Astro yang diimpor menggunakan `import.meta.glob` bertipe [`AstroInstance`](#astro-files). Anda dapat merender setiap instance komponen menggunakan properti `default`-nya:

```astro title="src/pages/component-library.astro" {8}
---
// mengimpor semua berkas yang berakhiran `.astro` di `./src/components/`
const components = Object.values(import.meta.glob('../components/*.astro', { eager: true }));
---
<!-- Tampilkan semua komponen kita -->
{components.map((component) => (
  <div>
    <component.default size={24} />
  </div>
))}
```

### Nilai yang didukung

Fungsi `import.meta.glob()` milik Vite hanya mendukung string literal statis. Fungsi ini tidak mendukung variabel dinamis dan interpolasi string.

Solusi umum adalah dengan mengimpor set berkas yang lebih besar yang mencakup semua berkas yang Anda butuhkan, lalu memfilternya:

```astro {6-7}
---
// src/components/featured.astro
const { postSlug } = Astro.props;
const pathToMyFeaturedPost = `src/pages/blog/${postSlug}.md`;

const posts = Object.values(import.meta.glob("../pages/blog/*.md", { eager: true }));
const myFeaturedPost = posts.find(post => post.file.includes(pathToMyFeaturedPost));
---

<p>
  Lihat postingan favorit saya, <a href={myFeaturedPost.url}>{myFeaturedPost.frontmatter.title}</a>!
</p>
```

### Utilitas tipe impor

#### Berkas Markdown

Berkas Markdown yang dimuat dengan `import.meta.glob()` mengembalikan antarmuka `MarkdownInstance` berikut:

```ts
export interface MarkdownInstance<T extends Record<string, any>> {
  /* Data apa pun yang ditentukan di frontmatter YAML/TOML berkas ini */
  frontmatter: T;
  /* Path berkas absolut dari berkas ini */
  file: string;
  /* Path terender dari berkas ini */
  url: string | undefined;
  /* Komponen Astro yang merender isi berkas ini */
  Content: AstroComponentFactory;
  /** (Hanya Markdown) Konten berkas Markdown mentah, tidak termasuk layout HTML dan frontmatter YAML/TOML */
  rawContent(): string;
  /** (Hanya Markdown) Berkas Markdown yang dikompilasi menjadi HTML, tidak termasuk layout HTML */
  compiledContent(): string;
  /* Fungsi yang mengembalikan array elemen h1...h6 di berkas ini */
  getHeadings(): Promise<{ depth: number; slug: string; text: string }[]>;
  default: AstroComponentFactory;
}
```

Anda dapat secara opsional memberikan tipe untuk variabel `frontmatter` menggunakan generic TypeScript.

```astro
---
import type { MarkdownInstance } from 'astro';
interface Frontmatter {
    title: string;
    description?: string;
}

const posts = Object.values(import.meta.glob<MarkdownInstance<Frontmatter>>('./posts/**/*.md', { eager: true }));
---

<ul>
  {posts.map(post => <li>{post.frontmatter.title}</li>)}
</ul>
```

#### Berkas Astro

Berkas Astro memiliki antarmuka berikut:

```ts
export interface AstroInstance {
  /* Path berkas dari berkas ini */
  file: string;
  /* URL untuk berkas ini (jika berada di direktori pages) */
  url: string | undefined;
  default: AstroComponentFactory;
}
```

#### Berkas lainnya

Berkas lain dapat memiliki berbagai antarmuka berbeda, tetapi `import.meta.glob()` menerima generic TypeScript jika Anda tahu persis apa yang dikandung tipe berkas yang tidak dikenali.

```ts
---
interface CustomDataFile {
  default: Record<string, any>;
}
const data = import.meta.glob<CustomDataFile>('../data/**/*.js');
---
```

### Pola glob

Pola glob adalah path berkas yang mendukung karakter wildcard khusus. Ini digunakan untuk mereferensikan beberapa berkas dalam proyek Anda sekaligus.

Sebagai contoh, pola glob `./pages/**/*.{md,mdx}` dimulai dari subdirektori pages, menelusuri semua subdirektorinya (`/**`), dan mencocokkan nama berkas apa pun (`/*`) yang berakhiran `.md` atau `.mdx` (`.{md,mdx}`).

#### Pola Glob di Astro

Untuk digunakan dengan `import.meta.glob()`, pola glob harus berupa string literal dan tidak boleh berisi variabel apa pun.

Selain itu, pola glob harus diawali dengan salah satu dari berikut:
- `./` (untuk mulai di direktori saat ini)
- `../` (untuk mulai di direktori induk)
- `/` (untuk mulai di root proyek)
 
[Baca selengkapnya tentang sintaks pola glob](https://github.com/micromatch/picomatch#globbing-features).

### `import.meta.glob()` vs `getCollection()`

[Content collections](/id/guides/content-collections/) menyediakan API [`getCollection()`](/id/reference/modules/astro-content/#getcollection) untuk memuat banyak berkas alih-alih `import.meta.glob()`. Jika berkas konten Anda (mis. Markdown, MDX, Markdoc) berada dalam koleksi di dalam direktori `src/content/`, gunakan `getCollection()` untuk [men-query sebuah koleksi](/id/guides/content-collections/#querying-collections) dan mengembalikan entri konten.

## WASM

```js
// Memuat dan menginisialisasi berkas WASM yang diminta
const wasm = await WebAssembly.instantiateStreaming(fetch('/example.wasm'));
```

Astro mendukung pemuatan berkas WASM langsung ke aplikasi Anda menggunakan API [`WebAssembly`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/WebAssembly) milik browser.


## Fitur built-in Node

Astro mendukung built-in Node.js, dengan beberapa batasan, menggunakan prefiks `node:` yang lebih baru. Mungkin ada perbedaan antara pengembangan dan produksi, dan beberapa fitur mungkin tidak kompatibel dengan rendering on-demand. Beberapa [adapter](/id/guides/on-demand-rendering/) juga mungkin tidak kompatibel dengan modul built-in ini atau memerlukan konfigurasi untuk mendukung subset tertentu (mis., [Cloudflare Workers](/id/guides/integrations-guide/cloudflare/) atau [Deno](https://github.com/denoland/deno-astro-adapter)).

Contoh berikut mengimpor modul `util` dari Node untuk mengurai tipe media (MIME):

```astro title="src/components/MyComponent.astro"
---
// Contoh: impor built-in "util" dari Node.js
import util from 'node:util';

export interface Props {
  mimeType: string,
}

const mime = new util.MIMEType(Astro.props.mimeType)
---

<span>Tipe: {mime.type}</span>
<span>SubTipe: {mime.subtype}</span>
```

## Memperluas dukungan jenis file

Dengan **Vite** dan plugin **Rollup** yang kompatibel, Anda dapat mengimpor jenis berkas yang tidak didukung secara native oleh Astro. Pelajari di mana menemukan plugin yang Anda butuhkan di bagian [Finding Plugins](https://vite.dev/guide/using-plugins.html#finding-plugins) pada Dokumentasi Vite.

:::note[Konfigurasi plugin]
Lihat dokumentasi plugin Anda untuk opsi konfigurasi, dan cara memasangnya dengan benar.
:::
