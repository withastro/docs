## Mengapa menggunakan media ter-host?

Media ter-host membantu individu, tim, dan organisasi menyimpan, mengelola, mengoptimalkan, serta mengantarkan aset gambar dan video dengan API khusus dari satu lokasi pusat.

Sentralisasi ini bermanfaat, terutama saat menggunakan single source of truth untuk aset Anda di berbagai properti web atau mobile. Ini penting jika Anda bagian dari organisasi yang meminta banyak tim menggunakan aset yang sama, atau saat berintegrasi dengan sistem konten lain seperti PIM (Product Information Manager) untuk menghubungkan aset Anda ke produk.

Layanan hosting gambar dapat melakukan transformasi dan optimasi gambar, secara otomatis mengantarkan versi yang telah dioptimalkan untuk pengunjung Anda. [Gambar jarak jauh](/id/guides/images/#remote-images) ini dapat digunakan di komponen bawaan Astro `<Image />` dan `<Picture />`, dan tersedia untuk semua jenis berkas di proyek Anda, termasuk Markdown, MDX, dan komponen UI Framework.

Layanan hosting video seperti [Mux](/id/guides/media/mux/) dapat menyediakan pengantaran video on‑demand dan live‑streaming yang andal serta pemutar video yang dapat dikustomisasi, memberi manfaat besar dalam keandalan dan skala dibanding menangani konten lokal. Mereka akan menangani transcoding, kompresi, dan transformasi video untuk memberikan pengalaman pengguna yang mulus. Platform seperti Mux juga dapat menyertakan analitik untuk membantu Anda memahami keterlibatan pengguna.

## Layanan media ter-host apa yang cocok dengan Astro?

Mirip saat menggunakan CMS, Anda akan ingin memakai layanan ter-host yang memungkinkan Anda mengambil dan berinteraksi dengan aset melalui API atau SDK. Beberapa layanan juga menyertakan komponen Astro native untuk menampilkan gambar atau video Anda.

## Bisakah saya menggunakan Astro tanpa layanan media ter-host?

Bisa! Astro menyediakan cara bawaan untuk [menyimpan gambar](/id/guides/images/#where-to-store-images), termasuk dukungan untuk mereferensikan gambar jarak jauh.

Namun, belum ada dukungan video native di Astro, dan kami merekomendasikan memilih layanan seperti [Mux](/id/guides/media/mux/) untuk menangani kebutuhan optimasi dan streaming konten video.
