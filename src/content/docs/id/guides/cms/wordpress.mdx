[WordPress](https://wordpress.org/) adalah sistem manajemen konten yang menyertakan frontend-nya sendiri, tetapi juga dapat digunakan sebagai headless CMS untuk menyediakan konten ke proyek Astro Anda.

## Integrating with Astro

WordPress hadir dengan [WordPress REST API](https://developer.wordpress.org/rest-api/) bawaan untuk menghubungkan data WordPress Anda ke Astro. Anda juga dapat memasang [WPGraphQL](https://wordpress.org/plugins/wp-graphql/) atau [Gato GraphQL](https://wordpress.org/plugins/gatographql/) di situs Anda untuk menggunakan GraphQL.

### Prerequisites

Untuk memulai, Anda memerlukan:

1. **Proyek Astro** â€“ Jika Anda belum memiliki proyek Astro, [panduan instalasi](/id/install-and-setup/) kami akan membantu Anda menyiapkannya dengan cepat.
2. **Situs WordPress** â€“ REST API situs Anda adalah `[YOUR_SITE]/wp-json/wp/v2/` dan tersedia secara default pada situs WordPress apa pun. Anda juga dapat [menyiapkan WordPress di lingkungan lokal](https://wordpress.org/support/article/installing-wordpress-on-your-own-computer/).

### Setting up Credentials

REST API WordPress Anda secara default tersedia untuk permintaan eksternal guna pengambilan data tanpa autentikasi. Ini tidak mengizinkan pengguna untuk mengubah data atau pengaturan situs Anda dan memungkinkan Anda menggunakan data dalam proyek Astro Anda tanpa kredensial apa pun.

Anda dapat memilih untuk [mewajibkan autentikasi](https://developer.wordpress.org/rest-api/frequently-asked-questions/#require-authentication-for-all-requests) jika diperlukan.

### Fetching Data

Ambil data WordPress Anda melalui URL REST API unik situs Anda dan rute untuk konten Anda. (Untuk blog, ini biasanya `posts`.) Kemudian, Anda dapat merender properti data menggunakan direktif `set:html={}` milik Astro. 

Sebagai contoh, untuk menampilkan daftar judul posting dan kontennya:

```astro title="src/pages/index.astro"
---
const res = await fetch("https://[YOUR-SITE]/wp-json/wp/v2/posts");
const posts = await res.json();
---
<h1>Astro + WordPress ðŸš€</h1>
{
  posts.map((post) => (
      <h2 set:html={post.title.rendered} />
      <p set:html={post.content.rendered} />
  ))
}
```

WordPress REST API menyertakan [parameter global](https://developer.wordpress.org/rest-api/using-the-rest-api/global-parameters/) seperti `_fields` dan `_embed`. 

Sejumlah besar data tersedia melalui API ini, sehingga Anda mungkin hanya ingin mengambil field tertentu. Anda dapat membatasi respons dengan menambahkan parameter [`_fields`](https://developer.wordpress.org/rest-api/using-the-rest-api/global-parameters/#_fields) ke URL API, misalnya: `[YOUR-SITE]/wp/v2/posts?_fields=author,id,excerpt,title,link` 

API juga dapat mengembalikan konten yang terkait dengan posting Anda, seperti tautan ke posting induk, atau ke komentar pada posting tersebut. Anda dapat menambahkan parameter [`_embed`](https://developer.wordpress.org/rest-api/using-the-rest-api/global-parameters/#_embed) ke URL API (mis. `[YOUR-SITE]/wp/v2/posts?_embed`) untuk memberi tahu server agar respons menyertakan resource yang disematkan ini.

## Building a blog with WordPress and Astro

Contoh ini mengambil data dari API WordPress publik milik [https://norian.studio/dinosaurs/](https://norian.studio/dinosaurs/). Situs WordPress ini menyimpan informasi tentang dinosaurus individual pada rute `dinos`, sebagaimana blog menyimpan posting individual pada rute `posts`.

Contoh ini menunjukkan cara mereproduksi struktur situs tersebut di Astro: halaman indeks yang mencantumkan dinosaurus dengan tautan ke halaman dinosaurus individual yang dibuat secara dinamis.

:::note
Untuk menggunakan [Custom Post Types (CPT)](https://learn.wordpress.org/lesson-plan/custom-post-types/) dalam API WordPress Anda (tidak hanya `post` dan `page`), Anda harus [mengonfigurasinya di dasbor WordPress](https://stackoverflow.com/questions/48536646/how-can-i-get-data-from-custom-post-type-using-wp-rest-api) atau [menambahkan dukungan REST API untuk tipe konten kustom](https://developer.wordpress.org/rest-api/extending-the-rest-api/adding-rest-api-support-for-custom-content-types/) di WordPress.

Contoh ini mengambil data dari situs WordPress yang tipe kontennya telah dikonfigurasi dan diekspos ke REST API.
:::

### Displaying a list of WordPress posts

Halaman `src/pages/index.astro` mencantumkan setiap dinosaurus, dengan deskripsi dan tautan ke halamannya sendiri.

<FileTree title="Project Structure">
- src/
  - pages/
    - **index.astro**
    - dinos/
      - [slug].astro
- astro.config.mjs
- package.json
</FileTree>

Pengambilan melalui API mengembalikan objek yang menyertakan properti:

- `title.rendered` â€“ Berisi HTML hasil render judul posting.
- `content.rendered` â€“ Berisi HTML hasil render konten posting.
- `slug` â€“ Berisi slug posting. (Ini menyediakan tautan ke halaman dinosaurus individual yang dibuat secara dinamis.)

```astro title="/src/pages/index.astro"
---
import Layout from "../layouts/Layout.astro";

let res = await fetch("https://norian.studio/wp-json/wp/v2/dinos");
let posts = await res.json();
---
<Layout title="Dinos!">
  <section>
    <h1>List of Dinosaurs</h1>
    {
      posts.map((post) => (
        <article>
          <h2>
            <a href={`/dinos/${post.slug}/`} set:html={post.title.rendered} />
          </h2>
          <Fragment set:html={post.content.rendered} />
        </article>
      ))
    }
  </section>
</Layout>
```

### Using the WordPress API to generate pages

Halaman `src/pages/dinos/[slug].astro` [membuat halaman secara dinamis](/id/guides/routing/#dynamic-routes) untuk setiap dinosaurus.

```astro title="/src/pages/dinos/[slug].astro"
---
import Layout from '../../layouts/Layout.astro';

const { slug } = Astro.params;

let res = await fetch(`https://norian.studio/wp-json/wp/v2/dinos?slug=${slug}`);
let [post] = await res.json();

// The getStaticPaths() is required for static Astro sites.
// If using SSR, you will not need this function.
export async function getStaticPaths() {
  let data = await fetch("https://norian.studio/wp-json/wp/v2/dinos");
  let posts = await data.json();

  return posts.map((post) => ({
    params: { slug: post.slug },
    props: { post: post },
  }));
}
---
<Layout title={post.title.rendered}>
  <article>
    <h1 set:html={post.title.rendered} />
    <Fragment set:html={post.content.rendered} />
  </article>
</Layout>
```

### Returning embedded resources

Parameter kueri `_embed` menginstruksikan server untuk mengembalikan resource terkait (embedded).

```astro title="src/pages/dinos/[slug].astro" /&_embed/
---
const { slug } = Astro.params;

let res = await fetch(`https://norian.studio/wp-json/wp/v2/dinos?slug=${slug}&_embed`);
let [post] = await res.json();
---
```

Properti `_embedded['wp:featuredmedia']['0'].media_details.sizes.medium.source_url` dikembalikan, dan dapat digunakan untuk menampilkan featured image pada setiap halaman dinosaurus. (Ganti `medium` dengan ukuran gambar yang Anda inginkan.)

```astro title="/src/pages/dinos/[slug].astro" {3}
<Layout title={post.title.rendered}>
  <article>
    <img src={post._embedded['wp:featuredmedia']['0'].media_details.sizes.medium.source_url} />
    <h1 set:html={post.title.rendered} />
    <Fragment set:html={post.content.rendered} />
  </article>
</Layout>
```
