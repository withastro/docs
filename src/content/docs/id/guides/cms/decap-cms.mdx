[Decap CMS](https://www.decapcms.org/) (sebelumnya Netlify CMS) adalah sistem manajemen konten open-source berbasis Git.

Decap memungkinkan Anda memanfaatkan semua fitur Astro, termasuk optimisasi gambar dan content collections.

Decap menambahkan rute (biasanya `/admin`) ke proyek Anda yang akan memuat aplikasi React untuk memungkinkan pengguna berizin mengelola konten langsung dari situs yang dideploy. Decap akan melakukan commit perubahan langsung ke repositori sumber proyek Astro Anda.

## Menginstal DecapCMS

Ada dua opsi untuk menambahkan Decap ke Astro:

1. [Instal Decap via package manager](https://decapcms.org/docs/install-decap-cms/#installing-with-npm) dengan perintah berikut:


    <PackageManagerTabs>
      <Fragment slot="npm">
      ```shell
      npm install decap-cms-app
      ```
      </Fragment>
      <Fragment slot="pnpm">
      ```shell
      pnpm add decap-cms-app
      ```
      </Fragment>
      <Fragment slot="yarn">
      ```shell
      yarn add decap-cms-app
      ```
      </Fragment>
    </PackageManagerTabs>
 
 
2. Impor paket ke dalam tag `<script>` di `<body>` halaman

	```html title='/admin'
    <body>
      <!-- Sertakan skrip yang membangun halaman dan menjalankan Decap CMS -->
      <script src="https://unpkg.com/decap-cms@^3.1.2/dist/decap-cms.js"></script>
    </body>
	```
 

## Konfigurasi

<Steps>
1. Buat folder statis admin di `public/admin/`

2. Tambahkan `config.yml` ke `public/admin/`:
    <FileTree>
      - public
        - admin
          - config.yml
    </FileTree>

3. Untuk menambahkan dukungan content collections, konfigurasikan setiap schema di `config.yml`. Contoh berikut mengonfigurasi koleksi `blog`, mendefinisikan `label` untuk setiap properti frontmatter entri:

    ```yml title="/public/admin/config.yml"
    collections:
      - name: "blog" # Digunakan di rute, mis. /admin/collections/blog
        label: "Blog" # Digunakan di UI
        folder: "src/content/blog" # Path ke folder tempat dokumen disimpan
        create: true # Izinkan pengguna membuat dokumen baru dalam koleksi ini
        fields: # Field untuk setiap dokumen, biasanya di frontmatter
          - { label: "Layout", name: "layout", widget: "hidden", default: "blog" }
          - { label: "Title", name: "title", widget: "string" }
          - { label: "Publish Date", name: "date", widget: "datetime" }
          - { label: "Featured Image", name: "thumbnail", widget: "image" }
          - { label: "Rating (scale of 1-5)", name: "rating", widget: "number" }
          - { label: "Body", name: "body", widget: "markdown" }
    ```

4. Tambahkan rute `admin` untuk aplikasi React Anda di `src/pages/admin.html`.
    <FileTree>
    - public
      - admin
         - config.yml
    - src
      - pages
        - admin.html
    </FileTree>

    ```html title="/src/pages/admin.html" {7, 11}
    <!doctype html>
    <html lang="en">
      <head>
        <meta charset="utf-8" />
        <meta name="viewport" content="width=device-width, initial-scale=1.0" />
        <meta name="robots" content="noindex" />
        <link href="/admin/config.yml" type="text/yaml" rel="cms-config-url" />
        <title>Content Manager</title>
      </head>
      <body>
        <script src="https://unpkg.com/decap-cms@^3.1.2/dist/decap-cms.js"></script>
      </body>
    </html>
    ```

5. Untuk mengaktifkan unggahan media ke folder tertentu melalui editor Decap, tambahkan path yang sesuai:
    ```yml title="/public/admin/config.yml"
    media_folder: "src/assets/images" # Lokasi penyimpanan berkas di repo 
    public_folder: "src/assets/images" # Atribut src untuk media yang diunggah
    ```
</Steps>

Lihat [dokumentasi konfigurasi Decap CMS](https://decapcms.org/docs/configure-decap-cms/) untuk instruksi dan opsi lengkap.

## Penggunaan
Buka `yoursite.com/admin/` untuk menggunakan editor Decap CMS.

## Autentikasi

### Decap CMS dengan Netlify Identity

Decap CMS awalnya dikembangkan oleh Netlify dan memiliki dukungan kelas satu untuk [Netlify Identity](https://docs.netlify.com/security/secure-access-to-sites/identity/).

Saat deploy ke Netlify, konfigurasikan Identity untuk proyek Anda melalui dashboard Netlify dan sertakan [Netlify Identity Widget](https://github.com/netlify/netlify-identity-widget) pada rute `admin` proyek Anda. Opsional, sertakan juga Identity Widget pada halaman beranda jika Anda berencana mengundang pengguna baru via email.

### Decap CMS dengan External OAuth Clients

Saat deploy ke penyedia hosting selain Netlify, Anda harus membuat rute OAuth sendiri.

Di Astro, ini dapat dilakukan dengan rute on-demand rendering di proyek Anda yang dikonfigurasikan dengan [adapter](/id/guides/on-demand-rendering/) yang aktif.

Lihat [Dokumentasi OAuth Decap](https://decapcms.org/docs/external-oauth-clients/) untuk daftar klien OAuth komunitas yang kompatibel.
