[Tina CMS](https://tina.io/) adalah sistem manajemen konten (*headless CMS*) berbasis Git.

## Mengintegrasikan dengan Astro

Untuk memulai, Anda memerlukan proyek Astro yang sudah ada.

<Steps>
1. Jalankan perintah berikut untuk menginstal Tina ke dalam proyek Astro Anda.

    <PackageManagerTabs>
      <Fragment slot="npm">
      ```shell
      npx @tinacms/cli@latest init
      ```
      </Fragment>
      <Fragment slot="pnpm">
      ```shell
      pnpm dlx @tinacms/cli@latest init
      ```
      </Fragment>
      <Fragment slot="yarn">
      ```shell
      yarn dlx @tinacms/cli@latest init
      ```
      </Fragment>
    </PackageManagerTabs>

    - Saat diminta Cloud ID, tekan <kbd>Enter</kbd> untuk melewati. Anda bisa membuatnya nanti jika ingin menggunakan Tina Cloud.
    - Saat ditanya “What framework are you using”, pilih **Other**.
    - Saat ditanya lokasi aset publik, tekan <kbd>Enter</kbd>.

    Setelah selesai, sekarang Anda akan memiliki folder `.tina` di root proyek dan sebuah file `hello-world.md` yang dibuat di `content/posts`.

2. Ubah skrip `dev` di `package.json`:

    <PackageManagerTabs>
      <Fragment slot="npm">
      ```json del={4} ins={5}
      // package.json
      {
          "scripts": {
              "dev": "astro dev",
              "dev": "tinacms dev -c \"astro dev\""
          }
      }
      ```
      </Fragment>
      <Fragment slot="pnpm">
      ```json del={4} ins={5}
      // package.json
      {
          "scripts": {
              "dev": "astro dev",
              "dev": "tinacms dev -c \"astro dev\""
          }
      }
      ```
      </Fragment>
      <Fragment slot="yarn">
      ```json del={4} ins={5}
      // package.json
      {
          "scripts": {
              "dev": "astro dev",
              "dev": "tinacms dev -c \"astro dev\""
          }
      }
      ```
      </Fragment>
    </PackageManagerTabs>

3. TinaCMS sekarang disiapkan dalam *local mode*. Uji dengan menjalankan skrip `dev`, lalu buka `/admin/index.html#/collections/post`.

    Mengedit pos “Hello, World!” akan memperbarui file `content/posts/hello-world.md` di direktori proyek Anda.

4. Siapkan koleksi Tina Anda dengan mengedit properti `schema.collections` di `.tina/config.ts`.

    Misalnya, Anda bisa menambahkan properti frontmatter “tanggal terbit” yang wajib pada pos:
    
    ```js title=".tina/config.ts" ins={35-40}
    import { defineConfig } from "tinacms";

    // Penyedia hosting Anda kemungkinan mengekspos ini sebagai variabel lingkungan
    const branch = process.env.HEAD || process.env.VERCEL_GIT_COMMIT_REF || "main";

    export default defineConfig({
      branch,
      clientId: null, // Dapatkan ini dari tina.io
      token: null, // Dapatkan ini dari tina.io
      build: {
        outputFolder: "admin",
        publicFolder: "public",
      },
      media: {
        tina: {
          mediaRoot: "images",
          publicFolder: "public",
        },
      },
      schema: {
        collections: [
          {
            name: "posts",
            label: "Posts",
            path: "src/content/posts",
            format: 'mdx',
            fields: [
              {
                type: "string",
                name: "title",
                label: "Title",
                isTitle: true,
                required: true,
              },
              {
                type: "datetime",
                name: "posted",
                label: "Date Posted",
                required: true,
              },
              {
                type: "rich-text",
                name: "body",
                label: "Body",
                isBody: true,
              },
            ],
          },
        ],
      },
    });
    ```

    Pelajari lebih lanjut tentang koleksi Tina [di dokumentasi Tina](https://tina.io/docs/reference/collections/).

5. Di produksi, TinaCMS dapat *commit* perubahan langsung ke repositori GitHub Anda. Untuk menyiapkan TinaCMS di produksi, Anda bisa memilih menggunakan [Tina Cloud](https://tina.io/docs/tina-cloud/) atau *self-host* [Tina Data Layer](https://tina.io/docs/self-hosted/overview/). Anda dapat [membaca lebih lanjut tentang pendaftaran Tina Cloud](https://app.tina.io/register) di Dokumentasi Tina.
</Steps>
