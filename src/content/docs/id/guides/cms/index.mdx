## Mengapa menggunakan CMS?

Sistem Manajemen Konten (CMS) memungkinkan Anda menulis konten dan mengelola aset di luar proyek Astro Anda.

Ini membuka fitur baru untuk bekerja dengan konten. Sebagian besar CMS memberi Anda editor visual, kemampuan menentukan tipe konten standar, dan cara untuk berkolaborasi dengan orang lain. 

CMS berguna untuk konten yang mengikuti struktur tertentu, sering kali memberikan pengalaman seperti *dashboard* dan alat penyuntingan WYSIWYG. Anda bisa menggunakan CMS untuk menulis posting blog menggunakan editor *rich text* alih-alih file Markdown. Atau, Anda dapat menggunakan CMS untuk memelihara daftar produk toko eCommerce, menjadikan field tertentu wajib diisi agar tidak ada data yang kurang. 

Proyek Astro Anda kemudian dapat mengambil (fetch) konten dari CMS dan menampilkannya, kapan pun dan bagaimana pun Anda inginkan di situs Anda.


## CMS apa yang cocok dengan Astro?

Karena Astro menangani _presentasi_ konten Anda, Anda sebaiknya memilih CMS _headless_, seperti yang ada pada daftar di atas. Artinya, CMS membantu Anda menulis konten, tetapi tidak menghasilkan situs untuk menampilkannya. Sebagai gantinya, Anda mengambil data konten tersebut dan menggunakannya di proyek Astro Anda. 

Beberapa CMS headless, seperti Storyblok, menyediakan [integrasi](/id/guides/integrations-guide/) Astro yang membantu *fetch* konten secara khusus untuk situs Astro. Yang lain menyediakan JavaScript SDK, yaitu pustaka yang Anda pasang dan gunakan untuk mengambil konten jarak jauh Anda.
