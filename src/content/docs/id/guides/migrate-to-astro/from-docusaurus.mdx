[Docusaurus](https://Docusaurus.io) adalah pembuat situs dokumentasi populer yang dibangun di atas React.

## Persamaan Utama antara Docusaurus dan Astro

Docusaurus dan Astro memiliki beberapa kesamaan yang akan membantu Anda memigrasikan proyek Anda:

- Baik Astro maupun Docusaurus adalah pembangun situs modern berbasis JavaScript (Jamstack) yang ditujukan untuk [situs berorientasi konten](/id/concepts/why-astro/#content-driven), seperti situs dokumentasi. 

- Baik Astro maupun Docusaurus mendukung [halaman MDX](/id/guides/markdown-content/). Anda seharusnya dapat menggunakan berkas `.mdx` yang sudah ada dengan Astro.

- Baik Astro maupun Docusaurus menggunakan [routing berbasis file](/id/guides/routing/) untuk membuat rute halaman secara otomatis untuk setiap berkas MDX di `src/pages`. Menggunakan struktur berkas Astro untuk konten yang sudah ada dan saat menambah halaman baru akan terasa familiar.

- Astro memiliki [integrasi resmi untuk menggunakan komponen React](/id/guides/integrations-guide/react/). Perhatikan bahwa di Astro, berkas React **harus** berekstensi `.jsx` atau `.tsx`.

- Astro mendukung [memasang paket NPM](/id/guides/imports/#npm-packages), termasuk beberapa paket untuk React. Anda mungkin bisa mempertahankan sebagian atau seluruh komponen dan dependensi React yang sudah ada.

- [Sintaks Astro yang mirip JSX](/id/basics/astro-components/#the-component-template) akan terasa familiar jika Anda terbiasa menulis React.


## Perbedaan Utama antara Docusaurus dan Astro

Saat Anda membangun ulang situs Docusaurus di Astro, Anda akan melihat beberapa perbedaan penting:

- Docusaurus adalah aplikasi satu halaman (SPA) berbasis React. Situs Astro adalah aplikasi multi-halaman yang dibangun menggunakan [komponen `.astro`](/id/basics/astro-components/), tetapi juga dapat mendukung [React, Preact, Vue.js, Svelte, SolidJS, AlpineJS](/id/guides/framework-components/) dan templating HTML murni.

- Docusaurus dirancang untuk membangun situs dokumentasi dan memiliki beberapa fitur bawaan yang khusus untuk situs dokumentasi, yang perlu Anda bangun sendiri di Astro. Sebagai gantinya, Astro menawarkan sebagian fitur tersebut melalui [Starlight: tema dokumentasi resmi](https://starlight.astro.build). Situs ini adalah inspirasi untuk Starlight, dan kini berjalan di atasnya! Anda juga dapat menemukan lebih banyak [tema dokumentasi komunitas](https://astro.build/themes?search=&categories%5B%5D=docs) dengan fitur bawaan di **Themes Showcase** kami.

- Situs Docusaurus menggunakan halaman MDX untuk konten. Tema dokumentasi Astro menggunakan berkas Markdown (`.md`) secara bawaan dan tidak mewajibkan Anda menggunakan MDX. Anda dapat secara opsional [memasang integrasi MDX Astro](/id/guides/integrations-guide/mdx/) (sudah termasuk secara default di tema Starlight kami) dan menggunakan berkas `.mdx` selain berkas Markdown standar.


## Beralih dari Docusaurus ke Astro

Untuk mengonversi situs dokumentasi Docusaurus ke Astro, mulailah dengan [starter template tema dokumentasi Starlight](https://starlight.astro.build) resmi kami, atau jelajahi lebih banyak tema dokumentasi komunitas di [theme showcase](https://astro.build/themes?search=&categories%5B%5D=docs). 

Anda dapat menambahkan argumen `--template` ke perintah `create astro` untuk memulai proyek Astro baru dengan salah satu starter resmi kami. Atau, Anda dapat [memulai proyek baru dari repositori Astro mana pun di GitHub](/id/install-and-setup/#install-from-the-cli-wizard).

  <PackageManagerTabs>
    <Fragment slot="npm">
    ```shell
    npm create astro@latest -- --template starlight
    ```
    </Fragment>
    <Fragment slot="pnpm">
    ```shell
    pnpm create astro@latest --template starlight
    ```
    </Fragment>
    <Fragment slot="yarn">
    ```shell
    yarn create astro --template starlight
    ```
    </Fragment>
  </PackageManagerTabs>

Integrasi MDX Astro sudah termasuk secara default, sehingga Anda dapat [membawa berkas konten yang sudah ada ke Starlight](https://starlight.astro.build/getting-started/#add-content) segera.

Anda dapat menemukan starter dokumentasi Astro, dan template resmi lainnya, di [astro.new](https://astro.new). Anda akan menemukan tautan ke setiap repositori GitHub proyek, serta tautan sekali klik untuk membuka proyek yang berfungsi di lingkungan pengembangan online seperti IDX, StackBlitz, CodeSandbox, dan Gitpod.
