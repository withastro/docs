[WordPress](https://wordpress.org) adalah sistem penerbitan personal sumber terbuka yang dibangun di atas PHP dan MySQL.

:::tip
Anda dapat [menggunakan WordPress sebagai headless CMS untuk proyek Astro Anda](/id/guides/cms/wordpress/). Ikuti panduan kami untuk menggunakan konten WordPress Anda yang ada di proyek Astro baru.
:::

## Persamaan Utama antara WordPress dan Astro

WordPress dan Astro memiliki beberapa kesamaan yang akan membantu Anda memigrasikan proyek Anda:

- Baik WordPress maupun Astro ideal untuk [situs berorientasi konten](/id/concepts/why-astro/#content-driven) seperti blog dan mendukung penulisan konten dalam Markdown (di WordPress memerlukan plugin). Walaupun proses penambahan konten baru berbeda, [menulis dalam berkas Markdown](/id/guides/markdown-content/) untuk blog Astro Anda akan terasa familiar jika Anda sudah menggunakan sintaks Markdown di editor WordPress Anda.

- WordPress dan Astro mendorong Anda untuk [memikirkan desain situs dalam “blok”](/id/concepts/islands/) (komponen). Di Astro Anda mungkin akan [menulis lebih banyak kode sendiri untuk membuat blok‑blok ini](/id/basics/astro-components/) daripada mengandalkan plugin siap pakai. Namun, memikirkan potongan‑potongan situs Anda dan bagaimana cara menampilkannya di halaman akan terasa familiar.

## Perbedaan Utama antara WordPress dan Astro

Saat Anda membangun ulang situs WordPress di Astro, Anda akan melihat beberapa perbedaan penting:

- Situs WordPress diedit menggunakan dasbor daring. Di Astro, Anda akan menggunakan [editor kode](/id/editor-setup/) dan lingkungan pengembangan untuk memelihara situs Anda. Anda dapat mengembangkan secara lokal di mesin Anda, atau memilih editor/lingkungan pengembangan awan seperti IDX, StackBlitz, CodeSandbox, atau Gitpod.

- WordPress memiliki pasar plugin dan tema yang sangat luas. Di Astro, Anda akan menemukan beberapa tema dan [integrasi](https://astro.build/integrations/) yang tersedia, tetapi Anda mungkin kini harus membangun banyak fitur yang sebelumnya Anda dapatkan dari pihak ketiga. Atau, Anda bisa memilih untuk memulai dengan [tema Astro](https://astro.build/themes) yang sudah memiliki fitur bawaan!

- WordPress menyimpan konten Anda di basis data. Di Astro, Anda akan memiliki berkas individual (biasanya Markdown atau MDX) di [direktori proyek](/id/basics/project-structure/) untuk konten setiap halaman. Atau, Anda bisa memilih untuk menggunakan [CMS untuk konten Anda](/id/guides/cms/), bahkan WordPress yang sudah ada, dan menggunakan Astro untuk mengambil dan menampilkan datanya.

## Beralih dari WordPress ke Astro

Untuk mengonversi blog WordPress ke Astro, mulailah dengan *starter template* tema blog kami, atau jelajahi lebih banyak tema blog komunitas di [theme showcase](https://astro.build/themes). 

Anda dapat menambahkan argumen `--template` ke perintah `create astro` untuk memulai proyek Astro baru dengan salah satu *starter* resmi. Atau, Anda dapat [memulai proyek baru dari repositori Astro mana pun di GitHub](/id/install-and-setup/#install-from-the-cli-wizard).

  <PackageManagerTabs>
    <Fragment slot="npm">
    ```shell
    npm create astro@latest -- --template blog
    ```
    </Fragment>
    <Fragment slot="pnpm">
    ```shell
    pnpm create astro@latest --template blog
    ```
    </Fragment>
    <Fragment slot="yarn">
    ```shell
    yarn create astro --template blog
    ```
    </Fragment>
  </PackageManagerTabs>

Anda dapat terus [menggunakan blog WordPress yang ada sebagai CMS untuk Astro](/id/guides/cms/wordpress/), artinya Anda akan tetap menggunakan dasbor WordPress untuk menulis posting. Konten Anda akan dikelola di WordPress, tetapi semua aspek lain dari situs Astro Anda akan dibangun di lingkungan penyuntingan kode Anda, dan Anda akan [*deploy* situs Astro](/id/guides/deploy/) secara terpisah dari situs WordPress Anda. (Pastikan untuk memperbarui domain di penyedia hosting Anda agar URL situs tetap sama!)

Anda mungkin ingin mengikuti [Tutorial Membangun Blog Astro](/id/tutorial/0-introduction/) jika Anda baru bekerja di editor kode dan menggunakan GitHub untuk menyimpan dan menerbitkan situs. Tutorial itu akan memandu semua akun dan penyiapan yang Anda perlukan! Anda juga akan belajar cara [membangun komponen Astro sendiri](/id/tutorial/3-components/), dan akan ditunjukkan cara [menambahkan posting blog langsung di Astro](/id/tutorial/2-pages/2/) jika Anda memilih untuk tidak menggunakan WordPress untuk menulis konten.

Jika Anda ingin memindahkan semua konten posting Anda yang ada ke Astro, Anda mungkin akan terbantu oleh [alat untuk mengekspor Markdown dari WordPress ini](https://github.com/lonekorean/wordpress-export-to-markdown). Anda mungkin perlu melakukan beberapa penyesuaian pada hasilnya jika Anda harus [mengonversi situs WordPress yang besar atau rumit ke Markdown](https://swizec.com/blog/how-to-export-a-large-wordpress-site-to-markdown/).

Untuk mengonversi jenis situs lainnya, seperti portofolio atau situs dokumentasi, lihat lebih banyak *starter template* resmi di [astro.new](https://astro.new). Anda akan menemukan tautan ke setiap repositori GitHub proyek, serta tautan sekali klik untuk membuka proyek yang berfungsi di lingkungan pengembangan daring seperti IDX, StackBlitz, CodeSandbox, dan Gitpod.
