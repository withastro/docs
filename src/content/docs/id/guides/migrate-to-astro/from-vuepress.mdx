[VuePress](https://vuePress.vuejs.org) adalah *static site generator* sumber terbuka yang dibangun di atas Vue.

## Persamaan Utama antara VuePress dan Astro

VuePress dan Astro memiliki beberapa kesamaan yang akan membantu Anda memigrasikan proyek Anda:

- Baik VuePress maupun Astro adalah *static‑site generator* JavaScript modern dengan struktur berkas proyek yang serupa. Keduanya menggunakan [folder khusus `src/pages/` untuk routing berbasis file](/id/basics/astro-pages/). Membuat dan mengelola halaman untuk situs Anda akan terasa familiar.

- Astro dan VuePress sama‑sama dirancang untuk [situs berorientasi konten](/id/concepts/why-astro/#content-driven), dengan dukungan bawaan yang sangat baik untuk berkas Markdown. Menulis dalam Markdown akan terasa familiar, dan Anda akan dapat mempertahankan konten yang sudah ada.

- Astro memiliki [integrasi resmi untuk menggunakan komponen Vue](/id/guides/integrations-guide/vue/) dan mendukung [menginstal paket NPM](/id/guides/imports/#npm-packages), termasuk beberapa untuk Vue. Anda dapat menulis komponen UI Vue, dan mungkin dapat mempertahankan sebagian atau seluruh komponen dan dependensi Vue Anda yang sudah ada.

## Perbedaan Utama antara VuePress dan Astro

Saat Anda membangun ulang situs VuePress di Astro, Anda akan melihat beberapa perbedaan penting.

- VuePress adalah SPA (*single‑page application*) berbasis Vue. Situs Astro adalah aplikasi multi‑halaman yang dibangun menggunakan [komponen `.astro`](/id/basics/astro-components/), tetapi juga dapat mendukung [React, Preact, Vue.js, Svelte, SolidJS, AlpineJS](/id/guides/framework-components/) dan templating HTML murni.

- [Template *layout*](/id/basics/layouts/): Situs VuePress dibuat menggunakan berkas Markdown (`.md`) untuk konten halaman dan template HTML (`.html`) untuk *layout*. Astro berbasis komponen, dan menggunakan komponen Astro, yang mencakup templating HTML untuk halaman, *layout*, dan elemen UI individual. Astro juga dapat membuat [halaman dari berkas `.md` dan `.mdx`](/id/guides/markdown-content/), menggunakan komponen *layout* Astro untuk membungkus konten Markdown dalam template halaman.

- VuePress dirancang untuk membangun situs yang kaya konten dan berpusat pada Markdown, dan memiliki beberapa fitur khusus dokumentasi yang perlu Anda bangun sendiri di Astro. Sebagai gantinya, Astro menawarkan sejumlah fitur khusus dokumentasi melalui [tema dokumentasi resmi](https://starlight.astro.build). Situs ini adalah inspirasi untuk template tersebut! Anda juga dapat menemukan lebih banyak [tema dokumentasi komunitas](https://astro.build/themes?search=&categories%5B%5D=docs) dengan fitur bawaan di *Themes Showcase* kami.

## Beralih dari VuePress ke Astro

Untuk mengonversi situs dokumentasi VuePress ke Astro, mulailah dengan [starter template tema dokumentasi Starlight](https://starlight.astro.build) resmi kami, atau jelajahi lebih banyak tema dokumentasi komunitas di [theme showcase](https://astro.build/themes?search=&categories%5B%5D=docs). 

Anda dapat menambahkan argumen `--template` ke perintah `create astro` untuk memulai proyek Astro baru dengan salah satu *starter* resmi. Atau, Anda dapat [memulai proyek baru dari repositori Astro mana pun di GitHub](/id/install-and-setup/#install-from-the-cli-wizard).

  <PackageManagerTabs>
    <Fragment slot="npm">
    ```shell
    npm create astro@latest -- --template starlight
    ```
    </Fragment>
    <Fragment slot="pnpm">
    ```shell
    pnpm create astro@latest --template starlight

    <Fragment slot="yarn">

    yarn create astro --template starlight
    ```
    </Fragment>
  </PackageManagerTabs>

Bawa berkas konten Markdown Anda untuk [membuat halaman Markdown](/id/guides/markdown-content/). Anda tetap dapat memanfaatkan [routing berbasis file](/id/guides/routing/) dengan memindahkan dokumen ini dari `docs` di VuePress ke `src/pages/` di Astro. Buat folder dengan nama yang sesuai dengan proyek VuePress Anda yang ada, dan Anda seharusnya dapat mempertahankan URL yang sama.

VuePress, atau tema apa pun yang Anda pasang, mungkin menangani banyak *layout* dan metadata situs untuk Anda. Anda mungkin ingin membaca tentang [membangun *Layout* Astro sebagai *wrapper* halaman Markdown](/id/basics/layouts/#markdown-layouts) untuk melihat bagaimana mengelola templating sendiri di Astro, termasuk `<head>` halaman Anda.

Anda dapat menemukan *starter* dokumentasi Astro, dan template lainnya, di [astro.new](https://astro.new). Anda akan menemukan tautan ke setiap repositori GitHub proyek, serta tautan sekali klik untuk membuka proyek yang berfungsi di lingkungan pengembangan daring seperti IDX, StackBlitz, CodeSandbox, dan Gitpod.
