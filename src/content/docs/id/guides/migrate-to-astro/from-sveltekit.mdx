[SvelteKit](https://kit.svelte.dev) adalah kerangka kerja untuk membangun aplikasi web di atas Svelte.

## Persamaan Utama antara SvelteKit dan Astro

SvelteKit dan Astro memiliki beberapa kesamaan yang akan membantu Anda memigrasikan proyek Anda:

- Baik SvelteKit maupun Astro adalah *static‑site generator* JavaScript modern dan juga mendukung *server‑side rendering* (SSR). 

- Keduanya menggunakan [folder `src/` untuk berkas proyek](/id/basics/project-structure/#src) dan [folder khusus untuk routing berbasis file](/id/basics/astro-pages/). Membuat dan mengelola halaman untuk situs Anda akan terasa familiar.

- Astro memiliki [integrasi resmi untuk menggunakan komponen Svelte](/id/guides/integrations-guide/svelte/) dan mendukung [menginstal paket NPM](/id/guides/imports/#npm-packages), termasuk beberapa untuk Svelte. Anda dapat menulis komponen UI Svelte, dan mungkin dapat mempertahankan sebagian atau seluruh komponen serta dependensi yang sudah ada.

- Astro dan SvelteKit sama‑sama memungkinkan Anda menggunakan [headless CMS, API, atau berkas Markdown sebagai data](/id/guides/data-fetching/). Anda bisa terus menggunakan sistem penulisan konten pilihan Anda, dan tetap dapat mempertahankan konten yang sudah ada.

## Perbedaan Utama antara SvelteKit dan Astro

Saat Anda membangun ulang situs SvelteKit di Astro, Anda akan melihat beberapa perbedaan penting:

- Situs Astro adalah aplikasi multi‑halaman (MPA), sementara SvelteKit secara bawaan berorientasi SPA (*single‑page application*) dengan SSR, namun juga dapat membuat MPA, SPA tradisional, atau mencampur beberapa teknik ini dalam satu aplikasi.

- [Komponen](/id/basics/astro-components/): SvelteKit menggunakan [Svelte](https://svelte.dev). Halaman Astro dibangun menggunakan [komponen `.astro`](/id/basics/astro-components/), tetapi juga dapat mendukung [React, Preact, Vue.js, Svelte, SolidJS, AlpineJS](/id/guides/framework-components/) dan templating HTML murni.

- [content-driven](/id/concepts/why-astro/#content-driven): Astro dirancang untuk menonjolkan konten Anda dan memungkinkan *opt‑in* interaktivitas hanya saat diperlukan. Aplikasi SvelteKit yang ada mungkin dibangun dengan interaktivitas sisi klien yang tinggi. Astro memiliki kemampuan bawaan untuk bekerja dengan konten Anda, seperti pembuatan halaman, namun mungkin memerlukan teknik Astro tingkat lanjut untuk menyertakan elemen yang lebih menantang direplikasi menggunakan komponen `.astro`, seperti *dashboard*.

- [Siap Markdown](/id/guides/markdown-content/): Astro menyertakan dukungan Markdown bawaan, dan memiliki [properti frontmatter YAML khusus `layout`](/id/basics/layouts/#markdown-layouts) yang digunakan per berkas untuk templating halaman. Jika Anda mengonversi blog berbasis Markdown di SvelteKit, Anda tidak perlu memasang integrasi Markdown terpisah dan Anda tidak akan mengatur *layout* melalui berkas konfigurasi. Anda dapat membawa berkas Markdown Anda yang ada, tetapi mungkin perlu menata ulang karena routing berbasis file Astro tidak memerlukan folder untuk setiap rute halaman.

## Beralih dari SvelteKit ke Astro

Untuk mengonversi blog SvelteKit ke Astro, mulailah dengan *starter template* tema blog kami, atau jelajahi lebih banyak tema blog komunitas di [theme showcase](https://astro.build/themes/).

Anda dapat menambahkan argumen `--template` ke perintah `create astro` untuk memulai proyek Astro baru dengan salah satu *starter* resmi. Atau, Anda dapat [memulai proyek baru dari repositori Astro mana pun di GitHub](/id/install-and-setup/#install-from-the-cli-wizard).

  <PackageManagerTabs>
    <Fragment slot="npm">
    ```shell
    npm create astro@latest -- --template blog
    ```
    </Fragment>
    <Fragment slot="pnpm">
    ```shell
    pnpm create astro@latest --template blog
    ```
    </Fragment>
    <Fragment slot="yarn">
    ```shell
    yarn create astro --template blog
    ```
    </Fragment>
  </PackageManagerTabs>

Bawa berkas Markdown (atau MDX, dengan integrasi opsional kami) yang sudah ada sebagai konten untuk [membuat halaman Markdown atau MDX](/id/guides/markdown-content/).

Walaupun routing berbasis file dan komponen *layout* mirip di Astro, Anda mungkin ingin membaca tentang [struktur proyek Astro](/id/basics/project-structure/) untuk mempelajari lokasi berkas yang tepat.

Untuk mengonversi jenis situs lainnya, seperti portofolio atau situs dokumentasi, lihat lebih banyak *starter template* resmi di [astro.new](https://astro.new). Anda akan menemukan tautan ke setiap repositori GitHub proyek, serta tautan sekali klik untuk membuka proyek yang berfungsi di lingkungan pengembangan daring seperti IDX, StackBlitz, CodeSandbox, dan Gitpod.
