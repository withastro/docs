[Jekyll](https://jekyllrb.com) adalah static site generator yang dibangun dengan Ruby.

## Persamaan Utama antara Jekyll dan Astro

Jekyll dan Astro memiliki beberapa kesamaan yang akan membantu Anda memigrasikan proyek Anda:

- Baik Jekyll maupun Astro adalah static-site generator yang umum digunakan untuk membuat blog.

- Baik Jekyll maupun Astro memungkinkan Anda menulis konten dalam Markdown dan HTML. Keduanya menyediakan beberapa properti frontmatter YAML khusus untuk layout halaman dan draf postingan yang belum diterbitkan. Anda dapat terus menggunakan berkas Markdown yang sudah ada di Astro.

- Baik Jekyll maupun Astro menggunakan [routing berbasis file](/id/guides/routing/) untuk membuat halaman dari postingan blog Anda. Astro menyediakan [direktori khusus `src/pages/` untuk semua halaman dan postingan](/id/basics/project-structure/#srcpages). Jekyll menggunakan folder khusus bernama `_posts/` untuk postingan blog Markdown Anda, namun halaman situs Anda bisa berada di tempat lain. Membuat postingan blog baru akan terasa familiar.


## Perbedaan Utama antara Jekyll dan Astro

Saat Anda membangun ulang situs Jekyll di Astro, Anda akan melihat beberapa perbedaan penting:

- Karena Jekyll terutama merupakan platform blogging, beberapa fitur blog sudah tersedia bawaan dan mungkin harus Anda bangun sendiri di Astro. Atau, pilih [tema starter template blog](https://astro.build/themes?search=&categories%5B%5D=blog) yang menyertakan fitur-fitur ini. Sebagai contoh, Jekyll memiliki dukungan bawaan untuk tag dan kategori yang bisa Anda temukan di beberapa tema blog Astro, tetapi tidak termasuk dalam proyek Astro minimal.

- Jekyll menggunakan template Liquid untuk elemen layout yang dapat digunakan kembali dan templating. Astro menggunakan berkas [`.astro` yang mirip JSX untuk templating dan komponen](/id/basics/astro-components/). Setiap berkas `.astro` dapat menjadi komponen, layout, atau halaman penuh, dan dapat mengimpor serta merender komponen Astro lainnya. Anda juga dapat membangun menggunakan [komponen kerangka UI lain (mis. React, Svelte, Vue, Solid)](/id/guides/framework-components/) serta konten atau metadata dari [berkas lain di proyek Anda](/id/guides/imports/), seperti Markdown atau MDX.


## Beralih dari Jekyll ke Astro

Untuk mengonversi blog Jekyll ke Astro, mulailah dengan starter template tema blog kami, atau jelajahi lebih banyak tema blog komunitas di [theme showcase](https://astro.build/themes/).

Anda dapat menambahkan argumen `--template` ke perintah `create astro` untuk memulai proyek Astro baru dengan salah satu starter resmi kami. Atau, Anda dapat [memulai proyek baru dari repositori Astro mana pun di GitHub](/id/install-and-setup/#install-from-the-cli-wizard).

  <PackageManagerTabs>
    <Fragment slot="npm">
    ```shell
    npm create astro@latest -- --template blog
    ```
    </Fragment>
    <Fragment slot="pnpm">
    ```shell
    pnpm create astro@latest --template blog
    ```
    </Fragment>
    <Fragment slot="yarn">
    ```shell
    yarn create astro --template blog
    ```
    </Fragment>
  </PackageManagerTabs>

Bawa berkas Markdown Anda yang sudah ada sebagai konten untuk [membuat halaman Markdown](/id/guides/markdown-content/), menggunakan [layout Markdown Astro](/id/basics/layouts/#markdown-layouts) alih-alih template Liquid. 

Sebagian besar konten halaman HTML Anda yang ada dapat dikonversi menjadi [halaman Astro](/id/basics/astro-pages/), dan Anda juga akan dapat [menggunakan variabel, ekspresi mirip JSX, serta impor komponen langsung di templating HTML Anda](/id/reference/astro-syntax/#jsx-like-expressions).

Astro tidak memiliki properti `permalink` yang menerima placeholder. Anda mungkin perlu membaca lebih lanjut tentang [routing halaman Astro](/id/guides/routing/) jika ingin mempertahankan struktur URL yang sudah ada. Atau, pertimbangkan untuk [mengatur redirect di host seperti Netlify](https://docs.netlify.com/routing/redirects/).

Untuk mengonversi jenis situs lain, seperti portofolio atau situs dokumentasi, lihat lebih banyak starter template resmi di [astro.new](https://astro.new). Anda akan menemukan tautan ke setiap repositori GitHub proyek, serta tautan sekali klik untuk membuka proyek yang berfungsi di lingkungan pengembangan online seperti IDX, StackBlitz, CodeSandbox, dan Gitpod.
