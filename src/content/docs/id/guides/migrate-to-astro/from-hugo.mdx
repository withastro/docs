[Hugo](https://gohugo.io) adalah static site generator open-source yang dibangun dengan Go.

## Persamaan Utama antara Hugo dan Astro

Hugo dan Astro memiliki beberapa kesamaan yang akan membantu Anda memigrasikan proyek Anda:

- Hugo dan Astro keduanya merupakan static-site generator modern, sangat cocok untuk [situs berorientasi konten](/id/concepts/why-astro/#content-driven) seperti blog.

- Hugo dan Astro sama-sama memungkinkan Anda [menulis konten dalam Markdown](/id/guides/markdown-content/). Namun, Hugo menyertakan beberapa properti frontmatter khusus dan mengizinkan Anda menulis frontmatter dalam YAML, TOML, atau JSON. Meskipun banyak properti frontmatter Hugo yang ada tidak akan menjadi “khusus” di Astro, Anda tetap dapat menggunakan berkas Markdown dan nilai frontmatter YAML (atau TOML) yang sudah ada.

- Hugo dan Astro sama-sama memungkinkan Anda menambah fitur situs dengan berbagai [integrasi dan paket eksternal](https://astro.build/integrations/).



## Perbedaan Utama antara Hugo dan Astro

Saat Anda membangun ulang situs Hugo di Astro, Anda akan melihat beberapa perbedaan penting:

- Hugo menggunakan Go Templating untuk templating halaman. [Sintaks Astro](/id/basics/astro-components/) adalah superset HTML yang mirip JSX.

- Astro tidak menggunakan shortcode untuk konten dinamis di berkas Markdown standar, namun [integrasi MDX Astro](/id/guides/integrations-guide/mdx/) memungkinkan Anda menggunakan JSX dan mengimpor komponen di berkas MDX.

- Sementara Hugo dapat menggunakan “partial” untuk elemen layout yang dapat digunakan kembali, [Astro sepenuhnya berbasis komponen](/id/basics/astro-components/). Setiap berkas `.astro` dapat menjadi komponen, layout, atau halaman utuh, dan dapat mengimpor serta merender komponen Astro lainnya. Komponen Astro juga dapat menyertakan [komponen kerangka UI lain (mis. React, Svelte, Vue, Solid)](/id/guides/framework-components/) serta konten atau metadata dari [berkas lain di proyek Anda](/id/guides/imports/), seperti Markdown atau MDX.

## Beralih dari Hugo ke Astro

Untuk mengonversi blog Hugo ke Astro, mulailah dengan starter template tema blog kami, atau jelajahi lebih banyak tema blog komunitas di [theme showcase](https://astro.build/themes/).

Anda dapat menambahkan argumen `--template` ke perintah `create astro` untuk memulai proyek Astro baru dengan salah satu starter resmi kami. Atau, Anda dapat [memulai proyek baru dari repositori Astro mana pun di GitHub](/id/install-and-setup/#install-from-the-cli-wizard).

  <PackageManagerTabs>
    <Fragment slot="npm">
    ```shell
    npm create astro@latest -- --template blog
    ```
    </Fragment>
    <Fragment slot="pnpm">
    ```shell
    pnpm create astro@latest --template blog
    ```
    </Fragment>
    <Fragment slot="yarn">
    ```shell
    yarn create astro --template blog
    ```
    </Fragment>
  </PackageManagerTabs>

Bawa berkas Markdown (atau MDX, dengan integrasi opsional kami) Anda sebagai konten untuk [membuat halaman Markdown atau MDX](/id/guides/markdown-content/). Astro mengizinkan frontmatter YAML atau TOML di berkas-berkas ini, jadi jika Anda menggunakan frontmatter JSON, Anda perlu mengonversinya.

Untuk terus menggunakan konten dinamis seperti variabel, ekspresi, atau komponen UI di dalam konten Markdown Anda, tambahkan integrasi MDX Astro opsional dan konversi berkas Markdown yang sudah ada menjadi [halaman MDX](/id/guides/markdown-content/). MDX mendukung frontmatter YAML dan TOML, sehingga Anda dapat mempertahankan properti frontmatter yang sudah ada. Namun, Anda harus mengganti sintaks shortcode apa pun dengan [sintaks MDX](https://mdxjs.com/docs/what-is-mdx/#mdx-syntax), yang memungkinkan ekspresi JSX dan/atau impor komponen.

Untuk mengonversi jenis situs lain, seperti portofolio atau situs dokumentasi, lihat lebih banyak starter template resmi di [astro.new](https://astro.new). Anda akan menemukan tautan ke setiap repositori GitHub proyek, serta tautan sekali klik untuk membuka proyek yang berfungsi di lingkungan pengembangan online seperti IDX, StackBlitz, CodeSandbox, dan Gitpod.
