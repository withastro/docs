[Pelican](https://getpelican.com) adalah static site generator open-source yang dibangun dengan Python.

## Persamaan Utama antara Pelican dan Astro

Pelican dan Astro memiliki beberapa kesamaan yang akan membantu Anda memigrasikan proyek Anda:

- Pelican dan Astro sama‑sama static-site generator, sangat cocok untuk [situs berorientasi konten](/id/concepts/why-astro/#content-driven) seperti blog.

- Pelican dan Astro sama‑sama memiliki dukungan bawaan untuk [menulis dalam Markdown](/id/guides/markdown-content/), termasuk properti frontmatter YAML untuk metadata halaman. Namun, Astro memiliki sangat sedikit properti frontmatter yang dipesan dibandingkan Pelican. Meskipun banyak properti frontmatter Pelican Anda yang ada tidak akan menjadi "spesial" di Astro, Anda tetap dapat menggunakan berkas Markdown dan nilai frontmatter yang sudah ada.

## Perbedaan Utama antara Pelican dan Astro

Saat Anda membangun ulang situs Pelican di Astro, Anda akan melihat beberapa perbedaan penting:

- Pelican mendukung penulisan konten dalam Markdown dan reStructured Text (`.rst`). Astro mendukung [membuat halaman dari berkas Markdown dan MDX](/id/guides/markdown-content/), tetapi tidak mendukung reStructured Text.

- Pelican menggunakan berkas HTML dan sintaks Jinja untuk templating. [Sintaks Astro](/id/basics/astro-components/) adalah superset HTML yang mirip JSX. Semua HTML yang valid juga valid sebagai sintaks `.astro`.

- Pelican dirancang untuk membangun situs kaya konten seperti blog dan mempunyai beberapa fitur blog bawaan yang perlu Anda bangun sendiri di Astro. Sebagai gantinya, Astro menawarkan sebagian fitur ini di [tema blog resmi](https://github.com/withastro/astro/tree/latest/examples/blog).

## Beralih dari Pelican ke Astro

Untuk mengonversi situs dokumentasi Pelican ke Astro, mulailah dengan [starter template tema docs Starlight](https://starlight.astro.build) resmi kami, atau jelajahi lebih banyak tema komunitas di [theme showcase](https://astro.build/themes/). 

Anda dapat menambahkan argumen `--template` ke perintah `create astro` untuk memulai proyek Astro baru dengan salah satu starter resmi kami. Atau, Anda dapat [memulai proyek baru dari repositori Astro mana pun di GitHub](/id/install-and-setup/#install-from-the-cli-wizard).

  <PackageManagerTabs>
    <Fragment slot="npm">
    ```shell
    npm create astro@latest -- --template starlight
    ```
    </Fragment>
    <Fragment slot="pnpm">
    ```shell
    pnpm create astro@latest --template starlight
    ```
    </Fragment>
    <Fragment slot="yarn">
    ```shell
    yarn create astro --template starlight
    ```
    </Fragment>
  </PackageManagerTabs>

Bawa berkas konten Markdown Anda yang sudah ada untuk [membuat halaman Markdown](/id/guides/markdown-content/). Anda tetap dapat memanfaatkan [routing berbasis file](/id/guides/routing/) dengan menyalin dokumen ini dari folder `content/` milik Pelican ke `src/pages/` di Astro. Anda mungkin ingin membaca tentang [struktur proyek Astro](/id/basics/project-structure/) untuk mempelajari di mana berkas harus ditempatkan.

Pelican mungkin telah menangani banyak layout situs dan metadata untuk Anda. Anda mungkin ingin membaca tentang [membangun Layout Astro sebagai *wrapper* halaman Markdown](/id/basics/layouts/#markdown-layouts) untuk melihat bagaimana mengelola templating sendiri di Astro, termasuk `<head>` halaman Anda.

Seperti Pelican, Astro memiliki banyak plugin yang memperluas fungsionalitasnya. Jelajahi [daftar integrasi resmi](/id/guides/integrations-guide/) untuk menambahkan fitur seperti dukungan MDX, dan temukan ratusan integrasi yang dikelola komunitas di [Direktori Integrasi Astro](https://astro.build/integrations/). Anda bahkan dapat menggunakan [Astro Integration API](/id/reference/integrations-reference/) untuk membangun integrasi kustom Anda sendiri guna memperluas fitur proyek.

Untuk mengonversi jenis situs lain, seperti portofolio atau blog, lihat lebih banyak starter template resmi di [astro.new](https://astro.new). Anda akan menemukan tautan ke setiap repositori GitHub proyek, serta tautan sekali klik untuk membuka proyek yang berfungsi di lingkungan pengembangan online seperti IDX, StackBlitz, CodeSandbox, dan Gitpod.
