## Persamaan Utama antara Next.js dan Astro

Next.js dan Astro memiliki beberapa kesamaan yang akan membantu Anda memigrasikan proyek Anda:

- [Sintaks file `.astro` mirip dengan JSX](/id/reference/astro-syntax/#differences-between-astro-and-jsx). Menulis Astro akan terasa familiar.
- Proyek Astro juga dapat berupa SSG atau [SSR dengan prerender tingkat halaman](/id/guides/on-demand-rendering/).
- Astro menggunakan routing berbasis file, dan [mengizinkan halaman bernama khusus untuk membuat rute dinamis](/id/guides/routing/#dynamic-routes).
- Astro [berbasis komponen](/id/basics/astro-components/), dan struktur markup Anda akan serupa sebelum dan sesudah migrasi.
- Astro memiliki [integrasi resmi untuk React, Preact, dan Solid](/id/guides/integrations-guide/react/) sehingga Anda dapat menggunakan komponen JSX yang sudah ada. Perhatikan bahwa di Astro, berkas-berkas ini **harus** berekstensi `.jsx` atau `.tsx`.
- Astro mendukung [memasang paket NPM](/id/guides/imports/#npm-packages), termasuk pustaka React. Banyak dependensi Anda saat ini akan berfungsi di Astro.

## Perbedaan Utama antara Next.js dan Astro

Saat Anda membangun ulang situs Next.js di Astro, Anda akan melihat beberapa perbedaan penting:

- Next.js adalah aplikasi React satu halaman, dan menggunakan `index.js` sebagai root proyek Anda. Astro adalah situs multi-halaman, dan `index.astro` adalah halaman beranda Anda.

- [Komponen `.astro`](/id/basics/astro-components/) tidak ditulis sebagai fungsi diekspor yang mengembalikan templating halaman. Sebagai gantinya, Anda akan membagi kode menjadi ‚Äúpagar kode‚Äù (code fence) untuk JavaScript dan body khusus untuk HTML yang Anda hasilkan.

- [content-driven](/id/concepts/why-astro/#content-driven): Astro dirancang untuk menonjolkan konten Anda dan memungkinkan Anda *opt-in* interaktivitas hanya saat diperlukan. Aplikasi Next.js yang ada mungkin dibangun untuk interaktivitas sisi klien yang tinggi dan dapat memerlukan teknik Astro yang lebih lanjut untuk menyertakan item yang lebih menantang direplikasi menggunakan komponen `.astro`, seperti dashboard.

## Konversi Proyek Next.js Anda

Setiap migrasi proyek akan terlihat berbeda, tetapi ada beberapa langkah umum yang akan Anda lakukan saat beralih dari Next.js ke Astro.

### Pasang integrasi (opsional)

Anda mungkin ingin memasang beberapa [integrasi opsional Astro](/id/guides/integrations-guide/) untuk digunakan saat mengonversi proyek Next Anda ke Astro:

- **@astrojs/react**: untuk menggunakan kembali beberapa komponen UI React yang ada di situs Astro baru Anda, atau terus menulis dengan komponen React.

- **@astrojs/mdx**: untuk membawa berkas MDX yang sudah ada dari proyek Next Anda, atau untuk menggunakan MDX di situs Astro baru Anda.

### Tempatkan kode sumber Anda di `src`

Mengikuti [struktur proyek Astro](/id/basics/project-structure/):

<Steps>
1. **Biarkan** folder `public/` milik Next apa adanya. 
   
    Astro menggunakan direktori `public/` untuk aset statis, sama seperti Next. Tidak diperlukan perubahan pada folder ini maupun isinya.

2. **Salin atau Pindahkan** berkas dan folder Next lainnya (mis. `pages`, `styles`, dll.) ke folder `src/` milik Astro saat Anda membangun ulang situs Anda, mengikuti [struktur proyek Astro](/id/basics/project-structure/).

    Seperti Next, folder `src/pages/` milik Astro adalah folder khusus yang digunakan untuk routing berbasis file. Folder lainnya bersifat opsional, dan Anda dapat mengatur isi folder `src/` sesuai keinginan. Folder umum lain di proyek Astro termasuk `src/layouts/`, `src/components`, `src/styles`, `src/scripts`.
</Steps>

### Berkas konfigurasi Astro

Astro memiliki berkas konfigurasi di root proyek Anda bernama [`astro.config.mjs`](/id/guides/configuring-astro/). Ini hanya digunakan untuk mengonfigurasi proyek Astro Anda dan integrasi apa pun yang terpasang, termasuk [adapter SSR](/id/guides/deploy/).

### Tips: Konversi berkas JSX menjadi berkas `.astro`

Berikut beberapa kiat untuk mengonversi komponen `.js` Next menjadi komponen `.astro`:

1. Gunakan JSX yang dikembalikan oleh fungsi komponen Next.js yang ada sebagai dasar template HTML Anda.

2. Ubah [sintaks Next atau JSX ke Astro](#reference-convert-nextjs-syntax-to-astro) atau ke standar web HTML. Ini mencakup `<Link>`, `<Script>`, `{children}`, dan `className`, misalnya.

3. Pindahkan JavaScript yang diperlukan, termasuk pernyataan impor, ke dalam [‚Äúpagar kode‚Äù (`---`)](/id/basics/astro-components/#the-component-script). Catatan: JavaScript untuk [merender konten secara kondisional](/id/reference/astro-syntax/#dynamic-html) sering kali ditulis langsung di dalam template HTML di Astro.

4. Gunakan [`Astro.props`](/id/reference/api-reference/#props) untuk mengakses properti tambahan apa pun yang sebelumnya diteruskan ke fungsi Next Anda.

5. Putuskan apakah komponen yang diimpor juga perlu diubah ke Astro. Dengan integrasi resmi terpasang, Anda dapat [menggunakan komponen React yang ada di berkas Astro](/id/guides/framework-components/). Namun, Anda mungkin ingin mengonversinya menjadi komponen `.astro`, terutama jika tidak perlu interaktif!

6. Ganti `getStaticProps()` dengan pernyataan impor atau [`import.meta.glob()`](/id/guides/imports/#importmetaglob) untuk menanyakan berkas lokal Anda. Gunakan `fetch()` untuk mengambil data eksternal.

Lihat [contoh berkas Next `.js` yang dikonversi langkah demi langkah](#guided-example-next-data-fetching-to-astro).

### Migrasi Berkas Layout

Anda mungkin ingin memulai dengan mengonversi layout dan template Next.js Anda menjadi [komponen layout Astro](/id/basics/layouts/).

Next memiliki dua metode berbeda untuk membuat berkas layout, yang masing-masing menangani layout dengan cara berbeda dari Astro:

- Direktori `pages`

- [Direktori `/app`](https://nextjs.org/docs/app/building-your-application/routing/pages-and-layouts#layouts)

Setiap halaman Astro secara eksplisit memerlukan tag `<html>`, `<head>`, dan `<body>`, sehingga umum untuk menggunakan ulang berkas layout di banyak halaman. Astro menggunakan [`<slot />`](/id/basics/astro-components/#slots) untuk konten halaman, tanpa perlu pernyataan impor. Perhatikan templating HTML standar, dan akses langsung ke `<head>`:  

```astro title="src/layouts/Layout.astro"
---
---
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <link rel="icon" type="image/svg+xml" href="/favicon.svg" />
    <meta name="viewport" content="width=device-width" />
    <meta name="generator" content={Astro.generator} />
    <title>Astro</title>
  </head>
  <body>
    <!-- Bungkus elemen slot dengan templating layout yang sudah ada -->
    <slot />
  </body>
</html]
```

#### Migrasi dari direktori `pages` milik Next.js

Proyek Next Anda mungkin memiliki berkas `pages/_document.jsx` yang mengimpor komponen React untuk menyesuaikan `<head>` aplikasi Anda:

```jsx title="pages/_document.jsx"
import Document, { Html, Head, Main, NextScript } from "next/document";

export default class MyDocument extends Document {
  render() {
    return (
      <Html lang="en">
        <Head>
          <link rel="icon" href="/favicon.ico" />
        </Head>
        <body>
          <Main />
          <NextScript />
        </body>
      </Html>
    );
  }
}
```
<Steps>
1. Buat berkas layout Astro baru menggunakan hanya JSX hasil return.

2. Ganti komponen React dengan tag HTML standar `<html>`, `<head>`, `<slot>`, dan lainnya.

    ```astro title="src/layouts/Document.astro"
    <html lang="en">
      <head>
          <link rel="icon" href="/favicon.ico" />
      </head>
      <body>
        <slot/>
      </body>
    </html>
    ```
</Steps>

#### Migrasi dari direktori `/app` milik Next.js

Berkas layout direktori `app/` Next.js dibuat dengan dua berkas: `layout.jsx` untuk menyesuaikan konten `<html>` dan `<body>`, serta `head.jsx` untuk menyesuaikan konten elemen `<head>`.

```jsx title="app/layout.jsx"
export default function Layout({ children }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  );
}
```

```jsx title="app/head.jsx"
export default function Head() {
  return (
    <>
      <title>My Page</title>
    </>
  );
}
```

<Steps>
1. Buat berkas layout Astro baru menggunakan hanya JSX hasil return.

2. Ganti kedua berkas ini dengan satu berkas layout Astro yang berisi kerangka halaman (tag `<html>`, `<head>`, dan `<body>`) serta `<slot/>` sebagai pengganti properti `{children}` milik React:

    ```astro title="src/layouts/Layout.astro"
    <html lang="en">
      <head>
          <title>My Page</title>
      </head>
      <body>
        <slot/>
      </body>
    </html>
    ```
</Steps>

### Migrasi Halaman dan Postingan

Di Next.js, postingan Anda berada di `/pages` atau di `/app/routeName/page.jsx`.

Di Astro, semua konten halaman Anda harus berada di dalam `src/` kecuali Anda menggunakan [content collections](/id/guides/content-collections/).

#### Halaman React

Halaman JSX (`.js`) Next yang ada harus [dikoversi dari berkas JSX menjadi halaman `.astro`](#tips-convert-jsx-files-to-astro-files). Anda tidak dapat menggunakan berkas halaman JSX yang ada di Astro.

[Halaman `.astro`](/id/basics/astro-pages/) ini harus berada di dalam `src/pages/` dan akan memiliki rute halaman yang dibuat secara otomatis berdasarkan path berkasnya.

#### Halaman Markdown dan MDX

Astro memiliki dukungan bawaan untuk Markdown dan integrasi opsional untuk berkas MDX. Anda dapat menggunakan kembali [berkas Markdown dan MDX](/id/guides/markdown-content/) apa pun yang sudah ada, tetapi mungkin memerlukan beberapa penyesuaian pada frontmatter-nya, seperti menambahkan [properti frontmatter khusus Astro `layout`](/id/basics/layouts/#markdown-layouts). Anda tidak lagi perlu membuat halaman secara manual untuk setiap rute yang dihasilkan oleh Markdown. Berkas-berkas ini dapat ditempatkan di dalam `src/pages/` untuk memanfaatkan routing berbasis file otomatis.

Sebagai alternatif, Anda dapat menggunakan [content collections](/id/guides/content-collections/) di Astro untuk menyimpan dan mengelola konten Anda. Anda akan mengambil konten sendiri dan [menghasilkan halaman tersebut secara dinamis](/id/guides/content-collections/#generating-routes-from-content).

### Migrasi Pengujian

Karena Astro menghasilkan HTML mentah, memungkinkan untuk menulis pengujian end-to-end menggunakan output dari langkah build. Pengujian end-to-end yang ditulis sebelumnya mungkin berfungsi langsung jika Anda mampu mencocokkan markup situs Next Anda. Pustaka pengujian seperti Jest dan React Testing Library dapat diimpor dan digunakan di Astro untuk menguji komponen React Anda.

Lihat [panduan pengujian](/id/guides/testing/) Astro untuk selengkapnya.

## Referensi: Konversi Sintaks Next.js ke Astro

### Next Links ke Astro

Ubah komponen Next `<Link to="">`, `<NavLink>`, dll. menjadi tag HTML `<a href="">`. 

```astro del={1} ins={2}
<Link to="/blog">Blog</Link>
<a href="/blog">Blog</a>
```

Astro tidak menggunakan komponen khusus untuk tautan, meskipun Anda dipersilakan untuk membuat komponen `<Link>` sendiri. Anda kemudian dapat mengimpor dan menggunakan `<Link>` ini sebagaimana komponen lainnya.

```astro title="src/components/Link.astro"
---
const { to } = Astro.props;
---
<a href={to}><slot /></a>
```

### Impor Next ke Astro

Perbarui [impor berkas](/id/guides/imports/) agar merujuk ke jalur berkas relatif secara tepat. Ini dapat dilakukan menggunakan [alias impor](/id/guides/typescript/#import-aliases), atau dengan menuliskan jalur relatif secara penuh. 

Perhatikan bahwa `.astro` dan beberapa jenis berkas lainnya harus diimpor dengan ekstensi berkas lengkapnya.

```astro title="src/pages/authors/Fred.astro"
---
import Card from "../../components/Card.astro";
---
<Card />
```

### Properti Children Next ke Astro

Ubah setiap instance `{children}` menjadi `<slot />` Astro. Astro tidak perlu menerima `{children}` sebagai properti fungsi dan akan secara otomatis merender konten anak di `<slot />`.

```astro title="src/components/MyComponent.astro" del={3-9} ins={11-13}
---
---
export default function MyComponent(props) { 
    return (
      <div>
        {props.children}
      </div>
    );  
}

<div>
  <slot />
</div>
```

Komponen React yang meneruskan beberapa set children dapat dimigrasikan ke komponen Astro menggunakan [slot bernama](/id/basics/astro-components/#named-slots).

Lihat lebih lanjut tentang [penggunaan `<slot />` spesifik di Astro](/id/basics/astro-components/#slots).

### Pengambilan Data Next ke Astro

Ubah setiap instance `getStaticProps()` menjadi `import.meta.glob()` atau `getCollection()`/`getEntry()` untuk mengakses data dari berkas lain di sumber proyek Anda. Untuk [mengambil data jarak jauh](/id/guides/data-fetching/), gunakan `fetch()`.

Permintaan data ini dibuat di frontmatter komponen Astro dan menggunakan *top-level await*.

```astro title="src/pages/index.astro"
---
import { getCollection } from 'astro:content';

// Ambil semua entri `src/content/blog/`
const allBlogPosts = await getCollection('blog');

// Ambil semua entri `src/pages/posts/`
const allPosts = Object.values(import.meta.glob('../pages/posts/*.md', { eager: true }));

const response = await fetch('https://randomuser.me/api/');
const data = await response.json();
const randomUser = data.results[0];
---
```

Lihat lebih lanjut tentang impor berkas lokal dengan [`import.meta.glob()`](/id/guides/imports/#importmetaglob), [query menggunakan Collections API](/id/guides/content-collections/#querying-collections) atau [mengambil data jarak jauh](/id/guides/data-fetching/).

### Penataan Gaya Next ke Astro

Anda mungkin perlu mengganti [pustaka CSS-in-JS](https://github.com/withastro/astro/issues/4432) (mis. styled-components) dengan opsi CSS lain yang tersedia di Astro.

Jika perlu, konversi objek gaya inline (`style={{ fontWeight: "bold" }}`) menjadi atribut gaya inline HTML (`style="font-weight:bold;"`). Atau, gunakan [tag `<style>` Astro](/id/guides/styling/#styling-in-astro) untuk gaya CSS berskala komponen.

```astro title="src/components/Card.astro" del={1} ins={2}
<div style={{backgroundColor: `#f4f4f4`, padding: `1em`}}>{message}</div>
<div style="background-color: #f4f4f4; padding: 1em;">{message}</div>
```

Tailwind didukung setelah memasang [plugin Vite Tailwind](/id/guides/styling/#tailwind). Tidak diperlukan perubahan pada kode Tailwind Anda yang sudah ada!

Lihat lebih lanjut tentang [Styling di Astro](/id/guides/styling/).

### Plugin Gambar Next ke Astro

Ubah komponen Next `<Image />` apa pun menjadi [komponen gambar milik Astro](/id/guides/images/) di berkas `.astro` atau `.mdx`, atau menjadi [tag HTML `<img>` standar / JSX `<img />`](/id/guides/images/#images-in-ui-framework-components) sesuai kebutuhan di komponen React Anda. 

Komponen `<Image />` milik Astro bekerja di berkas `.astro` dan `.mdx` saja. Lihat [daftar lengkap atribut komponennya](/id/reference/modules/astro-assets/#image-properties) dan perhatikan bahwa beberapa akan berbeda dari atribut milik Next. 

```astro title="src/pages/index.astro"
---
import { Image } from 'astro:assets';
import rocket from '../assets/rocket.png';
---
<Image src={rocket} alt="A rocketship in space." />
<img src={rocket.src} alt="A rocketship in space.">
```

Di komponen React (`.jsx`), gunakan sintaks gambar JSX standar (`<img />`). Astro tidak akan mengoptimalkan gambar-gambar ini, tetapi Anda dapat memasang paket NPM untuk fleksibilitas lebih lanjut.

Anda dapat mempelajari lebih lanjut tentang [menggunakan gambar di Astro](/id/guides/images/) di Images Guide.

## Contoh terpandu: pengambilan data Next ke Astro

Berikut contoh pengambilan data Pok√©dex di Next.js yang dikonversi ke Astro.
    
`pages/index.js` mengambil dan menampilkan daftar 151 Pok√©mon pertama menggunakan [REST Pok√©API](https://pokeapi.co/).

Berikut cara membuat ulang itu di `src/pages/index.astro`, mengganti `getStaticProps()` dengan `fetch()`.

<Steps>
1. Identifikasi JSX hasil return().

    ```jsx title="pages/index.js" {6-18}
    import Link from 'next/link'
    import styles from '../styles/poke-list.module.css';

    export default function Home({ pokemons }) {
        return (
            <>
                <ul className={`plain-list ${styles.pokeList}`}>
                    {pokemons.map((pokemon) => (
                        <li className={styles.pokemonListItem} key={pokemon.name}>
                            <Link className={styles.pokemonContainer} as={`/pokemon/${pokemon.name}`} href="/pokemon/[name]">
                                <p className={styles.pokemonId}>No. {pokemon.id}</p>
                                <img className={styles.pokemonImage} src={`https://raw.githubusercontent.com/PokeAPI/sprites/master/sprites/pokemon/${pokemon.id}.png`} alt={`${pokemon.name} picture`}></img>
                                <h2 className={styles.pokemonName}>{pokemon.name}</h2>
                            </Link>
                        </li>
                    ))}
                </ul>
            </>
        )
    }

    export const getStaticProps = async () => {
        const res = await fetch("https://pokeapi.co/api/v2/pokemon?limit=151")
        const resJson = await res.json();
        const pokemons = resJson.results.map(pokemon => {
            const name = pokemon.name;
            // https://pokeapi.co/api/v2/pokemon/1/
            const url = pokemon.url;
            const id = url.split("/")[url.split("/").length - 2];
            return {
                name,
                url,
                id
            }
        });
        return {
            props: {
                pokemons,
            },
        }
    }
    ```

2. Buat `src/pages/index.astro`

    Gunakan nilai return dari fungsi Next. Konversi sintaks Next atau React apa pun ke Astro, termasuk mengubah penulisan huruf pada [atribut global HTML](https://developer.mozilla.org/en-US/docs/Web/HTML/Global_attributes).

    Perhatikan bahwa:

    - `.map` tetap berfungsi!

    - `className` menjadi `class`.

    - `<Link>` menjadi `<a>`.

    - Fragmen `<> </>` tidak diperlukan di templating Astro.

    - `key` adalah atribut React, dan bukan atribut `li` di Astro.

    ```astro title="src/pages/index.astro" "class" "</a>" "<a"
    ---
    ---
    <ul class="plain-list pokeList">
        {pokemons.map((pokemon) => (
            <li class="pokemonListItem">
                <a class="pokemonContainer" href={`/pokemon/${pokemon.name}`}>
                    <p class="pokemonId">No. {pokemon.id}</p>
                    <img class="pokemonImage" src={`https://raw.githubusercontent.com/PokeAPI/sprites/master/sprites/pokemon/${pokemon.id}.png`} alt={`${pokemon.name} picture`}/>
                    <h2 class="pokemonName">{pokemon.name}</h2>
                </a>
            </li>
        ))}
    </ul>
    ```

3. Tambahkan impor, props, dan JavaScript yang diperlukan

    Perhatikan bahwa:

    - Fungsi `getStaticProps` tidak lagi diperlukan. Data dari API diambil langsung di pagar kode.
    - Komponen `<Layout>` diimpor dan membungkus templating halaman.

    ```astro ins={2,4-16,19,31} title="src/pages/index.astro"
    ---
    import Layout from '../layouts/layout.astro';

    const res = await fetch("https://pokeapi.co/api/v2/pokemon?limit=151");
    const resJson = await res.json();
    const pokemons = resJson.results.map(pokemon => {
        const name = pokemon.name;
        // https://pokeapi.co/api/v2/pokemon/1/
        const url = pokemon.url;
        const id = url.split("/")[url.split("/").length - 2];
        return {
            name,
            url,
            id
        }
    });
    ---

    <Layout>
      <ul class="plain-list pokeList">
          {pokemons.map((pokemon) => (
              <li class="pokemonListItem" key={pokemon.name}>
                  <a class="pokemonContainer" href={`/pokemon/${pokemon.name}`}>
                      <p class="pokemonId">No. {pokemon.id}</p>
                      <img class="pokemonImage" src={`https://raw.githubusercontent.com/PokeAPI/sprites/master/sprites/pokemon/${pokemon.id}.png`} alt={`${pokemon.name} picture`}/>
                      <h2 class="pokemonName">{pokemon.name}</h2>
                  </a>
              </li>
          ))}
      </ul>
    </Layout>
    ```
