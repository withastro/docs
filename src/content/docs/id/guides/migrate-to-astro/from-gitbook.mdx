[GitBook](https://gitbook.com) adalah platform berbasis web untuk membuat dan menerbitkan dokumentasi serta buku secara kolaboratif, dengan integrasi kontrol versi dan fitur yang dapat disesuaikan.


## Persamaan Utama antara GitBook dan Astro

GitBook dan Astro memiliki beberapa kesamaan yang akan membantu Anda memigrasikan proyek Anda:

- Baik Astro maupun GitBook mendukung [Markdown](/id/guides/markdown-content/). Anda dapat memigrasikan seluruh dokumentasi yang sudah ada dengan memanfaatkan fitur Git Sync milik GitBook.

- Baik Astro maupun GitBook menggunakan bentuk [routing berbasis file](/id/guides/routing/). Menggunakan struktur berkas Astro untuk konten yang sudah ada dan saat menambahkan halaman baru akan terasa familiar.

## Perbedaan Utama antara GitBook dan Astro

Saat Anda memigrasikan dokumentasi GitBook ke Astro, Anda akan melihat beberapa perbedaan penting:

- Situs GitBook diedit menggunakan dasbor online. Di Astro, Anda akan menggunakan [editor kode](/id/editor-setup/) dan lingkungan pengembangan untuk memelihara situs Anda. Anda bisa mengembangkan secara lokal di mesin Anda, atau memilih editor/lingkungan pengembangan cloud seperti IDX, StackBlitz, CodeSandbox, atau Gitpod.

- GitBook menyimpan konten Anda di dalam basis data. Di Astro, Anda akan memiliki berkas-berkas individual (biasanya Markdown atau MDX) di [direktori proyek](/id/basics/project-structure/) untuk konten setiap halaman. Atau, Anda dapat memilih untuk menggunakan [CMS untuk konten Anda](/id/guides/cms/) dan menggunakan Astro untuk mengambil serta menampilkan datanya.

- GitBook menggunakan sintaks khusus di atas Markdown untuk konten. Astro mendukung Markdoc melalui [integrasi Markdoc](/id/guides/integrations-guide/markdoc/) opsional, yang memiliki sintaks serupa dengan yang Anda gunakan di GitBook.


## Beralih dari GitBook ke Astro

Untuk mengonversi situs dokumentasi GitBook ke Astro, mulailah dengan [starter template tema dokumentasi Starlight](https://starlight.astro.build) resmi kami, atau jelajahi lebih banyak tema dokumentasi komunitas di [theme showcase](https://astro.build/themes?search=&categories%5B%5D=docs).

Anda dapat menambahkan argumen `--template` ke perintah `create astro` untuk memulai proyek Astro baru dengan salah satu starter resmi kami. Atau, Anda dapat [memulai proyek baru dari repositori Astro mana pun di GitHub](/id/install-and-setup/#install-from-the-cli-wizard).

  <PackageManagerTabs>
    <Fragment slot="npm">
    ```shell
    npm create astro@latest -- --template starlight
    ```
    </Fragment>
    <Fragment slot="pnpm">
    ```shell
    pnpm create astro@latest --template starlight
    ```
    </Fragment>
    <Fragment slot="yarn">
    ```shell
    yarn create astro --template starlight
    ```
    </Fragment>
  </PackageManagerTabs>

Setelah Anda memiliki proyek Astro baru, Anda dapat menyinkronkan konten GitBook yang sudah ada ke proyek Astro tersebut. GitBook memiliki [fitur Git Sync](https://docs.gitbook.com/product-tour/git-sync) yang akan menyinkronkan konten GitBook Anda secara otomatis ke repositori GitHub/GitLab.

Untuk menyinkronkan langsung ke *content collection* template dokumentasi, tentukan `src/content/docs/en` atau `src/content/docs` sebagai direktori proyek.

:::caution
Saat mengaktifkan Git Sync pastikan untuk memilih prioritas "**GitBook to GitHub**". Ini akan memastikan konten GitBook Anda disinkronkan ke repositori GitHub. Jika tidak, konten GitBook Anda yang ada dapat tertimpa.
:::

Setelah sinkronisasi, Anda sekarang memiliki salinan konten GitBook di repositori Astro Anda. Nonaktifkan Git Sync untuk mencegah sinkronisasi di masa depan dengan GitBook.

Perlu dicatat bahwa meskipun konten Anda kini telah dimigrasikan ke proyek Astro, konten tersebut tidak akan langsung dapat digunakan. Untuk menggunakan konten ini di situs Astro, Anda perlu meluangkan waktu mengubah sintaks GitBook ke format yang kompatibel dengan Astro. Khususnya:

- [Integrasi Markdoc Astro](/id/guides/integrations-guide/markdoc/) mengharuskan ekstensi berkas `.mdoc`. Ini untuk menghindari konflik dengan ekstensi Markdown lain seperti `.mdx` dan `.md`.
- Sintaks GitBook berbeda dari Markdoc di mana prefiks `/` yang menandai *closing tag* diganti menjadi `end` pada berkas GitBook. Anda perlu memperbarui penulisan ini di seluruh berkas.
- Beberapa fitur GitBook bergantung pada komponen khusus. Komponen-komponen ini tidak akan ada di Astro dan harus dibuat serta ditambahkan ke proyek Anda melalui [atribut `tags` pada konfigurasi Markdoc](/id/guides/integrations-guide/markdoc/#use-astro-components-as-markdoc-tags) atau dihapus dari berkas.
