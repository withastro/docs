[Gridsome](https://gridsome.org) adalah static site generator open-source yang dibangun di atas Vue dan GraphQL.

## Persamaan Utama antara Gridsome dan Astro

Gridsome dan Astro memiliki beberapa kesamaan yang akan membantu Anda memigrasikan proyek Anda:

- Baik Gridsome maupun Astro adalah static-site generator JavaScript modern dengan [struktur berkas proyek](/id/basics/project-structure/#directories-and-files) yang serupa. 

- Baik Gridsome maupun Astro menggunakan folder `src/` untuk berkas proyek Anda dan [folder khusus `src/pages/` untuk routing berbasis file](/id/basics/astro-pages/). Membuat dan mengelola halaman situs Anda akan terasa familiar.

- Astro memiliki [integrasi resmi untuk menggunakan komponen Vue](/id/guides/integrations-guide/vue/) dan mendukung [memasang paket NPM](/id/guides/imports/#npm-packages), termasuk beberapa paket untuk Vue. Anda akan dapat menulis komponen UI Vue, dan mungkin bisa mempertahankan sebagian atau seluruh komponen serta dependensi Vue Gridsome yang sudah ada.

- Astro dan Gridsome sama-sama memungkinkan Anda menggunakan [headless CMS, API, atau berkas Markdown untuk data](/id/guides/data-fetching/). Anda dapat terus menggunakan sistem penulisan konten pilihan Anda, dan tetap mempertahankan konten yang sudah ada.

## Perbedaan Utama antara Gridsome dan Astro

Saat Anda membangun ulang situs Gridsome di Astro, Anda akan melihat beberapa perbedaan penting:

- Gridsome adalah aplikasi satu halaman (SPA) berbasis Vue. Situs Astro adalah aplikasi multi-halaman yang dibangun menggunakan [komponen `.astro`](/id/basics/astro-components/), tetapi juga dapat mendukung [React, Preact, Vue.js, Svelte, SolidJS, AlpineJS](/id/guides/framework-components/) dan templating HTML murni.

- Sebagai SPA, Gridsome menggunakan `vue-router` untuk routing SPA, dan `vue-meta` untuk mengelola `<head>`. Di Astro, Anda akan membuat halaman HTML terpisah dan mengontrol `<head>` halaman secara langsung, atau di [komponen layout](/id/basics/layouts/).

- [Data berkas lokal](/id/guides/imports/): Gridsome menggunakan GraphQL untuk mengambil data dari berkas proyek Anda. Astro menggunakan impor ESM dan [`import.meta.glob()`](/id/guides/imports/#importmetaglob) untuk mengimpor data dari berkas lokal proyek. Sumber daya jarak jauh dapat dimuat menggunakan API standar `fetch()`. GraphQL dapat ditambahkan secara opsional ke proyek Anda, namun tidak termasuk secara default.

## Beralih dari Gridsome ke Astro

Untuk mengonversi blog Gridsome ke Astro, mulailah dengan starter template tema blog kami, atau jelajahi lebih banyak tema blog komunitas di [theme showcase](https://astro.build/themes/).

Anda dapat menambahkan argumen `--template` ke perintah `create astro` untuk memulai proyek Astro baru dengan salah satu starter resmi kami. Atau, Anda dapat [memulai proyek baru dari repositori Astro mana pun di GitHub](/id/install-and-setup/#install-from-the-cli-wizard).

  <PackageManagerTabs>
    <Fragment slot="npm">
    ```shell
    npm create astro@latest -- --template blog
    ```
    </Fragment>
    <Fragment slot="pnpm">
    ```shell
    pnpm create astro@latest --template blog
    ```
    </Fragment>
    <Fragment slot="yarn">
    ```shell
    yarn create astro --template blog
    ```
    </Fragment>
  </PackageManagerTabs>

Bawa berkas Markdown (atau MDX, dengan integrasi opsional kami) Anda sebagai konten untuk [membuat halaman Markdown atau MDX](/id/guides/markdown-content/).

Karena struktur proyek Gridsome mirip dengan Astro, Anda mungkin dapat menyalin beberapa berkas yang sudah ada dari proyek Anda ke lokasi yang sama di proyek Astro baru Anda. Namun, kedua struktur proyek tersebut tidak identik. Anda mungkin ingin meninjau [struktur proyek Astro](/id/basics/project-structure/) untuk melihat perbedaannya.

Karena Astro melakukan *query* dan impor berkas lokal Anda dengan cara yang berbeda dari Gridsome, Anda mungkin ingin membaca tentang cara memuat berkas menggunakan [`import.meta.glob()`](/id/guides/imports/#importmetaglob) untuk memahami cara bekerja dengan berkas lokal Anda.

Untuk mengonversi jenis situs lain, seperti portofolio atau situs dokumentasi, lihat lebih banyak starter template resmi di [astro.new](https://astro.new). Anda akan menemukan tautan ke setiap repositori GitHub proyek, serta tautan sekali klik untuk membuka proyek yang berfungsi di lingkungan pengembangan online seperti IDX, StackBlitz, CodeSandbox, dan Gitpod.
