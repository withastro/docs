**Integrasi Astro** menambahkan fungsionalitas dan perilaku baru pada proyek Anda hanya dengan beberapa baris kode. Anda dapat menggunakan integrasi resmi, [integrasi buatan komunitas](#finding-more-integrations), atau bahkan [membangun integrasi kustom sendiri](#building-your-own-integration).

Integrasi dapat…

- Mengaktifkan React, Vue, Svelte, Solid, dan UI framework populer lainnya dengan sebuah [renderer](/id/guides/framework-components/).
- Mengaktifkan rendering on‑demand dengan sebuah [adapter SSR](/id/guides/on-demand-rendering/).
- Mengintegrasikan alat seperti MDX dan Partytown hanya dengan beberapa baris kode.
- Menambahkan fitur baru ke proyek Anda, seperti pembuatan sitemap otomatis.
- Menulis kode kustom yang terhubung ke proses build, dev server, dan lainnya.

:::tip[Direktori integrasi]
Jelajahi atau cari kumpulan lengkap ratusan integrasi resmi dan komunitas di [direktori integrasi](https://astro.build/integrations/). Temukan paket untuk ditambahkan ke proyek Astro Anda untuk autentikasi, analitik, performa, SEO, aksesibilitas, UI, alat developer, dan banyak lagi.
:::

## Penyiapan Integrasi Otomatis

Astro menyertakan perintah `astro add` untuk mengotomatisasi penyiapan integrasi resmi. Beberapa plugin komunitas juga dapat ditambahkan menggunakan perintah ini. Silakan periksa dokumentasi masing‑masing integrasi untuk melihat apakah `astro add` didukung, atau Anda harus [menginstal secara manual](#manual-installation).

Jalankan perintah `astro add` menggunakan package manager pilihan Anda dan wizard integrasi otomatis kami akan memperbarui file konfigurasi Anda dan memasang dependensi yang diperlukan.

<PackageManagerTabs>
  <Fragment slot="npm">
  ```shell
  npx astro add react
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```shell
  pnpm astro add react
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```shell
  yarn astro add react
  ```
  </Fragment>
</PackageManagerTabs>

Anda bahkan bisa menambahkan beberapa integrasi sekaligus!

<PackageManagerTabs>
  <Fragment slot="npm">
  ```shell
  npx astro add react sitemap partytown
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```shell
  pnpm astro add react sitemap partytown
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```shell
  yarn astro add react sitemap partytown
  ```
  </Fragment>
</PackageManagerTabs>

:::note[Menangani dependensi integrasi]
Jika Anda melihat peringatan seperti `Cannot find package '[package-name]'` setelah menambahkan integrasi, package manager Anda mungkin tidak memasang [peer dependencies](https://nodejs.org/id/blog/npm/peer-dependencies/) untuk Anda. Untuk memasang paket yang hilang ini, jalankan perintah berikut:

<PackageManagerTabs>
  <Fragment slot="npm">
  ```shell
  npm install [package-name]
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```shell
  pnpm add [package-name]
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```shell
  yarn add [package-name]
  ```
  </Fragment>
</PackageManagerTabs>
:::

### Instalasi Manual

Integrasi Astro selalu ditambahkan melalui properti `integrations` pada file `astro.config.mjs` Anda.

Ada tiga cara umum untuk mengimpor integrasi ke proyek Astro Anda:

1. [Memasang integrasi paket npm](#installing-an-npm-package).
2. Mengimpor integrasi Anda sendiri dari file lokal di dalam proyek.
3. Menulis integrasi secara inline, langsung di file konfigurasi Anda.

    ```js
    // astro.config.mjs
    import { defineConfig } from 'astro/config';
    import installedIntegration from '@astrojs/vue';
    import localIntegration from './my-integration.js';

    export default defineConfig({
      integrations: [
        // 1. Diimpor dari paket npm terpasang
        installedIntegration(),
        // 2. Diimpor dari file JS lokal
        localIntegration(),
        // 3. Objek inline
        {name: 'namespace:id', hooks: { /* ... */ }},
      ]
    });
    ```

Lihat referensi [Integration API](/id/reference/integrations-reference/) untuk mempelajari semua cara menulis sebuah integrasi.

#### Memasang paket NPM

Pasang integrasi paket NPM menggunakan package manager, lalu perbarui `astro.config.mjs` secara manual.

Sebagai contoh, untuk memasang integrasi `@astrojs/sitemap`:

<Steps>
1. Pasang integrasi ke dependensi proyek menggunakan package manager pilihan Anda:

   <PackageManagerTabs>
     <Fragment slot="npm">
     ```shell
     npm install @astrojs/sitemap
     ```
     </Fragment>
     <Fragment slot="pnpm">
     ```shell
     pnpm add @astrojs/sitemap
     ```
     </Fragment>
     <Fragment slot="yarn">
     ```shell
     yarn add @astrojs/sitemap
     ```
     </Fragment>
   </PackageManagerTabs>

2. Impor integrasi ke file `astro.config.mjs`, dan tambahkan ke array `integrations[]`, beserta opsi konfigurasi apa pun:

    ```js title="astro.config.mjs" ins={2} ins="sitemap()"
    import { defineConfig } from 'astro/config';
    import sitemap from '@astrojs/sitemap';

    export default defineConfig({
      // ...
      integrations: [sitemap()],
      // ...
    });
    ```

    Perhatikan bahwa integrasi yang berbeda dapat memiliki pengaturan konfigurasi yang berbeda. Baca dokumentasi masing‑masing integrasi, dan terapkan opsi konfigurasi yang diperlukan pada integrasi pilihan Anda di `astro.config.mjs`.
</Steps>

### Opsi Kustom

Integrasi hampir selalu ditulis sebagai *factory function* yang mengembalikan objek integrasi. Ini memungkinkan Anda melewatkan argumen dan opsi ke *factory function* untuk menyesuaikan integrasi bagi proyek Anda.

```js
integrations: [
  // Contoh: Kustomisasi integrasi Anda dengan argumen fungsi
  sitemap({filter: true})
]
```

### Mengaktifkan/Menonaktifkan Integrasi

Integrasi yang bernilai falsy akan diabaikan, jadi Anda dapat menyalakan/mematikan integrasi tanpa khawatir meninggalkan nilai `undefined` atau boolean.

```js
integrations: [
  // Contoh: Lewati pembuatan sitemap di Windows
  process.platform !== 'win32' && sitemap()
]
```

## Meningkatkan Versi Integrasi

Untuk meningkatkan semua integrasi resmi sekaligus, jalankan perintah `@astrojs/upgrade`. Ini akan meningkatkan Astro dan semua integrasi resmi ke versi terbaru.

### Peningkatan Otomatis

<PackageManagerTabs>
  <Fragment slot="npm">
  ```shell
  # Upgrade Astro dan integrasi resmi bersama ke versi terbaru
  npx @astrojs/upgrade
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```shell
  # Upgrade Astro dan integrasi resmi bersama ke versi terbaru
  pnpm dlx @astrojs/upgrade
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```shell
  # Upgrade Astro dan integrasi resmi bersama ke versi terbaru
  yarn dlx @astrojs/upgrade
  ```
  </Fragment>
</PackageManagerTabs>

### Peningkatan Manual

Untuk meningkatkan satu atau beberapa integrasi secara manual, gunakan perintah yang sesuai untuk package manager Anda.

<PackageManagerTabs>
  <Fragment slot="npm">
  ```shell
  # Contoh: upgrade integrasi React dan Partytown
  npm install @astrojs/react@latest @astrojs/partytown@latest
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```shell
  # Contoh: upgrade integrasi React dan Partytown
  pnpm add @astrojs/react@latest @astrojs/partytown@latest
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```shell
  # Contoh: upgrade integrasi React dan Partytown
  yarn add @astrojs/react@latest @astrojs/partytown@latest
  ```
  </Fragment>
</PackageManagerTabs>

## Menghapus Integrasi

<Steps>
1. Untuk menghapus integrasi, pertama copot integrasi dari proyek Anda.

    <PackageManagerTabs>
      <Fragment slot="npm">
      ```shell
      npm uninstall @astrojs/react
      ```
      </Fragment>
      <Fragment slot="pnpm">
      ```shell
      pnpm remove @astrojs/react
      ```
      </Fragment>
      <Fragment slot="yarn">
      ```shell
      yarn remove @astrojs/react
      ```
      </Fragment>
    </PackageManagerTabs>

2. Selanjutnya, hapus integrasi dari file `astro.config.*` Anda:

    ```js title="astro.config.mjs" del={3,7}
    import { defineConfig } from 'astro/config';

    import react from '@astrojs/react';

    export default defineConfig({
      integrations: [
        react()
      ]
    });
    ```
</Steps>

Lihat referensi [Integration API](/id/reference/integrations-reference/) untuk mempelajari apa saja yang bisa dilakukan integrasi dan cara menulisnya sendiri.
