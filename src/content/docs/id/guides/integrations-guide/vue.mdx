**[Integrasi Astro][astro-integration]** ini memungkinkan rendering dan hidrasi sisi klien untuk komponen [Vue 3](https://vuejs.org/) Anda.

## Instalasi

Astro menyertakan perintah `astro add` untuk mengotomatisasi pemasangan integrasi resmi. Jika Anda mau, Anda dapat [menginstal integrasi secara manual](#manual-install) sebagai gantinya.

Untuk memasang `@astrojs/vue`, jalankan perintah berikut dari direktori proyek Anda dan ikuti petunjuknya:

<PackageManagerTabs>
  <Fragment slot="npm">
  ```sh
  npx astro add vue
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```sh
  pnpm astro add vue
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```sh
  yarn astro add vue
  ```
  </Fragment>
</PackageManagerTabs>

Jika Anda mengalami masalah, [silakan laporkan kepada kami di GitHub](https://github.com/withastro/astro/issues) dan coba langkah instalasi manual di bawah.

### Instalasi Manual

Pertama, pasang paket `@astrojs/vue`:

<PackageManagerTabs>
  <Fragment slot="npm">
  ```sh
  npm install @astrojs/vue
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```sh
  pnpm add @astrojs/vue
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```sh
  yarn add @astrojs/vue
  ```
  </Fragment>
</PackageManagerTabs>


Sebagian besar manajer paket juga akan memasang peer dependency terkait. Jika Anda melihat peringatan `Cannot find package 'vue'` (atau serupa) saat menjalankan Astro, Anda perlu memasang Vue:

<PackageManagerTabs>
  <Fragment slot="npm">
  ```sh
  npm install vue
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```sh
  pnpm add vue
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```sh
  yarn add vue
  ```
  </Fragment>
</PackageManagerTabs>

Lalu, terapkan integrasi ke file `astro.config.*` Anda menggunakan properti `integrations`:

```js ins={2} ins="vue()" title="astro.config.mjs"
import { defineConfig } from 'astro/config';
import vue from '@astrojs/vue';

export default defineConfig({
  // ...
  integrations: [vue()],
});
```

## Opsi

Integrasi ini didukung oleh `@vitejs/plugin-vue`. Untuk menyesuaikan compiler Vue, opsi dapat diberikan ke integrasi. Lihat [docs](https://www.npmjs.com/package/@vitejs/plugin-vue) `@vitejs/plugin-vue` untuk detail lebih lanjut.

```js title="astro.config.mjs"
import { defineConfig } from 'astro/config';
import vue from '@astrojs/vue';

export default defineConfig({
  // ...
  integrations: [
    vue({
      template: {
        compilerOptions: {
          // perlakukan tag yang diawali ion- sebagai custom element
          isCustomElement: (tag) => tag.startsWith('ion-'),
        },
      },
      // ...
    }),
  ],
});
```

### `appEntrypoint`

<p>

**Type:** `string`<br />
<Since pkg="@astrojs/vue" v="1.2.0" />
</p>

Anda dapat memperluas instance `app` Vue dengan menyetel opsi `appEntrypoint` ke sebuah import specifier relatifâ€‘root (contoh, `appEntrypoint: "/src/pages/_app"`).

Ekspor default file ini harus berupa fungsi yang menerima instance `App` Vue sebelum rendering, memungkinkan penggunaan [plugin Vue kustom](https://vuejs.org/guide/reusability/plugins.html), `app.use`, dan kustomisasi lain untuk kasus penggunaan lanjutan.

```js title="astro.config.mjs"
import { defineConfig } from 'astro/config';
import vue from '@astrojs/vue';

export default defineConfig({
  // ...
  integrations: [vue({ appEntrypoint: '/src/pages/_app' })],
});
```

```ts title="src/pages/_app.ts"
import type { App } from 'vue';
import i18nPlugin from 'my-vue-i18n-plugin';

export default (app: App) => {
  app.use(i18nPlugin);
};
```

### `jsx`

<p>

**Type:** `boolean | object`<br />
<Since pkg="@astrojs/vue" v="1.2.0" />
</p>

Anda dapat menggunakan Vue JSX dengan menyetel `jsx: true`.

```js title="astro.config.mjs"
import { defineConfig } from 'astro/config'];
import vue from '@astrojs/vue';

export default defineConfig({
  // ...
  integrations: [vue({ jsx: true })],
});
```

Ini akan mengaktifkan rendering untuk komponen Vue dan Vue JSX. Untuk menyesuaikan compiler Vue JSX, berikan objek opsi alihâ€‘alih boolean. Lihat [docs](https://www.npmjs.com/package/@vitejs/plugin-vue-jsx) `@vitejs/plugin-vue-jsx` untuk detail lebih lanjut.

```js title="astro.config.mjs"
import { defineConfig } from 'astro/config';
import vue from '@astrojs/vue';

export default defineConfig({
  // ...
  integrations: [
    vue({
      jsx: {
        // perlakukan tag yang diawali ion- sebagai custom element
        isCustomElement: (tag) => tag.startsWith('ion-'),
      },
    }),
  ],
});
```

### `devtools`

<p>

**Type:** `boolean | object`<br />
<Since pkg="@astrojs/vue" v="4.2.0" />
</p>

Anda dapat mengaktifkan [Vue DevTools](https://devtools-next.vuejs.org/) saat development dengan memberikan objek `devtools: true` pada konfigurasi integrasi `vue()` Anda:

```js title="astro.config.mjs"
import { defineConfig } from 'astro/config';
import vue from '@astrojs/vue';

export default defineConfig({
  // ...
  integrations: [vue({ devtools: true })],
});
```

#### Kustomisasi Vue DevTools

<p><Since pkg="@astrojs/vue" v="4.3.0" /></p>

Untuk kustomisasi lebih lanjut, Anda dapat memberikan opsi yang didukung oleh [Vue DevTools Vite Plugin](https://devtools-next.vuejs.org/guide/vite-plugin#options). (Catatan: `appendTo` tidak didukung.)

Sebagai contoh, Anda dapat menyetel `launchEditor` ke editor pilihan Anda bila tidak menggunakan Visual Studio Code:

```js title="astro.config.mjs"
import { defineConfig } from "astro/config";
import vue from "@astrojs/vue";

export default defineConfig({
  // ...
  integrations: [
    vue({
      devtools: { launchEditor: "webstorm" },
    }),
  ],
});
```
