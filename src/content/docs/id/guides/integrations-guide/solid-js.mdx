Integrasi **[Astro][astro-integration]** ini memungkinkan rendering dan hydration sisi klien untuk komponen [SolidJS](https://www.solidjs.com/) Anda.

## Instalasi

Astro menyertakan perintah `astro add` untuk mengotomatisasi penyiapan integrasi resmi. Jika Anda mau, Anda juga bisa [menginstal integrasi secara manual](#manual-install) sebagai alternatif.

Untuk memasang `@astrojs/solid-js`, jalankan perintah berikut dari direktori proyek Anda dan ikuti petunjuknya:

<PackageManagerTabs>
  <Fragment slot="npm">
  ```sh
  npx astro add solid
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```sh
  pnpm astro add solid
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```sh
  yarn astro add solid
  ```
  </Fragment>
</PackageManagerTabs>

Jika Anda mengalami kendala, [silakan laporkan di GitHub](https://github.com/withastro/astro/issues) dan coba langkah instalasi manual di bawah ini.

### Manual Install

Pertama, instal paket `@astrojs/solid-js`:

<PackageManagerTabs>
  <Fragment slot="npm">
  ```sh
  npm install @astrojs/solid-js
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```sh
  pnpm add @astrojs/solid-js
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```sh
  yarn add @astrojs/solid-js
  ```
  </Fragment>
</PackageManagerTabs>

Sebagian besar manajer paket juga akan memasang peer dependency terkait. Jika Anda melihat peringatan `Cannot find package 'solid-js'` (atau serupa) saat menjalankan Astro, Anda perlu memasang SolidJS:

<PackageManagerTabs>
  <Fragment slot="npm">
  ```sh
  npm install solid-js
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```sh
  pnpm add solid-js
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```sh
  yarn add solid-js
  ```
  </Fragment>
</PackageManagerTabs>

Lalu, terapkan integrasi ke berkas `astro.config.*` Anda menggunakan properti `integrations`:

```js title="astro.config.mjs" ins={2} ins="solidJs()"
import { defineConfig } from 'astro/config';
import solidJs from '@astrojs/solid-js';

export default defineConfig({
  // ...
  integrations: [solidJs()],
});
```

Dan tambahkan konfigurasi berikut pada berkas `tsconfig.json`:

```json title="tsconfig.json" ins={5-8}
{
  "extends": "astro/tsconfigs/strict",
  "include": [".astro/types.d.ts", "**/*"],
  "exclude": ["dist"],
  "compilerOptions": {
    "jsx": "preserve",
    "jsxImportSource": "solid-js"
  }
}
```

## Opsi

### Menggabungkan beberapa framework JSX

Saat Anda menggunakan beberapa framework JSX (React, Preact, Solid) dalam satu proyek, Astro perlu menentukan transformasi spesifik framework mana yang harus digunakan untuk masing-masing komponen Anda. Jika Anda hanya menambahkan satu integrasi framework JSX, Anda tidak memerlukan konfigurasi tambahan.

Gunakan opsi konfigurasi `include` (wajib) dan `exclude` (opsional) untuk menentukan berkas mana yang dimiliki oleh framework tertentu. Berikan array berisi berkas dan/atau folder pada `include` untuk setiap framework yang Anda gunakan. Pola wildcard dapat digunakan untuk memasukkan banyak path berkas sekaligus.

Kami merekomendasikan menempatkan komponen framework serupa dalam folder yang sama (mis. `/components/react/` dan `/components/solid/`) agar lebih mudah menentukan daftar `include`, namun ini tidak wajib:

```js title="astro.config.mjs"
import { defineConfig } from 'astro/config';
import preact from '@astrojs/preact';
import react from '@astrojs/react';
import svelte from '@astrojs/svelte';
import vue from '@astrojs/vue';
import solid from '@astrojs/solid-js';

export default defineConfig({
  // Aktifkan banyak framework untuk mendukung berbagai jenis komponen.
  // Tidak perlu `include` jika Anda hanya menggunakan satu framework JSX!
  integrations: [
    preact({
      include: ['**/preact/*'],
    }),
    react({
      include: ['**/react/*'],
    }),
    solid({
      include: ['**/solid/*', '**/node_modules/@suid/material/**'],
    }),
  ],
});
```

## Pemakaian

Gunakan komponen SolidJS seperti halnya [komponen framework UI](/id/guides/framework-components/) lainnya.

### Batas Suspense (Suspense Boundaries)

Untuk mendukung Solid Resources dan Lazy Components tanpa konfigurasi berlebihan, komponen server-only dan komponen yang di-hydrate secara otomatis dibungkus dengan Suspense boundary tingkat atas dan dirender di server menggunakan fungsi [`renderToStringAsync`][solid-render-to-string-async]. Dengan demikian, Anda tidak perlu menambahkan Suspense boundary tingkat atas di sekitar komponen asinkron.

Sebagai contoh, Anda dapat menggunakan [`createResource`][solid-create-resource] milik Solid untuk mengambil data jarak jauh secara asinkron di server. Data tersebut akan disertakan di HTML hasil render server awal dari Astro:

```tsx
// CharacterName.tsx
function CharacterName() {
  const [name] = createResource(() =>
    fetch('https://swapi.dev/api/people/1')
      .then((result) => result.json())
      .then((data) => data.name)
  );

  return (
    <>
      <h2>Name:</h2>
      {/* Luke Skywalker */}
      <div>{name()}</div>
    </>
  );
}
```

Demikian pula, [Lazy Components][solid-lazy-components] Solid juga akan di-resolve dan HTML-nya disertakan pada halaman hasil render server awal.

Komponen non-hydration dengan [`client:only`][astro-client-only] tidak akan dibungkus otomatis dalam Suspense boundary.

Silakan tambahkan Suspense boundary tambahan sesuai kebutuhan dan preferensi Anda.
