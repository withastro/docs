[Sentry](https://sentry.io) menawarkan layanan pemantauan aplikasi dan pelacakan error yang komprehensif, dirancang untuk membantu pengembang mengidentifikasi, mendiagnosis, dan menyelesaikan masalah secara waktu nyata.

Baca selengkapnya di blog kami tentang [kemitraan Astro dengan Sentry](https://astro.build/blog/sentry-official-monitoring-partner/) dan aplikasi toolbar dev Spotlight milik Sentry yang menghadirkan overlay debug kaya ke dalam lingkungan pengembangan Astro Anda. Spotlight menampilkan error, trace, dan konteks penting langsung di browser Anda selama pengembangan lokal.

SDK Astro Sentry memungkinkan pelaporan otomatis error dan data tracing di aplikasi Astro Anda.

## Konfigurasi Proyek

Daftar lengkap prasyarat dapat ditemukan di [panduan Sentry untuk Astro](https://docs.sentry.io/platforms/javascript/guides/astro/#prerequisites).

## Instal

Sentry menangkap data dengan menggunakan SDK di runtime aplikasi Anda.

Instal SDK dengan menjalankan perintah berikut untuk manajer paket pilihan Anda di Astro CLI:

<PackageManagerTabs>
  <Fragment slot="npm">
  ```shell
  npx astro add @sentry/astro
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```shell
  pnpm astro add @sentry/astro
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```shell
  yarn astro add @sentry/astro
  ```
  </Fragment>
</PackageManagerTabs>

Astro CLI akan memasang paket SDK dan menambahkan integrasi Sentry ke berkas `astro.config.mjs` Anda.

## Konfigurasi

Untuk mengonfigurasi integrasi Sentry, Anda perlu menyediakan kredensial berikut di berkas `astro.config.mjs` Anda.

1. **Client key (DSN)** - Anda dapat menemukan DSN di pengaturan proyek Sentry Anda di bawah *Client keys (DSN)*.
2. **Nama proyek** - Anda dapat menemukan nama proyek di pengaturan proyek Sentry Anda di bawah *General settings*.
3. **Auth token** - Anda dapat membuat auth token di pengaturan organisasi Sentry Anda di bawah *Auth tokens*.

:::note
Jika Anda membuat proyek Sentry baru, pilih Astro sebagai platform Anda untuk mendapatkan semua informasi yang diperlukan guna mengonfigurasi SDK.
:::

```js title="astro.config.mjs" ins={2, 6-12}
import { defineConfig } from 'astro/config';
import sentry from '@sentry/astro';

export default defineConfig({
  integrations: [
    sentry({
      dsn: 'https://examplePublicKey@o0.ingest.sentry.io/0',
      sourceMapsUploadOptions: {
        project: 'example-project',
        authToken: process.env.SENTRY_AUTH_TOKEN,
      },
    }),
  ],
});
```
Setelah Anda mengonfigurasi `sourceMapsUploadOptions` dan menambahkan `dsn`, SDK akan secara otomatis menangkap dan mengirim error serta event performa ke Sentry.

## Uji konfigurasi Anda

Tambahkan elemen `<button>` berikut ke salah satu halaman `.astro` Anda. Ini akan memungkinkan Anda memicu error secara manual sehingga Anda dapat menguji proses pelaporan error.

```astro title="src/pages/index.astro"
<button onclick="throw new Error('This is a test error')">Throw test error</button>
```

Untuk melihat dan menyelesaikan error yang tercatat, masuk ke [sentry.io](https://sentry.io/) dan buka proyek Anda.
