Astro adalah framework yang fleksibel dan tidak beropini yang memungkinkan Anda mengonfigurasi proyek dengan banyak cara berbeda. Ini berarti memulai proyek baru bisa terasa membingungkan: tidak ada "satu cara terbaik" untuk menyiapkan proyek Astro Anda!

Panduan dalam bagian "Configuration" ini akan membantu Anda mengenali berbagai berkas yang memungkinkan Anda mengonfigurasi dan menyesuaikan aspek proyek serta lingkungan pengembangan Anda.

Jika ini adalah proyek Astro pertama Anda, atau jika sudah lama sejak terakhir kali Anda menyiapkan proyek baru, gunakan panduan berikut dan referensi dokumentasi untuk bantuan.

## The Astro config File

[Berkas konfigurasi Astro](/id/reference/configuration-reference/) adalah berkas JavaScript yang disertakan di root setiap proyek *starter*:

```js
// astro.config.mjs
import { defineConfig } from "astro/config";

export default defineConfig({
  // opsi konfigurasi Anda di sini...
});
```

Berkas ini hanya diperlukan jika Anda memiliki sesuatu yang perlu dikonfigurasi, namun sebagian besar proyek akan menggunakannya. *Helper* `defineConfig()` menyediakan IntelliSense otomatis di IDE Anda dan menjadi tempat Anda menambahkan semua opsi konfigurasi untuk memberi tahu Astro bagaimana membangun dan merender proyek Anda ke HTML.

Kami merekomendasikan menggunakan format berkas default `.mjs` pada sebagian besar kasus, atau `.ts` jika Anda ingin menulis TypeScript di berkas konfigurasi. Namun, `astro.config.js` dan `astro.config.cjs` juga didukung.

<ReadMore>Baca [referensi konfigurasi](/id/reference/configuration-reference/) Astro untuk gambaran lengkap semua opsi konfigurasi yang didukung.</ReadMore>

## The TypeScript config File

Setiap proyek *starter* Astro menyertakan berkas `tsconfig.json`. [Skrip komponen](/id/basics/astro-components/#the-component-script) Astro adalah TypeScript, yang menyediakan *editor tooling* Astro dan memungkinkan Anda secara opsional menambahkan sintaks ke JavaScript untuk pengecekan tipe pada kode proyek Anda sendiri.

Gunakan berkas `tsconfig.json` untuk mengonfigurasi templat TypeScript yang akan melakukan pengecekan tipe pada kode Anda, mengonfigurasi plugin TypeScript, mengatur alias impor, dan lainnya.

<ReadMore>Baca [panduan TypeScript](/id/guides/typescript/) Astro untuk gambaran lengkap opsi TypeScript dan tipe utilitas bawaan Astro.</ReadMore>

## Development Experience

Saat bekerja dalam *development mode*, Anda dapat memanfaatkan editor kode dan alat lainnya untuk meningkatkan pengalaman pengembang Astro.

Astro menyediakan ekstensi VS Code resmi dan kompatibel dengan beberapa alat editor populer lainnya. Astro juga menyediakan toolbar yang dapat disesuaikan dan tampil di pratinjau browser saat *dev server* berjalan. Anda dapat memasang, bahkan membangun, aplikasi toolbar Anda sendiri untuk fungsionalitas tambahan.

<ReadMore>Baca panduan Astro tentang [opsi penyiapan editor](/id/editor-setup/) dan [menggunakan *dev toolbar*](/id/guides/dev-toolbar/) untuk mempelajari cara menyesuaikan pengalaman pengembangan Anda.</ReadMore>

## Common new project tasks

Berikut beberapa langkah awal yang bisa Anda lakukan pada proyek Astro baru.

### Add your deployment domain

Untuk membuat *sitemap* dan URL kanonis, konfigurasikan URL *deployment* Anda di opsi [`site`](/id/reference/configuration-reference/#site). Jika Anda melakukan *deploy* ke sebuah *path* (mis. `www.example.com/docs`), Anda juga dapat mengonfigurasi [`base`](/id/reference/configuration-reference/#base) sebagai root proyek Anda.

Selain itu, host *deployment* yang berbeda dapat memiliki perilaku berbeda terkait *trailing slash* di akhir URL Anda. (mis. `example.com/about` vs `example.com/about/`). Setelah situs Anda *deploy*, Anda mungkin perlu mengonfigurasi preferensi [`trailingSlash`](/id/reference/configuration-reference/#trailingslash).

```js title="astro.config.mjs"
import { defineConfig } from "astro/config";

export default defineConfig({
  site: "https://www.example.com",
  base: "/docs",
  trailingSlash: "always",
});
```

### Add site metadata

Astro tidak menggunakan berkas konfigurasinya untuk SEO umum atau *meta data*, hanya untuk informasi yang diperlukan untuk membangun kode proyek Anda dan merendernya ke HTML.

Sebagai gantinya, informasi ini ditambahkan ke `<head>` halaman Anda menggunakan tag HTML standar `<link>` dan `<meta>`, seperti saat Anda menulis halaman HTML biasa.

Salah satu pola umum untuk situs Astro adalah membuat komponen [`.astro`](/id/basics/astro-components/) `<Head />` yang dapat ditambahkan ke [komponen *layout*](/id/basics/layouts/) umum sehingga berlaku untuk semua halaman Anda.

```astro title="src/components/MainLayout.astro"
---
import Head from "./Head.astro";

const { ...props } = Astro.props;
---
<html>
  <head>
    <meta charset="utf-8">
    <Head />
    <!-- Elemen head tambahan -->
  </head>
  <body>
    <!-- Konten halaman berada di sini -->
  </body>
</html>
```

Karena `Head.astro` hanyalah komponen Astro biasa, Anda dapat mengimpor berkas dan menerima *props* yang diteruskan dari komponen lain, seperti judul halaman tertentu.

```astro title="src/components/Head.astro"
---
import Favicon from "../assets/Favicon.astro";
import SomeOtherTags from "./SomeOtherTags.astro";

const { title = "My Astro Website", ...props } = Astro.props;
---
<link rel="sitemap" href="/sitemap-index.xml">
<title>{title}</title>
<meta name="description" content="Welcome to my new Astro site!">

<!-- Web analytics -->
<script data-goatcounter="https://my-account.goatcounter.com/count" async src="//gc.zgo.at/count.js"></script>

<!-- Open Graph tags -->
<meta property="og:title" content="My New Astro Website" />
<meta property="og:type" content="website" />
<meta property="og:url" content="http://www.example.com/" />
<meta property="og:description" content="Welcome to my new Astro site!" />
<meta property="og:image" content="https://www.example.com/_astro/seo-banner.BZD7kegZ.webp">
<meta property="og:image:alt" content="">

<SomeOtherTags />

<Favicon />
```
